# Что такое back-end разработка

Back-end это скрытая от глаз пользователя часть сайта, на которой происходит большинство процессов, обеспечивающих его корректную работу. Васк-end разработка нацелена на создание программы, которая позволит сайту принимать, обрабатывать и передавать информацию вашему браузеру для формирования видимой части сайта. Без нее сайт был бы статичной картинкой с малой интерактивностью.

Например, вы пишете пост в своем аккаунте Twitter и нажимаете кнопку «Опубликовать». Теперь к работе приступает back-end. Ваш пост передается программе, которая сохраняет его в базе данных. Затем программа выгружает пост из базы, передает его видимой части сайта, и пост появляется в вашей ленте.
# Что такое сервер

Все процессы за пределами пользовательского интерфейса выполняются на сервере. Сервер представляет собой мощный вычислительный компьютер, который всегда подключен к сети интернет и электропитанию.

Он содержит всю информацию, которая нужна для функционирования сайта. Главная задача сервера - обеспечить постоянный доступ к файлам и данным. Именно на нем находятся серверное приложение, АРІ и база данных. Они работают автономно за исключением первичной настройки и внештатных ситуаций
# Что такое API

API (Application Programming Interface) — это Интерфейс Программирования Приложений. Он представлен в виде набора функций, которыми могут воспользоваться серверы других сайтОВ.

Например, вы можете войти в свою любимую игру через Facebook благодаря интерфейсу программирования приложений.

Кроме того, АРІ выступает переводчиком между браузером и сервером: они создаются на основе разных языков программирования. Запрос пользователя поступает на сервер, АРІ обрабатывает и передает его на языке, понятном для серверного приложения.

То же самое происходит при передаче ответа на запрос от сервера к браузеру
# Что такое серверное приложение 

Серверное приложение это программа, которую создают для обработки запросов, поступающих на сервер от пользователей. Серверное приложение создается на базе серверных языков программирования или фреймворков, например, Java, Ruby on Rails, PHP, Node.js и других. Оно получает запросы пользователя и отвечает на них.

Когда сайт отправляет ваши данные для входа (логин и пароль), серверное приложение обрабатывает ваш запрос и сравнивает данные с имеющимися в базе. Задача приложения сравнить данные и ответить пользователю: сообщить об ошибке или дать доступ.
# Что такое база данных

База данных это собранная и хранящаяся в одном месте информация, которую можно легко найти, изменить или обновить.

Компьютерные базы данных содержат большое количество файлов, в которых представлена информация, например, о денежных переводах или взаимодействии с клиентами.

Также в базах данных хранится вся указанная вами информация: логин, пароль, данные вашего профиля и т.д. Когда нужно получить эту информацию на сайте, соответствующий запрос поступает в серверное приложение. После этого запрос обрабатывается, серверное приложение находит в базе ваши данные и отправляет их браузеру.
# Что такое НТТР и HTTPS

НТТР - это протокол (набор правил) передачи данных в интернете. Он помогает браузеру загружать веб-страницы, а серверу — получить информацию, которую пользователь вводит на сайте. НТTPS — это тот же протокол НТТР, но по которому все данные передаются в зашифрованном виде.

Шифровать данные нужно, чтобы хакеры не смогли получить доступ к информации, если перехватят ее. Когда вы вводите что-то на сайте с HTTPS, браузер зашифровывает информацию перед отправкой данных на сервер. Чтобы расшифровать и прочитать её, нужен специальный ключ, который хранится только на сервере.
# Что такое домен
Домен это адрес сайта, который сообщает браузеру, на какой сервер посылать запрос в интернет-пространстве.

Например, выделенная часть URL является доменом сервиса YouTube. Домен состоит из двух частей: имени домена "youtube" и доменной зоны «.com». При вводе этого домена, вы попадаете на сервер YouTube.

Доменные зоны бывают географическими (RU - Россия, EU Европа, US — США и т.д.) или тематическими (COM коммерческая сфера, BIZ - бизнес-сфера и т.д.). Место, куда ведет домен и где хранится сервер, называется хостингом.
# Что такое путь к файлу

Часть URL, которая идет сразу после домена и отделяется символом «/», называется «путь к файлу». Веб-страница, которую вы видите, состоит из нескольких взаимосвязанных файлов. Поэтому, чтобы увидеть нужную страницу сайта, вам необходимо отправить запрос серверу на открытие этих файлов.

Сервер состоит из большого количества разделов (их можно представить в виде папок на компьютере), в которых хранятся файлы. В этом примере все видео по запросу отображаются в разделе «results».

# Что такое строковой запрос

Чтобы получить нужные видео по запросу, клиентская сторона отправляет серверу соответствующий запрос на информацию из базы данных. Для этого используется строковой запрос.

В URL он идет после знака «?» и состоит из трех частей: ключ запроса, имя переменной и значение переменной.

В URL справа ключ запроса «search» означает функцию поиска соответствующих видео для переменной «query» со значением «что такое back-end» и отображает их в браузере.
# Что можно сделать в серверной части сайта

Программирование серверной части (back-end) дает возможность сделать взаимодействие пользователя с сайтом удобнее.

Качественно разработанная серверная часть позволит:
- удобно оформлять информацию; 
- персонализировать веб-страницу;
- контролировать доступ к контенту;
- хранить информацию о сессии;
- отправлять уведомления.
# Как back-end позволяет удобно оформлять информацию

Подумайте, сколько товаров представлено на сайте «М.Видео» или сколько постов было опубликовано в «ВКонтакте». Создавать HTML-страницу для каждого товара или поста было бы затратно по времени.

Программирование серверной части позволяет создать единый НТМL-шаблон для похожих страниц и написать программу, которая будет сама наполнять этот шаблон информацией из базы данных.

# Как back-end позволяет персонализировать сайт

Серверы могут сохранять и использовать информацию о клиентах, чтобы делать веб- страницу персонализированной. Например, многие сайты предлагают сохранить данные ваших кредитных карт, чтобы вам не пришлось вводить их повторно.

Более глубокий анализ привычек пользователя может быть использован для предвидения его интересов и для дальнейших настроек ответов и уведомлений. К примеру, составить список ранее посещенных популярных мест.
# Как back-end контролирует доступ к контенту

Программирование серверной части дает возможность сайтам ограничивать доступ неавторизованных пользователей к данным и показывать лишь ту информацию, которую пользователю разрешено видеть.

Социальная сеть «ВКонтакте» позволяет своим пользователям ограничивать доступ к личным данным, например, разрешать только вашим друзьям просматривать и комментировать профиль пользователя.
# Как back-end хранит информацию о сессии

Серверы умеют использовать механизм, который позволяет сохранять информацию о текущем пользователе (сессии). На основе этой информации сервер дает пользователю определенные ответы.

Благодаря сессиям сайт помнит, что вы были авторизованы, и показывает вашу историю заказов. По такому же принципу сохраняется ваш прогресс в онлайн-играх или переписки в мессенджерах.
# Как back-end отправляет уведомления пользователям

Программисты могут научить сервер посылать общие или персональные уведомления через сайт, смс, электронную почту и другие каналы передачи информации.

Например, вы совершили покупку в крупном онлайн-магазине техники. Через несколько недель вы получите на электронную почту письмо от этого магазина с предложением посмотреть похожие товары.
