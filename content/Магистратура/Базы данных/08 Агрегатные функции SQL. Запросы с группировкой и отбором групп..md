### 1. Агрегатные функции

**Агрегатная функция** — это функция, которая выполняет вычисление на наборе значений (например, на всем столбце) и возвращает **единственное** итоговое значение.

Вот самые распространенные из них:

*   `COUNT()` — подсчитывает количество строк.
*   `SUM()` — вычисляет сумму значений.
*   `AVG()` — вычисляет среднее значение.
*   `MIN()` — находит минимальное значение.
*   `MAX()` — находит максимальное значение.

**Важная особенность:** За исключением `COUNT(*)`, все агрегатные функции игнорируют значения `NULL`.

**Примеры использования без группировки:**

1.  **`COUNT()`**: Посчитать общее количество сотрудников.

    ```sql
    SELECT COUNT(*) AS total_employees FROM Employees;
    ```
    *   `COUNT(*)` считает абсолютно все строки.
    *   `COUNT(column_name)` считает только строки, где в указанном столбце значение **не** `NULL`.

    Например, посчитаем, скольким сотрудникам назначен бонус:
    ```sql
    SELECT COUNT(bonus) AS employees_with_bonus FROM Employees;
    -- Результат будет 5, так как у двоих сотрудников bonus IS NULL.
    ```

2.  **`SUM()`**: Посчитать общую сумму зарплат всех сотрудников.

    ```sql
    SELECT SUM(salary) AS total_salary_fund FROM Employees;
    ```

3.  **`AVG()`**: Найти среднюю зарплату по компании.

    ```sql
    SELECT AVG(salary) AS average_salary FROM Employees;
    ```

4.  **`MIN()` и `MAX()`**: Найти самую низкую и самую высокую зарплату.

    ```sql
    SELECT
        MIN(salary) AS min_salary,
        MAX(salary) AS max_salary
    FROM Employees;
    ```

### 2. Запросы с группировкой (`GROUP BY`)

Что если мы хотим узнать не общую среднюю зарплату, а среднюю зарплату **по каждому отделу**? Или количество сотрудников **в каждой должности**?

Для этого используется клауза `GROUP BY`. Она объединяет строки с одинаковыми значениями в указанных столбцах в одну "группу", чтобы затем применить к каждой группе агрегатную функцию.

**Синтаксис:**

```sql
SELECT
    столбец_группировки_1,
    столбец_группировки_2,
    агрегатная_функция(столбец)
FROM имя_таблицы
GROUP BY
    столбец_группировки_1,
    столбец_группировки_2;
```

**Золотое правило `GROUP BY`:**
Все столбцы в списке `SELECT`, которые **не являются** агрегатными функциями, **должны** быть перечислены в клаузе `GROUP BY`.

**Примеры:**

1.  **Посчитать количество сотрудников в каждом отделе.**

    ```sql
    SELECT
        department,
        COUNT(*) AS number_of_employees
    FROM Employees
    GROUP BY department;
    ```
    *Результат:*

| department | number_of_employees |
| ---------- | ------------------- |
| HR         | 1                   |
| IT         | 4                   |
| Продажи    | 2                   |


2.  **Найти среднюю зарплату и суммарный бонус для каждой должности.**

    ```sql
    SELECT
        position,
        AVG(salary) AS avg_salary,
        SUM(bonus) AS total_bonuses
    FROM Employees
    GROUP BY position;
    ```
    *Результат:*

| position | avg_salary | total_bonuses |
|:---:|:---:|:---:|
| Аналитик | 105000.00 | NULL (или 0) |
| HR-специалист | 75000.00 | NULL (или 0) |
| Менеджер | 112500.00 | 40000.00 |
| Разработчик | 117500.00 | 38000.00 |
| Тестировщик | 90000.00 | 12000.00 |

    *(Примечание: `SUM(bonus)` для Аналитика и HR вернет `NULL`, так как складывать нечего. Часто используют `SUM(COALESCE(bonus, 0))` для получения 0 вместо `NULL`)*

3.  **Группировка по нескольким столбцам:**
    Посчитать количество сотрудников для каждой комбинации "отдел - должность".

    ```sql
    SELECT
        department,
        position,
        COUNT(*) as count_in_position
    FROM Employees
    GROUP BY department, position
    ORDER BY department, position;
    ```

### 3. Отбор групп (`HAVING`)

Мы знаем, как фильтровать строки до группировки — для этого используется `WHERE`.
Но что, если нам нужно отфильтровать данные **после** группировки? Например, показать только те отделы, в которых средняя зарплата превышает 100 000.

Для этого используется клауза `HAVING`. Она работает так же, как `WHERE`, но применяется к группам, а не к отдельным строкам.

**Ключевое различие `WHERE` и `HAVING`:**

*   **`WHERE`** фильтрует **строки** *до* того, как они будут сгруппированы.
*   **`HAVING`** фильтрует **группы** *после* того, как они были созданы `GROUP BY`.
*   В `WHERE` нельзя использовать агрегатные функции. В `HAVING` — можно и нужно.

**Примеры:**

1.  **Найти отделы, в которых работает более 2-х сотрудников.**

    ```sql
    SELECT
        department,
        COUNT(*) AS number_of_employees
    FROM Employees
    GROUP BY department
    HAVING COUNT(*) > 2;
    ```
    *Результат:*

| department | number_of_employees |
|---|---|
| IT | 4 |

2.  **Найти должности, средняя зарплата в которых выше 110 000.**

    ```sql
    SELECT
        position,
        AVG(salary) as avg_salary
    FROM Employees
    GROUP BY position
    HAVING AVG(salary) > 110000
    ORDER BY avg_salary DESC;
    ```

3.  **Комплексный пример: `WHERE`, `GROUP BY` и `HAVING` вместе.**
    Задача: Найти среднюю зарплату по должностям, но учитывать только сотрудников, нанятых после 2020 года. При этом показать только те должности, где средняя зарплата по этим "новым" сотрудникам превышает 90 000.

    ```sql
    SELECT
        position,
        AVG(salary) AS avg_salary_for_new_hires
    FROM Employees
    WHERE hire_date > '2020-12-31'  -- 1. Фильтруем строки ДО группировки
    GROUP BY position               -- 2. Группируем оставшиеся строки
    HAVING AVG(salary) > 90000;     -- 3. Фильтруем группы ПОСЛЕ группировки
    ```

### Логический порядок выполнения SQL-запроса

Понимание порядка, в котором СУБД "думает" о запросе, очень помогает писать их правильно:

1.  `FROM` — определяет рабочую таблицу.
2.  `WHERE` — фильтрует исходные строки.
3.  `GROUP BY` — группирует отфильтрованные строки.
4.  `HAVING` — фильтрует получившиеся группы.
5.  `SELECT` — выбирает столбцы и вычисляет выражения для оставшихся групп.
6.  `ORDER BY` — сортирует финальный результат.
7.  `LIMIT` / `OFFSET` — отбирает "страницу" из отсортированного результата.
