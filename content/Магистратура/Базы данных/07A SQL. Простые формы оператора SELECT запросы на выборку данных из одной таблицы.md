### 1. Простая форма оператора `SELECT`

Оператор `SELECT` используется для выборки данных из таблицы. Его основная структура:

```sql
SELECT список_атрибутов -- что мы хотим увидеть (столбцы)
FROM имя_таблицы;       -- откуда мы берем данные
```

**Терминология:**
*   **Кортеж** (tuple) — это синоним слова **строка** (row) в таблице.
*   **Атрибут** (attribute) — это синоним слова **столбец** (column).

**Примеры:**

1.  **Выбрать все атрибуты (столбцы) для всех кортежей (строк):**
    Звездочка `*` означает "все столбцы".

    ```sql
    SELECT * FROM Employees;
    ```

2.  **Выбрать только определенные атрибуты:**
    Перечисляем нужные столбцы через запятую.

    ```sql
    SELECT first_name, last_name, salary FROM Employees;
    ```

### 2. Условия отбора кортежей (клауза `WHERE`)

Чтобы отфильтровать строки по определенным критериям, используется клауза `WHERE`.

#### Операторы `AND`, `OR`, `NOT`

Эти логические операторы позволяют комбинировать условия:
*   `AND` (И): оба условия должны быть истинными.
*   `OR` (ИЛИ): хотя бы одно из условий должно быть истинным.
*   `NOT` (НЕ): инвертирует результат условия.

**Примеры:**

1.  **`AND`**: Найти разработчиков (`position = 'Разработчик'`) с зарплатой больше 110000.

    ```sql
    SELECT first_name, position, salary
    FROM Employees
    WHERE position = 'Разработчик' AND salary > 110000;
    ```

2.  **`OR`**: Найти всех, кто работает в отделе IT или в отделе продаж.

    ```sql
    SELECT first_name, department
    FROM Employees
    WHERE department = 'IT' OR department = 'Продажи';
    ```

3.  **`NOT`**: Найти всех сотрудников, которые **не** являются менеджерами.

    ```sql
    SELECT first_name, position
    FROM Employees
    WHERE NOT position = 'Менеджер';
    -- Это эквивалентно: WHERE position <> 'Менеджер' или WHERE position != 'Менеджер'
    ```

### 3. Специальные операторы в `WHERE`

#### `BETWEEN` (между)

Используется для проверки попадания значения в диапазон. **Важно:** `BETWEEN` включает границы диапазона.

**Примеры:**

1.  Найти сотрудников с зарплатой от 90000 до 110000 включительно.

    ```sql
    SELECT first_name, salary
    FROM Employees
    WHERE salary BETWEEN 90000 AND 110000;
    -- Эквивалентно: WHERE salary >= 90000 AND salary <= 110000
    ```

2.  Найти сотрудников, нанятых в 2021 году.

    ```sql
    SELECT first_name, hire_date
    FROM Employees
    WHERE hire_date BETWEEN '2021-01-01' AND '2021-12-31';
    ```

#### `LIKE` (похожий на)

Используется для поиска по шаблону в строковых данных.
*   `%` (процент): заменяет любую последовательность символов (включая пустую).
*   `_` (нижнее подчеркивание): заменяет ровно один любой символ.

**Примеры:**

1.  Найти сотрудников, чья фамилия заканчивается на "ов".

    ```sql
    SELECT last_name FROM Employees WHERE last_name LIKE '%ов';
    -- Результат: Иванов, Петров
    ```

2.  Найти сотрудников, чье имя начинается на "А".

    ```sql
    SELECT first_name FROM Employees WHERE first_name LIKE 'А%';
    -- Результат: Анна, Александр
    ```

3.  Найти сотрудников, у которых вторая буква в имени "е".

    ```sql
    SELECT first_name FROM Employees WHERE first_name LIKE '_е%';
    -- Результат: Петр, Сергей, Елена
    ```

#### `IS [NOT] NULL` (является [не] пустым)

`NULL` — это специальное значение, означающее "отсутствие данных". Его нельзя сравнивать с помощью `=` или `<>`. Для проверки на `NULL` используются операторы `IS NULL` и `IS NOT NULL`.

**Примеры:**

1.  Найти сотрудников, у которых **не указан** бонус.

    ```sql
    SELECT first_name, bonus
    FROM Employees
    WHERE bonus IS NULL;
    ```

2.  Найти сотрудников, у которых бонус **указан** (не важно, какой).

    ```sql
    SELECT first_name, bonus
    FROM Employees
    WHERE bonus IS NOT NULL;
    ```

### 4. Скалярные выражения в списке атрибутов

В списке `SELECT` можно не просто указывать столбцы, а производить с ними вычисления (арифметические, строковые и т.д.). Результат такого вычисления — это новый, вычисляемый столбец. Ему можно дать имя с помощью ключевого слова `AS` (псевдоним).

**Примеры:**

1.  **Арифметическое выражение:** Посчитать годовую зарплату.

    ```sql
    SELECT
        first_name,
        last_name,
        salary,
        salary * 12 AS annual_salary  -- вычисляемый столбец с псевдонимом
    FROM Employees;
    ```

2.  **Общий доход:** Посчитать зарплату вместе с бонусом. Функция `COALESCE` возвращает первое не-NULL значение из списка, что позволяет заменить `NULL` на 0 для корректного сложения.

    ```sql
    SELECT
        first_name,
        salary,
        bonus,
        salary + COALESCE(bonus, 0) AS total_income
    FROM Employees;
    ```

3.  **Строковое выражение (конкатенация):** Объединить имя и фамилию в одну строку.
    *   Стандартный SQL: `||`
    *   SQL Server: `+`
    *   MySQL: функция `CONCAT()`

    ```sql
    -- Стандартный SQL (PostgreSQL, Oracle, SQLite)
    SELECT
        first_name || ' ' || last_name AS full_name
    FROM Employees;
    ```

### 5. Сортировка кортежей (`ORDER BY`)

Клауза `ORDER BY` используется для сортировки результирующего набора данных. Она всегда идет в конце запроса.
*   `ASC` (Ascending) — по возрастанию (используется по умолчанию).
*   `DESC` (Descending) — по убыванию.

**Примеры:**

1.  **Сортировка по одному столбцу:** Отсортировать сотрудников по убыванию зарплаты.

    ```sql
    SELECT first_name, salary
    FROM Employees
    ORDER BY salary DESC;
    ```

2.  **Сортировка по нескольким столбцам:** Отсортировать по отделу (в алфавитном порядке), а внутри каждого отдела — по зарплате (по убыванию).

    ```sql
    SELECT first_name, department, salary
    FROM Employees
    ORDER BY department ASC, salary DESC;
    ```
    Сначала все строки сортируются по `department`. Если значения `department` совпадают, то для этих строк применяется вторая сортировка по `salary`.

3.  **Сортировка по вычисляемому столбцу:**

    ```sql
    SELECT
        first_name || ' ' || last_name AS full_name,
        salary
    FROM Employees
    ORDER BY full_name; -- Сортировка по псевдониму
    ```
