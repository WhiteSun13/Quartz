Отличный выбор темы! Команды `INSERT`, `UPDATE` и `DELETE` — это основа языка манипулирования данными (DML - Data Manipulation Language). Они позволяют изменять *содержимое* таблиц, в то время как DDL-команды (`CREATE`, `ALTER`) меняют их *структуру*.

Давайте рассмотрим каждую команду подробно с примерами. Для примеров будем использовать гипотетическую таблицу `Сотрудники`:

```sql
CREATE TABLE Сотрудники (
    ID INT PRIMARY KEY,
    Имя VARCHAR(50),
    Фамилия VARCHAR(50),
    Должность VARCHAR(100),
    ID_Отдела INT,
    Зарплата DECIMAL(10, 2)
);
```

---

### **1. `INSERT` — Добавление новых данных**

Команда `INSERT` используется для добавления одной или нескольких новых строк (записей) в таблицу.

#### **Синтаксис 1: Явное указание столбцов (рекомендуемый способ)**

Это самый надежный и предпочтительный способ. Вы явно перечисляете столбцы, в которые вставляете данные. Порядок значений в `VALUES` должен соответствовать порядку перечисленных столбцов.

**Шаблон:**
```sql
INSERT INTO имя_таблицы (столбец1, столбец2, столбец3)
VALUES (значение1, значение2, значение3);
```

**Преимущества:**
*   **Надежность:** Запрос не сломается, если в таблицу добавят новый столбец или изменят порядок существующих.
*   **Гибкость:** Можно вставлять данные только в определенные столбцы (остальные получат значение по умолчанию или `NULL`, если это разрешено).

**Пример:** Добавим нового сотрудника.
```sql
INSERT INTO Сотрудники (ID, Имя, Фамилия, Должность, ID_Отдела, Зарплата)
VALUES (101, 'Иван', 'Петров', 'Разработчик', 1, 90000.00);
```

#### **Синтаксис 2: Без указания столбцов (не рекомендуется)**

Этот способ короче, но более хрупкий. Вы должны перечислить значения для **всех** столбцов таблицы в том порядке, в котором они были определены при создании таблицы.

**Шаблон:**
```sql
INSERT INTO имя_таблицы
VALUES (значение1, значение2, значение3, ...);
```

**Пример:**
```sql
-- Работает, только если порядок столбцов: ID, Имя, Фамилия, Должность, ID_Отдела, Зарплата
INSERT INTO Сотрудники
VALUES (102, 'Анна', 'Сидорова', 'Аналитик', 2, 85000.00);
```

#### **Синтаксис 3: Вставка нескольких строк**

Большинство СУБД позволяют вставить несколько строк одним запросом, перечисляя наборы значений через запятую. Это гораздо эффективнее, чем выполнять несколько отдельных `INSERT`.

**Шаблон:**
```sql
INSERT INTO имя_таблицы (столбец1, столбец2)
VALUES
    (значение1_1, значение1_2),
    (значение2_1, значение2_2),
    (значение3_1, значение3_2);
```

**Пример:**
```sql
INSERT INTO Сотрудники (ID, Имя, Фамилия, Должность, ID_Отдела, Зарплата)
VALUES
    (103, 'Сергей', 'Васильев', 'Тестировщик', 1, 75000.00),
    (104, 'Ольга', 'Николаева', 'HR-менеджер', 3, 70000.00);
```

#### **Синтаксис 4: Вставка данных из другой таблицы (`INSERT ... SELECT`)**

Очень мощная возможность, позволяющая копировать и преобразовывать данные из одной таблицы в другую.

**Пример:** Предположим, у нас есть таблица `Кандидаты`, и мы хотим перенести всех одобренных кандидатов в таблицу `Сотрудники`.
```sql
INSERT INTO Сотрудники (ID, Имя, Фамилия, Должность, Зарплата)
SELECT ID_Кандидата, Имя, Фамилия, ЖелаемаяДолжность, ПредложеннаяЗарплата
FROM Кандидаты
WHERE Статус = 'Одобрен';
```

### **2. `UPDATE` — Обновление существующих данных**

Команда `UPDATE` используется для изменения значений в одной или нескольких существующих строках.

<br>
<div style="border: 2px solid red; padding: 10px;">
    <strong>ВАЖНО!</strong> Всегда используйте условие <code>WHERE</code> с командой <code>UPDATE</code>. Если вы забудете <code>WHERE</code>, команда обновит <strong>ВСЕ</strong> строки в таблице!
</div>
<br>

**Шаблон:**
```sql
UPDATE имя_таблицы
SET столбец1 = новое_значение1,
    столбец2 = новое_значение2
WHERE условие; -- Условие определяет, какие строки нужно обновить
```

**Пример 1: Обновление одной строки**
Повысим Ивана Петрова до "Ведущего разработчика" и увеличим ему зарплату.
```sql
UPDATE Сотрудники
SET Должность = 'Ведущий разработчик',
    Зарплата = 120000.00
WHERE ID = 101;
```

**Пример 2: Обновление нескольких строк**
Дадим всем сотрудникам отдела с `ID_Отдела = 1` прибавку к зарплате в 10%.
```sql
UPDATE Сотрудники
SET Зарплата = Зарплата * 1.10
WHERE ID_Отдела = 1;
```

### **3. `DELETE` — Удаление данных**

Команда `DELETE` используется для удаления одной или нескольких строк из таблицы.
<br>
<div style="border: 2px solid red; padding: 10px;">
    <strong>ОЧЕНЬ ВАЖНО!</strong> Как и с <code>UPDATE</code>, всегда используйте условие <code>WHERE</code> с командой <code>DELETE</code>. Если вы забудете <code>WHERE</code>, команда удалит <strong>ВСЕ ДАННЫЕ</strong> из таблицы безвозвратно (если нет бэкапа).
</div>


**Шаблон:**
```sql
DELETE FROM имя_таблицы
WHERE условие; -- Условие определяет, какие строки нужно удалить
```

**Пример 1: Удаление одной строки**
Удалим сотрудника с `ID = 103`.
```sql
DELETE FROM Сотрудники
WHERE ID = 103;
```

**Пример 2: Удаление нескольких строк по условию**
Удалим всех сотрудников с зарплатой меньше 50000.
```sql
DELETE FROM Сотрудники
WHERE Зарплата < 50000.00;
```

**Пример 3: Удаление ВСЕХ строк из таблицы (ОПАСНО!)**
Этот запрос очистит таблицу `Сотрудники` полностью.
```sql
-- Выполняйте, только если вы на 100% уверены, что хотите очистить таблицу
DELETE FROM Сотрудники;
```

### **Краткое сравнение: `DELETE` vs `TRUNCATE`**

Для удаления всех данных из таблицы есть еще одна команда — `TRUNCATE`. Важно понимать разницу.

| Критерий                        | `DELETE FROM table;`                                  | `TRUNCATE TABLE table;`                         |
| :------------------------------ | :---------------------------------------------------- | :---------------------------------------------- |
| **Что делает**                  | Удаляет строки одну за другой (построчно).            | Мгновенно удаляет все данные, освобождая место. |
| **Скорость**                    | Медленнее на больших таблицах.                        | Очень быстро.                                   |
| **Условие `WHERE`**             | Можно использовать.                                   | Нельзя использовать (всегда удаляет всё).       |
| **Триггеры**                    | Активирует триггеры `ON DELETE` для каждой строки.    | Не активирует триггеры.                         |
| **Логирование**                 | Каждое удаление строки записывается в лог транзакций. | Минимальное логирование (только сама операция). |
| **Счетчики (`AUTO_INCREMENT`)** | Не сбрасывает счетчик.                                | **Сбрасывает** счетчик на начальное значение.   |
| **Откат (Rollback)**            | Можно откатить в рамках транзакции.                   | Не всегда можно откатить (зависит от СУБД).     |

**Вывод:**
*   Используйте `DELETE`, когда нужно удалить **конкретные строки** по условию.
*   Используйте `TRUNCATE`, когда нужно **полностью и быстро очистить** таблицу и сбросить ее к "заводским настройкам" (например, при тестировании).
