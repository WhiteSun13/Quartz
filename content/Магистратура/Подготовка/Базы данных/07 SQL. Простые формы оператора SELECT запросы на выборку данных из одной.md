Отлично, переходим к практической части работы с данными — языку SQL и его основному оператору `SELECT`.

### **SQL. Оператор `SELECT`**

`SELECT` — это основной оператор языка SQL, предназначенный для извлечения (выборки) данных из одной или нескольких таблиц базы данных. Результатом выполнения запроса `SELECT` всегда является новая таблица (набор результатов), состоящая из строк (`кортежей`) и столбцов (`атрибутов`).

#### **Общая структура простого запроса**

```sql
SELECT список_атрибутов -- Что выбираем (столбцы)
FROM имя_таблицы       -- Откуда выбираем (таблица)
WHERE условие_отбора;  -- Какие строки выбираем (фильтр)
ORDER BY поле_сортировки [ASC | DESC]; -- Как сортируем результат
```

*   `SELECT` и `FROM` — обязательные части.
*   `WHERE` и `ORDER BY` — необязательные.

Для всех примеров будем использовать воображаемую таблицу `Employees` (Сотрудники):

**Таблица `Employees`**

### **1. Простые формы оператора `SELECT` (выборка из одной таблицы)**

**1.1. Выборка всех столбцов**

Используется символ `*` (звёздочка), который означает "все атрибуты".
```sql
-- Выбрать все данные из таблицы Employees
SELECT * FROM Employees;
```
*Результат: вся таблица, представленная выше.*

**1.2. Выборка определенных столбцов**

Перечисляем имена нужных столбцов через запятую.

```sql
-- Выбрать только имя, фамилию и должность сотрудников
SELECT first_name, last_name, position FROM Employees;
```
*Результат:*

| first_name | last_name | position |
| ---------- | --------- | -------- |
| Иван       | Иванов    | Менеджер |
| Петр       | Петров    | Аналитик |
| ...        | ...       | ...      |

### **2. Использование операторов в `WHERE` для отбора кортежей (строк)**

Клауза `WHERE` фильтрует строки, оставляя в результате только те, которые удовлетворяют заданному условию.

#### **`BETWEEN ... AND ...`**

Выбирает значения в заданном диапазоне (включая границы).

```sql
-- Найти сотрудников с зарплатой от 100000 до 120000 включительно
SELECT * FROM Employees
WHERE salary BETWEEN 100000 AND 120000;
```
*Результат: строки с id 1, 3, 5.*

Можно использовать и с датами:
```sql
-- Найти сотрудников, нанятых в 2020 году
SELECT * FROM Employees
WHERE hire_date BETWEEN '2020-01-01' AND '2020-12-31';
```
*Результат: строки с id 1, 3.*

#### **`LIKE`**

Используется для поиска по шаблону в текстовых полях.
*   `%` (процент) — заменяет любую последовательность символов (в том числе пустую).
*   `_` (подчеркивание) — заменяет ровно один любой символ.

```sql
-- Найти всех, чья фамилия заканчивается на "ов"
SELECT * FROM Employees
WHERE last_name LIKE '%ов';
```
*Результат: строки с id 1, 2, 4, 6.*

```sql
-- Найти всех, чье имя состоит из 4 букв и начинается на "Анн"
SELECT * FROM Employees
WHERE first_name LIKE 'Анн_';
```
*Результат: строка с id 3 (Анна).*

#### **`IS NULL` и `IS NOT NULL`**

Проверяет, является ли значение в поле `NULL` (пустым, неопределенным). **Важно:** `NULL` нельзя сравнивать через = (например, `manager_id = NULL` не сработает). Нужно использовать `IS`.

```sql
-- Найти сотрудника, у которого нет руководителя
SELECT * FROM Employees
WHERE manager_id IS NULL;
```
*Результат: строка с id 1 (Иван Иванов).*

```sql
-- Найти всех сотрудников, у которых есть руководитель
SELECT * FROM Employees
WHERE manager_id IS NOT NULL;
```
*Результат: все, кроме Ивана Иванова.*

#### **`AND`, `OR`, `NOT`**

Логические операторы для комбинирования условий:
*   `AND` — И (оба условия должны быть истинны).
*   `OR` — ИЛИ (хотя бы одно условие должно быть истинно).
*   `NOT` — НЕ (инвертирует условие).

```sql
-- Найти всех аналитиков с зарплатой больше 100000
SELECT * FROM Employees
WHERE position = 'Аналитик' AND salary > 100000;
```
*Результат: строка с id 5 (Мария Кузнецова).*

```sql
-- Найти всех программистов или тимлидов
SELECT * FROM Employees
WHERE position = 'Программист' OR position = 'Тимлид';
```
*Результат: строки с id 3, 4, 6.*

При комбинировании `AND` и `OR` используйте скобки `()` для определения порядка операций.
```sql
-- Найти всех аналитиков и программистов, чья зарплата меньше 100000
SELECT * FROM Employees
WHERE (position = 'Аналитик' OR position = 'Программист') AND salary < 100000;
```
*Результат: строки с id 2, 6.*

### **3. Скалярные выражения в списке атрибутов запроса**

В списке `SELECT` можно использовать не только имена столбцов, но и выражения, которые вычисляются для каждой строки.

**3.1. Арифметические операции**
```sql
-- Показать зарплату сотрудников после повышения на 10%
-- AS используется для присвоения столбцу псевдонима (alias)
SELECT first_name, last_name, salary, salary * 1.1 AS new_salary
FROM Employees;
```
*Результат:*

| first_name | last_name | salary | new_salary |
|------------|-----------|--------|------------|
| Иван       | Иванов    | 120000 | 132000.0   |
| Петр       | Петров    | 95000  | 104500.0   |
| ...        | ...       | ...    | ...        |

**3.2. Объединение строк (конкатенация)**
Синтаксис может отличаться в разных СУБД (`||` в PostgreSQL/Oracle, `+` в MS SQL Server, функция `CONCAT()` в MySQL).

```sql
-- Вывести полное имя сотрудника в одном столбце (синтаксис MySQL/PostgreSQL)
SELECT CONCAT(first_name, ' ', last_name) AS full_name
FROM Employees;
```
*Результат:*

| full_name   |
| ----------- |
| Иван Иванов |
| Петр Петров |
| ...         |


**3.3. Использование встроенных функций**
Например, `UPPER()` для перевода в верхний регистр или `YEAR()` для извлечения года из даты.

```sql
-- Показать фамилию в верхнем регистре и год приема на работу
SELECT UPPER(last_name) AS upper_lastname, YEAR(hire_date) AS hire_year
FROM Employees;
```
*Результат:*

| upper_lastname | hire_year |
| -------------- | --------- |
| ИВАНОВ         | 2020      |
| ПЕТРОВ         | 2021      |
| ...            | ...       |

### **4. Сортировка кортежей (`ORDER BY`)**

Клауза `ORDER BY` сортирует строки в результирующем наборе. Она всегда идет после `WHERE`.

*   `ASC` (Ascending) — по возрастанию (используется по умолчанию).
*   `DESC` (Descending) — по убыванию.

**4.1. Сортировка по одному полю**
```sql
-- Отсортировать сотрудников по зарплате по убыванию
SELECT first_name, last_name, salary
FROM Employees
ORDER BY salary DESC;
```
*Результат: первым будет Сергей Сергеев (150000), последним — Петр Петров (95000).*

**4.2. Сортировка по нескольким полям**
Сортировка происходит сначала по первому полю. Если значения в нем совпадают, то для этих строк применяется сортировка по второму полю, и так далее.

```sql
-- Отсортировать сотрудников по должности (в алфавитном порядке),
-- а в рамках одной должности - по зарплате (по убыванию)
SELECT first_name, last_name, position, salary
FROM Employees
ORDER BY position ASC, salary DESC;
```
*Результат: сначала все "Аналитики" (Мария, потом Петр), затем "Менеджер", затем "Программисты" (Анна, потом Андрей) и т.д.*

**4.3. Сортировка по вычисляемому полю**
Можно сортировать по псевдониму вычисляемого столбца.

```sql
-- Вывести полное имя и отсортировать по нему
SELECT CONCAT(first_name, ' ', last_name) AS full_name
FROM Employees
ORDER BY full_name;
```
*Результат: сотрудники будут отсортированы по полному имени в алфавитном порядке.*
