Отличный следующий шаг! Соединение таблиц и подзапросы — это сердце реляционных баз данных. Они позволяют извлекать сложную, взаимосвязанную информацию.

### Подготовительный этап: Создаем связанные таблицы

Для демонстрации нам понадобится несколько таблиц. Модифицируем нашу схему, чтобы она лучше отражала реальную структуру данных (нормализацию).

1.  **`Departments` (Отделы):** Таблица-справочник.
2.  **`Employees` (Сотрудники):** Теперь будет ссылаться на `Departments` через `department_id`. Также добавим `manager_id` для демонстрации соединения таблицы с самой собой.

```sql
-- Удаляем старую таблицу, если она есть
DROP TABLE IF EXISTS Employees;
DROP TABLE IF EXISTS Departments;

-- Создаем таблицу Отделов
CREATE TABLE Departments (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    city VARCHAR(50)
);

-- Заполняем Отделы
INSERT INTO Departments (id, name, city) VALUES
(10, 'Продажи', 'Москва'),
(20, 'IT', 'Санкт-Петербург'),
(30, 'HR', 'Москва'),
(40, 'Финансы', 'Екатеринбург'); -- Отдел без сотрудников для примера

-- Создаем новую таблицу Сотрудников
CREATE TABLE Employees (
    id INT PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    salary DECIMAL(10, 2),
    department_id INT, -- Внешний ключ (ссылка на Departments)
    manager_id INT,    -- Ссылка на другого сотрудника (на самого себя)
    FOREIGN KEY (department_id) REFERENCES Departments(id),
    FOREIGN KEY (manager_id) REFERENCES Employees(id)
);

-- Заполняем Сотрудников
INSERT INTO Employees (id, first_name, last_name, salary, department_id, manager_id) VALUES
(1, 'Александр', 'Кузнецов', 130000, 10, NULL), -- Директор, у него нет менеджера
```

### 1. Способы объединения таблиц (`JOIN`)

`JOIN` — это оператор, который комбинирует строки из двух или более таблиц на основе связанного между ними столбца.

#### `INNER JOIN` (Внутреннее соединение)

Возвращает только те строки, для которых **есть совпадение в обеих таблицах**. Это самый частый тип соединения.

**Задача:** Вывести имя каждого сотрудника и название его отдела.

```sql
SELECT
    e.first_name,
    e.last_name,
    d.name AS department_name
FROM
    Employees AS e
INNER JOIN
    Departments AS d ON e.department_id = d.id;
```
*   `Employees AS e` и `Departments AS d` — это псевдонимы (алиасы) для таблиц, чтобы запрос был короче и читабельнее.
*   `ON e.department_id = d.id` — это **условие соединения**. Оно говорит SQL, как именно связаны таблицы.
*   **Результат:** Николай Федоров (без отдела) и отдел Финансов (без сотрудников) в результат не попадут.

#### `LEFT JOIN` (или `LEFT OUTER JOIN` - Левое внешнее соединение)

Возвращает **все строки из левой таблицы** (первой, указанной в `FROM`) и совпадающие строки из правой таблицы. Если совпадения в правой таблице нет, ее столбцы будут заполнены `NULL`.

**Задача:** Вывести ВСЕХ сотрудников и названия их отделов. Если отдел не указан, так и показать.

```sql
SELECT
    e.first_name,
    e.last_name,
    d.name AS department_name
FROM
    Employees AS e
LEFT JOIN
    Departments AS d ON e.department_id = d.id;
```
*   **Результат:** Включит всех сотрудников. У Николая Федорова в столбце `department_name` будет `NULL`.

#### `RIGHT JOIN` (или `RIGHT OUTER JOIN` - Правое внешнее соединение)

Зеркальное отражение `LEFT JOIN`. Возвращает **все строки из правой таблицы** и совпадающие из левой.

**Задача:** Вывести ВСЕ ОТДЕЛЫ и имена сотрудников в них. Если в отделе нет сотрудников, он все равно должен быть в списке.

```sql
SELECT
    e.first_name,
    e.last_name,
    d.name AS department_name
FROM
    Employees AS e
RIGHT JOIN
    Departments AS d ON e.department_id = d.id;
```
*   **Результат:** Включит все отделы. Для отдела "Финансы" столбцы `first_name` и `last_name` будут `NULL`.

#### `FULL OUTER JOIN` (Полное внешнее соединение)

Возвращает все строки, когда есть совпадение хотя бы в одной из таблиц. Это комбинация `LEFT` и `RIGHT JOIN`.

**Задача:** Показать всех сотрудников и все отделы, сопоставляя их, где возможно.

```sql
SELECT
    e.first_name,
    d.name AS department_name
FROM
    Employees AS e
FULL OUTER JOIN
    Departments AS d ON e.department_id = d.id;
```
*   **Результат:** Включит и Николая Федорова (с `NULL` в отделе), и отдел "Финансы" (с `NULL` в имени сотрудника).

### 2. Соединение таблицы со своей копией (`SELF JOIN`)

Это не отдельный тип `JOIN`, а техника, когда таблица соединяется сама с собой. Это полезно для иерархических данных (например, "сотрудник-менеджер").

**Задача:** Для каждого сотрудника вывести имя его руководителя.

```sql
SELECT
    -- Данные сотрудника
    e.first_name AS employee_name,
    e.last_name AS employee_lastname,
    -- Данные его менеджера
    m.first_name AS manager_name,
    m.last_name AS manager_lastname
FROM
    Employees AS e -- Таблица для сотрудников
LEFT JOIN
    Employees AS m ON e.manager_id = m.id; -- Та же таблица, но для менеджеров
```
*   Ключевой момент — использование **разных псевдонимов** (`e` и `m`) для одной и той же таблицы.
*   Мы используем `LEFT JOIN`, чтобы в результат попал и директор (Александр Кузнецов), у которого `manager_id` равен `NULL`. Для него поля менеджера будут `NULL`.

### 3. Простые запросы с подзапросами (вложенными запросами)

**Подзапрос** — это `SELECT`-запрос, вложенный внутрь другого SQL-запроса.

#### Подзапрос в условии `WHERE`

Чаще всего используется для фильтрации данных на основе результата, полученного из другого запроса.

1.  **Скалярный подзапрос (возвращает одно значение):**

    **Задача:** Найти сотрудников, получающих зарплату выше средней по компании.

    ```sql
    -- Шаг 1: Найти среднюю зарплату (этот запрос станет подзапросом)
    -- SELECT AVG(salary) FROM Employees;
    
    -- Шаг 2: Использовать результат в основном запросе
    SELECT first_name, salary
    FROM Employees
    WHERE salary > (SELECT AVG(salary) FROM Employees);
    ```
    Внутренний запрос `(SELECT AVG(salary) FROM Employees)` выполняется первым, возвращает одно число, которое затем используется для фильтрации в `WHERE`.

2.  **Подзапрос, возвращающий список значений (с оператором `IN`):**

    **Задача:** Найти всех сотрудников, работающих в отделах, расположенных в Москве.

    ```sql
    -- Шаг 1: Найти ID всех отделов в Москве
    -- SELECT id FROM Departments WHERE city = 'Москва';
    
    -- Шаг 2: Использовать список ID для фильтрации сотрудников
    SELECT first_name, last_name
    FROM Employees
    WHERE department_id IN (SELECT id FROM Departments WHERE city = 'Москва');
    ```
    Оператор `IN` проверяет, содержится ли `department_id` в списке, который вернул подзапрос.

#### Подзапрос в `FROM` (производная таблица)

Подзапрос в клаузе `FROM` создает временную, "виртуальную" таблицу, с которой затем работает внешний запрос. Такой подзапрос **обязательно** должен иметь псевдоним.

**Задача:** Вывести для каждого отдела его название и количество сотрудников в нем.

```sql
SELECT
    d.name,
    emp_counts.number_of_employees
FROM
    Departments AS d
JOIN
    -- Начало производной таблицы
    (
        SELECT
            department_id,
            COUNT(*) AS number_of_employees
        FROM Employees
        WHERE department_id IS NOT NULL
        GROUP BY department_id
    ) AS emp_counts -- Псевдоним для производной таблицы
    ON d.id = emp_counts.department_id;
```
*   Сначала выполняется подзапрос, который группирует сотрудников по `department_id` и считает их количество.
*   Результат этого подзапроса (`department_id`, `number_of_employees`) становится временной таблицей `emp_counts`.
*   Затем внешний запрос соединяет (`JOIN`) реальную таблицу `Departments` с этой временной таблицей `emp_counts`.
