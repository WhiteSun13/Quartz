### **1. Метод Динамического Программирования (ДП)**

**Основная идея:** "Разделяй, властвуй и запоминай". Сложная задача разбивается на множество более простых, **перекрывающихся** подзадач. Решение каждой подзадачи вычисляется только один раз и сохраняется (мемоизируется) в таблице. Решение исходной задачи конструируется из уже готовых решений подзадач.

**Ключевые условия применимости:**
1.  **Оптимальная подструктура:** Оптимальное решение всей задачи должно содержать в себе оптимальные решения ее подзадач.
2.  **Перекрывающиеся подзадачи:** В процессе решения многократно возникают одни и те же подзадачи. Без запоминания мы бы решали их снова и снова.

**Как это работает? (На примере задачи о рюкзаке 0/1)**
*   **Задача:** Есть $N$ предметов с весами $w_i$ и ценностями $v_i$, и рюкзак вместимостью $W$. Найти набор предметов с максимальной суммарной ценностью, не превышающей $W$.
*   **Создание таблицы:** Строится 2D-таблица `dp[i][j]`, где `dp[i][j]` — максимальная ценность, которую можно получить, используя только первые `i` предметов при ограничении веса рюкзака `j`.
*   **Заполнение (рекуррентное соотношение):** Для каждого предмета `i` и каждого веса `j` у нас есть два варианта:
    1.  **Не брать предмет `i`:** Тогда максимальная ценность будет такой же, как и для `i-1` предметов при том же весе: `dp[i-1][j]`.
    2.  **Взять предмет `i` (если он помещается, т.е. $w_i \le j$):** Тогда ценность будет $v_i$ плюс максимальная ценность, которую можно было собрать из `i-1` предметов при оставшемся весе $j - w_i$: `v_i + dp[i-1][j - w_i]`.
*   **Формула:** `dp[i][j] = max(dp[i-1][j], v_i + dp[i-1][j - w_i])`.
*   **Результат:** Ответ на исходную задачу будет храниться в последней ячейке таблицы `dp[N][W]`.

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Гарантирует нахождение глобального оптимума.** | ❌ **Высокие требования к памяти.** Размер таблицы может быть очень большим (проклятие размерности). |
| ✅ **Очень эффективен** для задач, обладающих нужной структурой. | ❌ **Не универсален.** Подходит только для узкого класса задач с оптимальной подструктурой. |
| ✅ **Полиномиальная сложность** для многих задач (например, $O(NW)$ для рюкзака), что намного лучше экспоненциальной. | ❌ **Сложность в формулировании** рекуррентного соотношения для некоторых задач. |

**Сфера применения:** Задача о рюкзаке, нахождение кратчайшего пути в графе (алгоритм Флойда-Уоршелла), вычисление редакционного расстояния, выравнивание биологических последовательностей.

### **2. Метод Ветвей и Границ (Branch and Bound)**

**Основная идея:** "Умный перебор". Метод строит дерево поиска, где каждый узел представляет собой подмножество всех возможных решений. Ключевая идея — отсекать целые ветви этого дерева, если мы можем доказать, что в них нет решений лучше, чем уже найденное.

**Ключевые компоненты:**
1.  **Ветвление (Branching):** Разделение задачи на две или более подзадачи (например, для задачи коммивояжера: "включаем ребро (A,B) в маршрут" и "не включаем ребро (A,B)").
2.  **Оценка (Bounding):** Для каждой подзадачи вычисляется **оценочная функция** — нижняя граница (для минимизации) или верхняя граница (для максимизации) качества лучшего решения в этой подзадаче. Оценка должна быть "оптимистичной" (например, для TSP — длина минимального остовного дерева, которая всегда меньше или равна длине любого гамильтонова цикла).
3.  **Отсечение (Pruning):** Ветвь отбрасывается, если её оценка хуже, чем уже найденный **рекорд** (лучшее на данный момент полное решение).

**Как это работает?**
1.  Находим некое начальное решение (например, жадным методом) и объявляем его рекордом.
2.  Создаем очередь узлов, изначально содержащую корень дерева (вся задача).
3.  Пока очередь не пуста:
    a. Извлекаем узел (подзадачу).
    b. Вычисляем для него оценку.
    c. Если оценка хуже рекорда, отбрасываем узел.
    d. Иначе, ветвим узел на несколько новых подузлов.
    e. Если подузел является полным решением и лучше рекорда, обновляем рекорд.
    f. Иначе, добавляем подузлы в очередь.

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Гарантирует нахождение глобального оптимума.** | ❌ **В худшем случае сложность экспоненциальная** (сводится к полному перебору). |
| ✅ **Значительно эффективнее** полного перебора на практике. | ❌ **Производительность сильно зависит** от качества оценочной функции. Плохая оценка почти ничего не отсекает. |
| ✅ **Гибкий:** можно использовать разные стратегии ветвления и оценки. | ❌ **Требует больше памяти,** чем простые эвристики, для хранения дерева. |

**Сфера применения:** Целочисленное линейное программирование, задача коммивояжера (TSP), задача о назначениях, планирование расписаний. Это основа большинства современных коммерческих решателей.

### **3. Жадный Метод (Greedy Method)**

**Основная идея:** "Бери лучшее и не оглядывайся". На каждом шаге алгоритм делает выбор, который кажется наилучшим в данный момент, без учета последствий этого выбора для будущего. Это близорукая стратегия.

**Как это работает? (На примере TSP)**
1.  Начать из произвольного города.
2.  На каждом шаге переходить в ближайший из еще не посещенных городов.
3.  Когда все города посещены, вернуться в начальный.

**Ключевое свойство:** Для того чтобы жадный алгоритм давал глобальный оптимум, задача должна обладать **свойством жадного выбора** (локально оптимальный выбор всегда ведет к глобально оптимальному решению). Большинство NP-трудных задач этим свойством не обладают.

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Очень быстрый.** Обычно имеет низкую полиномиальную сложность ($O(N^2)$ для TSP). | ❌ **Почти никогда не находит глобальный оптимум** для сложных задач. |
| ✅ **Прост в реализации** и понимании. | ❌ **Может давать очень плохие результаты.** Качество решения непредсказуемо. |
| ✅ **Идеален для задач, где он работает оптимально** (см. применение). | ❌ **Не имеет механизма для исправления** ранее сделанных неудачных выборов. |

**Сфера применения:**
*   **Где он оптимален:** Построение минимального остовного дерева (алгоритмы Прима и Крускала), поиск кратчайшего пути от одной вершины (алгоритм Дейкстры), задача о непрерывном рюкзаке.
*   **Где он используется как эвристика:** Для быстрого получения начального решения в более сложных алгоритмах (например, в методе ветвей и границ или генетических алгоритмах).

### **4. Муравьиный Алгоритм (Ant Colony Optimization, ACO)**

**Основная идея:** Популяционная метаэвристика, вдохновленная поведением муравьев, ищущих пищу. Муравьи оставляют на пути химический след (феромон). Короткие пути проходятся быстрее, на них феромон накапливается интенсивнее, что привлекает других муравьев. Возникает положительная обратная связь, которая и приводит к нахождению оптимального пути.

**Как это работает?**
1.  **Инициализация:** На всех ребрах графа (путях) размещается небольшое начальное количество феромона.
2.  **Цикл поколений:**
    a. **Построение решений:** "Колония" из $k$ искусственных муравьев одновременно строит решения (маршруты). Находясь в узле $i$, муравей выбирает следующий узел $j$ с вероятностью, которая зависит от двух факторов:
        *   **Привлекательность пути (эвристика):** Насколько "хорош" этот путь сам по себе (например, $1/d_{ij}$, где $d$ — расстояние).
        *   **Количество феромона:** Сколько феромона на пути $(i,j)$.
    b. **Обновление феромонов:**
        *   **Испарение:** Количество феромона на всех путях немного уменьшается. Это позволяет "забывать" плохие пути.
        *   **Нанесение:** Муравьи (обычно те, кто построил лучшие маршруты) наносят новый феромон на ребра, которые они использовали. Количество наносимого феромона обратно пропорционально длине найденного маршрута.

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Хорошо справляется с динамическими задачами,** где граф может меняться со временем. | ❌ **Медленная сходимость** по сравнению с другими метаэвристиками. |
| ✅ **Положительная обратная связь** позволяет быстро находить хорошие решения. | ❌ **Много параметров для настройки** (влияние феромона, эвристики, скорость испарения). |
| ✅ **Присущий параллелизм** (каждый муравей работает независимо). | ❌ **Склонен к стагнации,** когда все муравьи начинают ходить по одному и тому же, не обязательно оптимальному, пути. |

**Сфера применения:** Задача коммивояжера, маршрутизация транспорта (VRP), маршрутизация пакетов в сетях, задачи составления расписаний.

### **5. Генетические Методы (Genetic Algorithms, GA)**

**Основная идея:** Популяционная метаэвристика, основанная на принципах биологической эволюции: естественный отбор, скрещивание и мутация. Алгоритм оперирует "популяцией" решений ("хромосом"), которые эволюционируют в сторону улучшения.

**Как это работает?**
1.  **Инициализация:** Создается начальная популяция из $N$ случайных решений (хромосом). (Для TSP хромосома — это перестановка городов).
2.  **Цикл поколений:**
    a. **Оценка (Fitness):** Для каждого решения в популяции вычисляется его "приспособленность" (значение целевой функции, например, $1 / \text{длина маршрута}$).
    b. **Селекция:** Выбираются "родительские" особи для создания нового поколения. Особи с лучшей приспособленностью имеют больше шансов быть выбранными (например, "метод рулетки").
    c. **Скрещивание (Crossover):** Две родительские хромосомы обмениваются частями своего генетического материала, создавая одного или двух "потомков". (Для TSP это сложные операторы, сохраняющие корректность перестановки, например, PMX).
    d. **Мутация:** С небольшой вероятностью в хромосоме потомка происходит случайное изменение (например, меняются местами два города). Это вносит разнообразие и помогает избежать застревания в локальных оптимумах.
3.  Новое поколение заменяет старое, и цикл повторяется.

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Очень универсальны** и могут быть применены к широчайшему спектру задач. | ❌ **Требуют тщательной настройки** множества параметров (размер популяции, вероятности скрещивания и мутации и т.д.). |
| ✅ **Эффективны для исследования больших** и сложных пространств поиска со множеством локальных оптимумов. | ❌ **Вычислительно дороги.** Работа с целой популяцией требует много ресурсов. |
| ✅ **Хорошо распараллеливаются.** | ❌ **Нет гарантии нахождения оптимума.** Качество решения зависит от настроек и случайности. |

**Сфера применения:** Оптимизация параметров в машинном обучении, проектирование (аэродинамических профилей, схем), финансовое моделирование, игровая индустрия (обучение AI), составление расписаний.
