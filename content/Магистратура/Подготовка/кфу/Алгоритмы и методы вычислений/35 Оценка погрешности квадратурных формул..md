Конечно! Оценка погрешности квадратурных формул — это центральная тема в численном анализе, так как она позволяет нам не просто получить число, а понять, насколько этому числу можно доверять.

### **Оценка погрешности квадратурных формул**

#### **1. Введение: Постановка задачи**

**Квадратурная формула** — это приближенная формула для вычисления определенного интеграла:

`I(f) = ∫[a,b] f(x) dx ≈ Σ(i=0 to n) w_i * f(x_i) = I_n(f)`

где:
*   `x_i` — узлы квадратурной формулы.
*   `w_i` — веса квадратурной формулы.

**Погрешность (или остаточный член) квадратурной формулы** — это разница между точным значением интеграла и его приближенным значением:

`R_n(f) = I(f) - I_n(f) = ∫[a,b] f(x) dx - Σ(i=0 to n) w_i * f(x_i)`

Основная задача — оценить величину `|R_n(f)|`, не зная точного значения `I(f)`. Существует два принципиально разных подхода к этой задаче: теоретический (априорный) и практический (апостериорный).

---

### **2. Теоретическая (априорная) оценка погрешности**

Этот подход позволяет получить формулу для погрешности, которая зависит от свойств подынтегральной функции (ее производных) и параметров сетки (длины интервала). Эти оценки получаются до вычислений и используются для анализа методов.

Для **интерполяционных квадратурных формул** (которые строятся на основе интерполяционного многочлена Лагранжа), остаточный член имеет общую структуру:

`R_n(f) = C * f^(k)(ξ)`

где `C` — константа, зависящая от узлов и интервала, `k` — порядок первой производной, которую формула не интегрирует точно, а `ξ` — некоторая неизвестная точка из интервала `(a, b)`.

Рассмотрим остаточные члены для основных формул.

#### **2.1. Формула прямоугольников (элементарная)**
*   **Левых прямоугольников:** `I ≈ (b-a) * f(a)`
    *   **Погрешность:** `R(f) = (b-a)²/2 * f'(ξ)`, `ξ ∈ (a, b)`
    *   **Порядок точности:** Первый (`O(h²)`).
*   **Правых прямоугольников:** `I ≈ (b-a) * f(b)`
    *   **Погрешность:** `R(f) = -(b-a)²/2 * f'(ξ)`, `ξ ∈ (a, b)`
    *   **Порядок точности:** Первый (`O(h²)`).
*   **Средних прямоугольников (наиболее точная из них):** `I ≈ (b-a) * f((a+b)/2)`
    *   **Погрешность:** `R(f) = (b-a)³/24 * f''(ξ)`, `ξ ∈ (a, b)`
    *   **Порядок точности:** Второй (`O(h³)`). Удивительно, но она на порядок точнее двух предыдущих!

#### **2.2. Формула трапеций (элементарная)**
`I ≈ (b-a)/2 * [f(a) + f(b)]`
*   **Погрешность:** `R(f) = -(b-a)³/12 * f''(ξ)`, `ξ ∈ (a, b)`
*   **Порядок точности:** Второй (`O(h³)`).

#### **2.3. Формула Симпсона (парабол)**
`I ≈ (b-a)/6 * [f(a) + 4f((a+b)/2) + f(b)]`
*   **Погрешность:** `R(f) = -(b-a)⁵/2880 * f^(4)(ξ)`, `ξ ∈ (a, b)`
*   **Алгебраический порядок точности:** Третий (точна для многочленов до 3-й степени включительно).
*   **Порядок точности погрешности:** Четвертый (`O(h⁵)`). Формула Симпсона исключительно точна для своей простоты.

#### **2.4. Погрешность составных (составных) формул**

На практике элементарные формулы применяют к малым подынтервалам. Интервал `[a, b]` разбивается на `N` частей длиной `h = (b-a)/N`.

Общая логика: если погрешность на одном шаге `h` равна `C * h^(p+1)`, то суммарная погрешность на `N` шагах будет:
`R_total ≈ N * (C * h^(p+1)) = ((b-a)/h) * C * h^(p+1) = C * (b-a) * h^p`
Таким образом, порядок точности составной формулы на единицу меньше, чем порядок погрешности элементарной.

*   **Составная формула трапеций:**
    `R(f) = -(b-a)/12 * h² * f''(ξ)`
    **Порядок точности: `O(h²)`**

*   **Составная формула Симпсона:**
    `R(f) = -(b-a)/180 * h⁴ * f^(4)(ξ)`
    **Порядок точности: `O(h⁴)`**

**Практическая проблема теоретических оценок:**
Мы почти никогда не знаем точного значения `max|f^(k)(x)|` на интервале `[a,b]`. Поэтому эти формулы отлично подходят для теоретического сравнения методов, но редко используются для получения конкретного числового значения погрешности.

### **3. Практическая (апостериорная) оценка погрешности: Правило Рунге**

Это самый важный и широко используемый практический метод. Идея полностью аналогична его применению для решения ОДУ: **выполнить вычисления с двумя разными шагами и сравнить результаты.**

Пусть `I` — точное значение интеграла.
Пусть некий численный метод имеет порядок точности `p` (например, для составной формулы трапеций `p=2`, для Симпсона `p=4`). Тогда погрешность можно записать как `I - I_h ≈ C * h^p`, где `I_h` — результат, полученный с шагом `h`.

**Алгоритм:**
1.  Вычисляем интеграл `I_h`, разбив `[a, b]` на `N` отрезков с шагом `h = (b-a)/N`.
2.  Вычисляем интеграл `I_{h/2}`, разбив `[a, b]` на `2N` отрезков с шагом `h/2`. Это более точное значение.
3.  Записываем выражения для погрешности:
    *   `I - I_h ≈ C * h^p`
    *   `I - I_{h/2} ≈ C * (h/2)^p = (C * h^p) / 2^p`

4.  Вычитаем второе уравнение из первого:
    `I_{h/2} - I_h ≈ C*h^p * (1 - 1/2^p)`
5.  Из первого уравнения `C*h^p ≈ I - I_h`. Подставляем его в предыдущее выражение. На самом деле, удобнее выразить `I - I_{h/2}` (погрешность более точного метода) через разность `I_{h/2} - I_h`.
    Решая эту систему, получаем знаменитую формулу:

> **Погрешность `I_{h/2}` ≈ |`I_{h/2}` - `I_h`| / (`2^p` - 1)**

**Практическое применение для основных методов:**

*   **Для составной формулы трапеций (p=2):**
    `|R| ≈ |I_{h/2} - I_h| / (2² - 1) = |I_{h/2} - I_h| / 3`

*   **Для составной формулы Симпсона (p=4):**
    `|R| ≈ |I_{h/2} - I_h| / (2⁴ - 1) = |I_{h/2} - I_h| / 15`
    Это одна из самых используемых формул в численных расчетах.

**Уточненное значение по Ричардсону:**
Можно не только оценить погрешность, но и получить еще более точное значение интеграла:
`I_уточн = I_{h/2} + (I_{h/2} - I_h) / (2^p - 1)`
Для метода Симпсона это `I_уточн = I_{h/2} + (I_{h/2} - I_h) / 15`.

### **4. Алгоритм адаптивной квадратуры**

Правило Рунге является ядром **адаптивных алгоритмов**, которые автоматически сгущают сетку там, где функция ведет себя "плохо" (имеет большие производные), и используют редкую сетку, где она гладкая.

**Рекурсивный алгоритм адаптивного интегрирования (на примере Симпсона):**

`function AdaptiveIntegrate(f, a, b, tolerance):`
1.  `c = (a+b)/2`
2.  `I_h = Simpson(f, a, b)`  // Вычисление с шагом `H = b-a`
3.  `I_{h/2} = Simpson(f, a, c) + Simpson(f, c, b)` // Вычисление с шагом `h = (b-a)/2`
4.  `error = |I_{h/2} - I_h| / 15`
5.  **if** `error < tolerance`:
        `return I_{h/2}` // Точность достигнута, возвращаем результат
    **else**:
        // Точность не достигнута, делим интервал и рекурсивно вызываем для каждой половины
        // с половиной допустимой погрешности.
        `left_integral = AdaptiveIntegrate(f, a, c, tolerance/2)`
        `right_integral = AdaptiveIntegrate(f, c, b, tolerance/2)`
        `return left_integral + right_integral`

Этот элегантный алгоритм концентрирует вычислительные усилия именно там, где это необходимо, обеспечивая заданную точность с минимальными затратами.
