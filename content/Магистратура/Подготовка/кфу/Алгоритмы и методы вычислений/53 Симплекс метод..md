Конечно! **Симплекс-метод** — это краеугольный камень линейного программирования и один из самых важных алгоритмов в области оптимизации. Разберем его максимально подробно.

---

### **Симплекс-Метод: Полное Руководство**

**Автор:** Джордж Данциг (George Dantzig), 1947 год.

**Что он делает?**
Симплекс-метод — это итерационный алгоритм для решения задач **линейного программирования (ЛП)**. Задача ЛП состоит в том, чтобы найти максимальное или минимальное значение некоторой линейной функции (называемой **целевой функцией**) при наличии набора линейных равенств или неравенств (называемых **ограничениями**).

**Пример задачи ЛП:**
Максимизировать прибыль $Z = 3x_1 + 5x_2$ (где $x_1$ и $x_2$ — количество производимых продуктов A и B)
При ограничениях на ресурсы:
1.  $x_1 \le 4$ (ресурс 1)
2.  $2x_2 \le 12$ (ресурс 2)
3.  $3x_1 + 2x_2 \le 18$ (ресурс 3)
4.  $x_1 \ge 0, x_2 \ge 0$ (условие неотрицательности)

#### **1. Геометрическая Интерпретация (Ключ к Пониманию)**

Чтобы понять, *как* работает симплекс-метод, нужно представить задачу геометрически.

1.  **Область допустимых решений (ОДР):** Каждое линейное ограничение-неравенство отсекает в пространстве переменных (в нашем примере — на плоскости $x_1, x_2$) полуплоскость. Пересечение всех этих полуплоскостей образует **выпуклый многогранник** (или многоугольник в 2D). Это и есть область допустимых решений. Любая точка внутри или на границе этого многогранника является допустимым решением.

2.  **Фундаментальная теорема ЛП:** Оптимальное решение задачи (если оно существует) всегда находится в одной из **вершин** (углов) этого многогранника.

3.  **Идея симплекс-метода:** Вместо того чтобы проверять все бесконечное множество точек внутри многогранника, достаточно проверить только его вершины. Симплекс-метод — это "умный" способ "путешествовать" по ребрам этого многогранника от одной вершины к другой, причем на каждом шаге мы переходим в соседнюю вершину, где значение целевой функции **не хуже** (а обычно — лучше), чем в предыдущей. Путешествие заканчивается, когда мы достигаем вершины, из которой нельзя перейти в соседнюю вершину с улучшением целевой функции. Эта вершина и будет оптимальной.

#### **2. Алгебраическая Формулировка (Как это работает на практике)**

Геометрия — это хорошо, но компьютеры работают с числами и матрицами. Поэтому перейдем к алгебре.

**Шаг 1: Приведение к канонической форме**

Чтобы алгоритм работал, задачу нужно привести к стандартному виду:
1.  Это задача **максимизации**. (Если у вас минимизация $Z$, просто максимизируйте $-Z$).
2.  Все ограничения — это равенства.
3.  Все переменные неотрицательны.

Как превратить неравенства в равенства? С помощью **балансовых (или остаточных) переменных (slack variables)**.
Для каждого ограничения типа "$\le$" мы вводим новую неотрицательную переменную, которая показывает, "сколько ресурса осталось неиспользованным".

В нашем примере:
1.  $x_1 + s_1 = 4$
2.  $2x_2 + s_2 = 12$
3.  $3x_1 + 2x_2 + s_3 = 18$

Теперь у нас система линейных уравнений с 5 переменными ($x_1, x_2, s_1, s_2, s_3$). Целевая функция остается прежней: $Z = 3x_1 + 5x_2$.

**Шаг 2: Симплекс-таблица**

Вся информация о задаче удобно записывается в виде таблицы, называемой **симплекс-таблицей**.

*   **Базисные переменные:** На каждой итерации у нас есть $m$ **базисных** переменных (где $m$ — число ограничений). В начальный момент это наши балансовые переменные ($s_1, s_2, s_3$). Геометрически они соответствуют начальной вершине — началу координат ($x_1=0, x_2=0$). Все остальные переменные называются **свободными (небазисными)** и равны нулю.
*   **Индексная строка (Z-строка):** Последняя строка таблицы содержит коэффициенты целевой функции, взятые с противоположным знаком.

**Начальная симплекс-таблица для нашего примера:**

| Базис | $x_1$ | $x_2$ | $s_1$ | $s_2$ | $s_3$ | Своб. члены (RHS) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| $s_1$ | 1 | 0 | 1 | 0 | 0 | 4 |
| $s_2$ | 0 | 2 | 0 | 1 | 0 | 12 |
| $s_3$ | 3 | 2 | 0 | 0 | 1 | 18 |
| **Z** | **-3** | **-5** | **0** | **0** | **0** | **0** |

*   **Как читать таблицу:** Сейчас $s_1=4, s_2=12, s_3=18$ (значения в столбце свободных членов), а $x_1=0, x_2=0$. Прибыль $Z=0$.

#### **3. Алгоритм Симплекс-Метода (Итерации)**

**Шаг 3: Проверка на оптимальность**
Смотрим на индексную Z-строку. **Если все коэффициенты в ней неотрицательны ($\ge 0$), то текущее решение оптимально.** Алгоритм завершен.
В нашем случае есть отрицательные коэффициенты (-3 и -5), значит, решение можно улучшить.

**Шаг 4: Выбор ведущего столбца (какую переменную ввести в базис)**
Выбираем столбец с **наиболее отрицательным** коэффициентом в Z-строке. Это столбец $x_2$ (-5). Это означает, что увеличение $x_2$ дает наибольший прирост прибыли. Этот столбец называется **ведущим (разрешающим)**.

**Шаг 5: Выбор ведущей строки (какую переменную вывести из базиса)**
Чтобы определить, какая из базисных переменных ($s_1, s_2, s_3$) станет равной нулю и покинет базис, нужно провести **симплексное отношение (ratio test)**:
Делим каждый элемент столбца свободных членов на соответствующий положительный элемент ведущего столбца.
1.  Для строки $s_1$: нет элемента в столбце $x_2$ (или он равен 0), пропускаем.
2.  Для строки $s_2$: $12 / 2 = 6$
3.  Для строки $s_3$: $18 / 2 = 9$

Выбираем строку с **наименьшим неотрицательным** отношением. Это строка $s_2$ (отношение 6). Эта строка называется **ведущей (разрешающей)**. Переменная $s_2$ покинет базис.

**Элемент на пересечении ведущего столбца и ведущей строки (в нашем случае, 2) называется разрешающим элементом.**

**Шаг 6: Пересчет таблицы (переход в новую вершину)**
Теперь мы должны выполнить преобразования (похожие на метод Гаусса), чтобы на месте разрешающего элемента получилась 1, а все остальные элементы в ведущем столбце стали нулями.
1.  **Новая ведущая строка:** Делим всю старую ведущую строку ($s_2$) на разрешающий элемент (2).
    `[0, 2, 0, 1, 0, 12] / 2 = [0, 1, 0, 1/2, 0, 6]`
2.  **Обновление остальных строк:** Для каждой другой строки (включая Z-строку) вычитаем из нее новую ведущую строку, умноженную на элемент этой строки в ведущем столбце.
    *   **Новая строка $s_1$:** Она не меняется, т.к. в столбце $x_2$ у нее 0. `[1, 0, 1, 0, 0, 4]`
    *   **Новая строка $s_3$:** `старая_s3 - 2 * новая_ведущая = [3, 2, 0, 0, 1, 18] - 2 * [0, 1, 0, 1/2, 0, 6] = [3, 0, 0, -1, 1, 6]`
    *   **Новая Z-строка:** `старая_Z - (-5) * новая_ведущая = [-3, -5, 0, 0, 0, 0] + 5 * [0, 1, 0, 1/2, 0, 6] = [-3, 0, 0, 5/2, 0, 30]`

**Итерация 1 завершена. Новая таблица:**

| Базис | $x_1$ | $x_2$ | $s_1$ | $s_2$ | $s_3$ | Своб. члены |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| $s_1$ | 1 | 0 | 1 | 0 | 0 | 4 |
| $x_2$ | 0 | 1 | 0 | 1/2 | 0 | 6 |
| $s_3$ | 3 | 0 | 0 | -1 | 1 | 6 |
| **Z** | **-3** | **0** | **0** | **5/2** | **0** | **30** |

*   **Как читать:** Теперь $x_2=6$, $s_1=4$, $s_3=6$. $x_1=0, s_2=0$. Прибыль $Z=30$. Мы перешли в новую вершину.

**Повторяем процесс:**
1.  **Проверка на оптимальность:** В Z-строке есть отрицательный элемент (-3). Продолжаем.
2.  **Ведущий столбец:** $x_1$.
3.  **Ведущая строка (ratio test):**
    *   Строка $s_1$: $4 / 1 = 4$
    *   Строка $x_2$: пропускаем (элемент 0).
    *   Строка $s_3$: $6 / 3 = 2$
    Наименьшее отношение — 2. **Ведущая строка — $s_3$**. Разрешающий элемент — 3.
4.  **Пересчет таблицы:** ... (аналогичные преобразования)

**Итерация 2 завершена. Финальная таблица:**

| Базис | $x_1$ | $x_2$ | $s_1$ | $s_2$ | $s_3$ | Своб. члены |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| $s_1$ | 0 | 0 | 1 | 1/3 | -1/3 | 2 |
| $x_2$ | 0 | 1 | 0 | 1/2 | 0 | 6 |
| $x_1$ | 1 | 0 | 0 | -1/3 | 1/3 | 2 |
| **Z** | **0** | **0** | **0** | **3/2** | **1** | **36** |

1.  **Проверка на оптимальность:** Все коэффициенты в Z-строке неотрицательны. **Решение оптимально!**

#### **4. Чтение Результата**

*   Смотрим на столбец "Базис" и "Своб. члены".
    *   $x_1 = 2$
    *   $x_2 = 6$
    *   $s_1 = 2$ (осталось 2 единицы ресурса 1)
*   Свободные (небазисные) переменные равны нулю: $s_2 = 0, s_3 = 0$ (ресурсы 2 и 3 использованы полностью).
*   Оптимальное значение целевой функции находится в правом нижнем углу: **Z = 36**.

**Ответ:** Для получения максимальной прибыли 36 нужно производить 2 единицы продукта А и 6 единиц продукта Б.

#### **5. Особые Случаи**

*   **Неограниченность:** Если на шаге 5 (выбор ведущей строки) все элементы в ведущем столбце оказываются отрицательными или нулевыми, это значит, что целевая функция не ограничена сверху и может расти до бесконечности.
*   **Вырожденность:** Если на шаге 5 в ratio test получается ничья (несколько одинаковых минимальных отношений), то на следующей итерации одна из базисных переменных будет равна 0. Это может (хотя и редко) привести к зацикливанию алгоритма. Для борьбы с этим есть специальные правила (например, правило Блэнда).
*   **Отсутствие допустимых решений:** Если после введения искусственных переменных (в более сложных версиях метода) они остаются в базисе с ненулевым значением, это значит, что ОДР пуста.

#### **6. Преимущества и Недостатки**

| Преимущества | Недостатки |
| :--- | :--- |
| ✅ **Гарантирует нахождение глобального оптимума,** если он существует. | ❌ **В худшем случае имеет экспоненциальную сложность.** Существуют специальные примеры (куб Клее-Минти), где симплекс-метод перебирает все вершины. |
| ✅ **Очень эффективен на практике.** В среднем работает за полиномиальное время. | ❌ **Может быть чувствителен к ошибкам округления** при компьютерных вычислениях (вычислительная неустойчивость). |
| ✅ **Дает много дополнительной информации** (двойственные переменные, анализ чувствительности). | ❌ Для очень больших задач (миллионы переменных) может уступать **методам внутренней точки**. |

**Вывод:** Симплекс-метод — это элегантный, мощный и исторически важный алгоритм. Несмотря на теоретические недостатки по сложности в худшем случае, он остается одним из самых надежных и широко используемых инструментов для решения практических задач оптимизации.
