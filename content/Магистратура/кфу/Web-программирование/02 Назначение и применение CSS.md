### **1. Введение: Что такое CSS и зачем он нужен?**

Если представить веб-страницу как дом, то:
*   **HTML (HyperText Markup Language)** – это его структура: фундамент, стены, крыша, комнаты. Он определяет, *что* находится на странице (заголовки, параграфы, картинки, списки).
*   **CSS (Cascading Style Sheets)** – это его дизайн и отделка: цвет стен, тип обоев, расстановка мебели, освещение. Он определяет, *как* выглядят элементы на странице.
*   **JavaScript** – это инженерные системы и бытовая техника: электричество, водопровод, лифт, умный дом. Он добавляет интерактивность и динамику.

**CSS** — это формальный язык, используемый для описания внешнего вида документа, написанного с использованием языка разметки (чаще всего HTML).

#### Принцип разделения контента и представления

Главная идея CSS — отделить **содержимое** документа (HTML) от его **визуального представления** (CSS).

#### Преимущества использования CSS:

1.  **Ускорение разработки:** Вы можете написать CSS один раз и затем применять его к множеству HTML-страниц.
2.  **Простота поддержки:** Для изменения внешнего вида всего сайта достаточно отредактировать один CSS-файл.
3.  **Улучшение доступности (Accessibility):** Пользователи могут настраивать стили под себя, например, увеличивать шрифт.
4.  **Оптимизация для поисковых систем (SEO):** Четкое разделение структуры и стилей делает код чище и понятнее для поисковых роботов.
5.  **Адаптивность:** С помощью CSS сайт можно заставить хорошо выглядеть на любых устройствах: от огромных мониторов до экранов смартфонов.

### **2. Глава 1: Основы CSS. Синтаксис и способы подключения**

#### 2.1. Базовый синтаксис CSS-правила

Любое CSS-правило состоит из двух основных частей: **селектора** и **блока объявлений**.

```css
/*  Селектор   Блок объявлений       */
/*   |         |                      | */
    p {
/*      |Свойство |  Значение |        */
        color: red;
        font-size: 16px; /* Каждое объявление заканчивается точкой с запятой */
    }
```

*   **Селектор (`p`):** Указывает, к какому HTML-элементу (или элементам) будет применено правило. В данном случае — ко всем тегам `<p>`.
*   **Блок объявлений (`{...}`):** Содержит одно или несколько объявлений, заключенных в фигурные скобки.
*   **Объявление (`color: red;`):** Состоит из **свойства** и его **значения**, разделенных двоеточием.

#### 2.2. Способы подключения стилей к HTML-документу

##### Внешние стили (External CSS) – **Рекомендуемый способ**

Создается отдельный файл с расширением `.css` (например, `style.css`), который подключается к HTML-документу с помощью тега `<link>` внутри `<head>`.

**HTML (`index.html`):**
```html
<!DOCTYPE html>
<html>
<head>
    <title>Моя страница</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    ...
</body>
</html>
```
**CSS (`style.css`):**
```css
body {
    background-color: #f0f0f0;
}
h1 {
    color: navy;
}
```
**Преимущества:**
*   Один файл стилей для всего сайта.
*   HTML-код остается чистым.
*   Браузер может кешировать CSS-файл, ускоряя загрузку последующих страниц.

##### Внутренние стили (Internal CSS)

CSS-правила размещаются внутри тега `<style>` непосредственно в `<head>` HTML-документа.

```html
<!DOCTYPE html>
<html>
<head>
    <title>Моя страница</title>
    <style>
        body {
            background-color: #f0f0f0;
        }
        h1 {
            color: navy;
        }
    </style>
</head>
<body>
    ...
</body>
</html>
```
**Применение:** Полезно для стилей, которые уникальны для одной конкретной страницы, или для быстрой отладки.

##### Встроенные стили (Inline CSS)

Стили добавляются напрямую к конкретному HTML-тегу через атрибут `style`.

```html
<h1 style="color: navy; margin-left: 20px;">Это заголовок</h1>
```
**Применение:** Следует избегать этого способа. Он нарушает принцип разделения контента и представления. Используется в исключительных случаях, например, когда стили генерируются динамически с помощью JavaScript.

### **3. Глава 2: Селекторы – Как выбрать нужный элемент?**

#### 3.1. Что такое селектор?

Селектор — это паттерн, который определяет, к каким элементам на странице применить CSS-правила. Умение правильно составлять селекторы — ключевой навык в CSS.

#### 3.2. Основные типы селекторов

| Тип селектора         | Пример             | Описание                                           |
| --------------------- | ------------------ | -------------------------------------------------- |
| **По тегу**           | `p`                | Выбирает все элементы `<p>`                        |
| **По классу**         | `.product-card`    | Выбирает все элементы с `class="product-card"`     |
| **По ID**             | `#main-header`     | Выбирает единственный элемент с `id="main-header"` |
| **По атрибуту**       | `a[target="_blank"]` | Выбирает ссылки, открывающиеся в новой вкладке    |
| **Универсальный**     | `*`                | Выбирает абсолютно все элементы на странице        |

#### 3.3. Комбинаторы селекторов

Комбинаторы позволяют создавать более точные правила, описывая связь между элементами.

*   **Дочерний селектор (`>`):** `ul > li` — выберет только те `<li>`, которые являются прямыми потомками `<ul>`.
*   **Селектор потомков (пробел):** `div p` — выберет все `<p>`, которые находятся внутри `<div>`, на любом уровне вложенности.
*   **Соседний селектор (`+`):** `h2 + p` — выберет первый `<p>`, который идет сразу после `<h2>` на том же уровне.
*   **Общий сестринский селектор (`~`):** `h2 ~ p` — выберет все `<p>`, которые идут после `<h2>` на том же уровне.

#### 3.4. Псевдоклассы и псевдоэлементы

*   **Псевдоклассы (`:`):** Определяют особое состояние элемента.
    *   `:hover` — стиль применяется при наведении мыши.
    *   `:focus` — стиль для элемента в фокусе (например, поля ввода).
    *   `:nth-child(n)` — выбирает дочерний элемент по его порядковому номеру (например, `:nth-child(2n)` выберет все четные элементы).

*   **Псевдоэлементы (`::`):** Позволяют стилизовать определенную часть элемента.
    *   `::before` и `::after` — добавляют контент до или после содержимого элемента.
    *   `::first-letter` — стилизует первую букву текста.
    *   `::first-line` — стилизует первую строку текста.

### **4. Глава 3: Каскад, специфичность и наследование**

Эти три механизма определяют, какое CSS-правило в итоге будет применено к элементу.

#### 4.1. Каскад (Cascade)

Браузер применяет стили в определенном порядке:
1.  Стили браузера по умолчанию.
2.  Стили пользователя (если заданы в браузере).
3.  **Стили автора** (то, что пишете вы):
    *   Сначала стили из внешних файлов.
    *   Затем внутренние стили (в теге `<style>`).
    *   И наконец, встроенные стили (в атрибуте `style`).

**Простое правило:** если два правила имеют одинаковую важность, побеждает то, которое было объявлено **позже** в коде.

#### 4.2. Специфичность (Specificity)

Когда несколько правил нацелены на один и тот же элемент, браузер вычисляет "вес" или **специфичность** каждого селектора, чтобы решить, какое из них применить.

**Упрощенная иерархия специфичности (от большей к меньшей):**

1.  **Встроенные стили (Inline styles):** `style="..."` — имеют наивысший приоритет.
2.  **Селекторы по ID (`#id`):** Очень высокий вес.
3.  **Селекторы по классу (`.class`), атрибуту (`[type="..."]`) и псевдоклассу (`:hover`):** Средний вес.
4.  **Селекторы по тегу (`div`) и псевдоэлементу (`::before`):** Низкий вес.

**Пример:**
```html
<p id="intro" class="text">Какой-то текст.</p>
```
```css
p { color: blue; }         /* Низкая специфичность */
.text { color: green; }     /* Средняя специфичность - ПРИМЕНИТСЯ ЭТОТ */
#intro { color: red; }      /* Высокая специфичность - ЕСЛИ БЫ ОН БЫЛ, ПОБЕДИЛ БЫ ОН */
```
Текст будет **зеленым**, так как селектор по классу специфичнее селектора по тегу.

#### 4.3. Наследование (Inheritance)

Некоторые CSS-свойства (например, `color`, `font-family`, `font-size`) передаются от родительского элемента к дочерним. Это называется наследованием. Свойства, связанные с размерами и отступами (`width`, `padding`, `margin`, `border`), обычно **не наследуются**.

#### 4.4. Свойство `!important`

Добавление `!important` к объявлению делает его самым приоритетным, перебивая любую специфичность и порядок.

```css
p {
    color: blue !important;
}
```
**Использование `!important` считается плохой практикой.** Оно усложняет отладку и нарушает естественный каскад. Прибегайте к нему только в крайних случаях.

### **5. Глава 4: Основные свойства для стилизации**

#### 5.1. Работа с текстом и шрифтами
*   `font-family`: задает семейство шрифтов (`Arial`, `Times New Roman`).
*   `font-size`: размер шрифта (`16px`, `1.2em`, `1rem`).
*   `font-weight`: насыщенность шрифта (`normal`, `bold`).
*   `color`: цвет текста.
*   `text-align`: горизонтальное выравнивание (`left`, `center`, `right`).
*   `line-height`: межстрочный интервал.
*   `text-decoration`: подчеркивание, перечеркивание (`none`, `underline`).

#### 5.2. Цвета и фоны
*   `color`: цвет текста.
*   `background-color`: цвет фона.
*   `background-image`: фоновое изображение (`url('path/to/image.jpg')`).
*   `background-repeat`: повторение фона (`no-repeat`, `repeat-x`).
*   `background-position`: позиция фона (`center`, `top right`).

#### 5.3. Блочная модель (Box Model)

Каждый элемент на странице представляет собой прямоугольный блок. Его модель состоит из:
*   **Content:** Содержимое (текст, картинка).
*   **Padding:** Внутренний отступ (пространство между содержимым и рамкой).
*   **Border:** Рамка вокруг padding.
*   **Margin:** Внешний отступ (пространство до соседних элементов).

**Важное свойство `box-sizing`:**
*   `box-sizing: content-box;` (по умолчанию): `width` и `height` задают только размер контента. `padding` и `border` добавляются сверху.
*   `box-sizing: border-box;` (**рекомендуется**): `width` и `height` задают общий размер блока, включая `padding` и `border`. Это делает верстку гораздо предсказуемей.

```css
/* "Сброс" для всех элементов */
* {
  box-sizing: border-box;
}
```

### **6. Глава 5: Управление макетом страницы (Layout)**

#### 6.1. Свойство `display`
*   `block`: Элемент занимает всю доступную ширину, начинается с новой строки (`<div>`, `<p>`, `<h1>`).
*   `inline`: Элемент занимает столько ширины, сколько нужно его содержимому, и не создает переноса строки (`<a>`, `<span>`, `<strong>`). Нельзя задать `width`, `height`, `margin-top/bottom`.
*   `inline-block`: Комбинация: ведет себя как `inline` (встраивается в строку), но ему можно задавать размеры и отступы, как `block`.
*   `none`: Полностью скрывает элемент.

#### 6.2. Позиционирование элементов: `position`
*   `static` (по умолчанию): Элемент находится в обычном потоке документа.
*   `relative`: Элемент остается в потоке, но его можно сдвигать относительно его *исходной* позиции с помощью свойств `top`, `right`, `bottom`, `left`.
*   `absolute`: Элемент **вырывается из потока** и позиционируется относительно ближайшего родителя с `position` не `static`. Если такого нет — относительно `<body>`.
*   `fixed`: Элемент **вырывается из потока** и позиционируется относительно окна браузера. Он не прокручивается вместе со страницей.
*   `sticky`: Гибрид `relative` и `fixed`. Элемент ведет себя как `relative`, пока не "прилипнет" к краю экрана при прокрутке.

#### 6.3. Современные методы раскладки

##### **Flexbox (`display: flex`)**
Идеален для создания одномерных макетов (элементы в ряд или в колонку).
*   **Контейнер (`display: flex`)**:
    *   `flex-direction`: направление (`row`, `column`).
    *   `justify-content`: выравнивание по основной оси (`flex-start`, `center`, `space-between`).
    *   `align-items`: выравнивание по поперечной оси (`flex-start`, `center`, `stretch`).
*   **Элементы (flex-items)**:
    *   `flex-grow`: способность элемента растягиваться.

##### **Grid Layout (`display: grid`)**
Мощнейший инструмент для создания сложных двумерных (сеточных) макетов.
*   **Контейнер (`display: grid`)**:
    *   `grid-template-columns`: определяет количество и ширину колонок (`1fr 1fr 1fr` — три равные колонки).
    *   `grid-template-rows`: определяет высоту строк.
    *   `grid-gap` (или `gap`): задает расстояние между ячейками сетки.

### **7. Глава 6: Продвинутые возможности CSS**

#### 7.1. Адаптивный дизайн и медиазапросы (`@media`)
Медиазапросы позволяют применять разные стили в зависимости от характеристик устройства (например, ширины экрана).
```css
/* Стили для всех экранов */
.container {
  width: 90%;
}

/* Стили, которые применятся, только если ширина экрана 768px или больше */
@media (min-width: 768px) {
  .container {
    width: 750px;
  }
}
```

#### 7.2. Переходы (`transition`) и анимации (`@keyframes`)
*   **`transition`:** Плавно изменяет свойство за определенное время.
    ```css
    .button {
        background-color: blue;
        transition: background-color 0.3s ease;
    }
    .button:hover {
        background-color: darkblue;
    }
    ```
*   **`animation`:** Создает сложные, пошаговые анимации с помощью ключевых кадров (`@keyframes`).

#### 7.3. CSS-переменные (Custom Properties)
Позволяют хранить значения в переменных для многократного использования.
```css
:root {
  --main-bg-color: #f0f0f0;
  --main-text-color: #333;
}

body {
  background-color: var(--main-bg-color);
  color: var(--main-text-color);
}
```

### **8. Глава 7: Практическая работа и лучшие практики**

#### 8.1. Создание стилизованной карточки товара

**HTML:**
```html
<div class="product-card">
    <img src="path/to/image.jpg" alt="Товар" class="product-image">
    <h3 class="product-title">Название товара</h3>
    <p class="product-price">1999 ₽</p>
    <button class="product-button">В корзину</button>
</div>
```
**CSS:**
```css
.product-card {
  width: 300px;
  border: 1px solid #ddd;
  border-radius: 8px; /* Скругленные углы */
  padding: 16px;
  text-align: center;
  box-shadow: 0 4px 8px rgba(0,0,0,0.1); /* Тень для объема */
  transition: transform 0.2s; /* Плавный переход для hover-эффекта */
}

.product-card:hover {
  transform: translateY(-5px); /* Приподнимаем карточку при наведении */
}

.product-image {
  max-width: 100%;
  height: auto;
```

#### 8.2. Лучшие практики написания CSS

1.  **Используйте внешние таблицы стилей.**
2.  **Придерживайтесь методологии** (например, **BEM** – Блок, Элемент, Модификатор) для именования классов, чтобы избежать конфликтов.
3.  **Используйте классы для стилизации, а ID — для JavaScript-хуков.**
4.  **Сбрасывайте стили браузера по умолчанию** (`reset.css` или `normalize.css`) для кроссбраузерной консистентности.
5.  **Используйте `box-sizing: border-box;`**.
6.  **Применяйте современные методы раскладки** (Flexbox и Grid) вместо устаревших (`float`).
7.  **Пишите код с подходом "Mobile First"**: сначала стили для мобильных, затем расширяйте их для больших экранов через `@media`.

### **9. Заключение**

CSS — это мощный и гибкий инструмент, который превращает безликие HTML-документы в красивые, удобные и адаптивные веб-сайты. Освоение его основ — от синтаксиса и селекторов до блочной модели и современных методов раскладки — является обязательным шагом для каждого, кто хочет создавать качественные веб-интерфейсы.

### **10. Глоссарий**

*   **BEM (БЭМ):** Методология именования CSS-классов для создания независимых, переиспользуемых компонентов.
*   **Кроссбраузерность:** Способность сайта одинаково отображаться и работать во всех популярных браузерах.
*   **Mobile First:** Подход к разработке, при котором проектирование и стилизация начинаются с мобильной версии сайта.
