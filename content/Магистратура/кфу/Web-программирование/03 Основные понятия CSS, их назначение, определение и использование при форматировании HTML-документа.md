### Введение: Что такое CSS и зачем он нужен?

#### Определение CSS

**CSS (Cascading Style Sheets — Каскадные таблицы стилей)** — это язык формального описания внешнего вида документа, написанного с использованием языка разметки (чаще всего HTML). CSS отвечает на вопрос: "Как должен выглядеть элемент?".

Представьте, что **HTML — это скелет** веб-страницы, ее структура и содержание (заголовки, параграфы, картинки, списки). А **CSS — это одежда, макияж и прическа** для этого скелета. CSS определяет цвета, шрифты, отступы, расположение элементов и многое другое.

#### Принцип разделения контента и представления

На заре интернета стили часто прописывали прямо внутри HTML-тегов с помощью специальных атрибутов (например, `<p color="red">`). Это создавало огромные проблемы:

*   **Сложность поддержки:** Чтобы изменить цвет всех заголовков на сайте из 100 страниц, пришлось бы вручную отредактировать каждую из 100 страниц.
*   **"Грязный" код:** HTML-файл становился огромным и нечитаемым из-за смешения структуры и стилей.
*   **Низкая производительность:** Больший размер HTML-файлов замедлял их загрузку.

CSS решает эти проблемы, вынося все правила оформления в отдельный файл или блок. Это позволяет:

*   **Централизованно управлять стилями:** Изменив одно правило в CSS-файле, можно мгновенно поменять внешний вид всех элементов на всем сайте.
*   **Сохранять HTML чистым и семантичным:** HTML отвечает только за структуру, а CSS — за внешний вид.
*   **Ускорить загрузку:** Браузер кеширует (сохраняет) CSS-файл после первой загрузки, и для последующих страниц ему не нужно скачивать его заново.

### Глава 1: Подключение CSS к HTML-документу

Существует три способа применить стили к HTML-элементу.

#### 1. Встроенные стили (Inline Styles)

Стили прописываются напрямую в атрибуте `style` нужного HTML-тега.

*   **Использование:** Крайне редко, для локальных и быстрых тестов или когда стиль должен быть применен к одному-единственному элементу и генерируется динамически (например, с помощью JavaScript).
*   **Недостаток:** Обладает наивысшим приоритетом (специфичностью), что затрудняет его переопределение. Загрязняет HTML.

**Пример:**
```html
<p style="color: blue; font-size: 16px;">Этот параграф будет синего цвета.</p>
```

#### 2. Внутренние таблицы стилей (Internal Styles)

Стили прописываются внутри тега `<style>`, который размещается в `<head>` HTML-документа.

*   **Использование:** Полезно для одностраничных сайтов или когда стили уникальны для конкретной страницы и не будут использоваться на других.
*   **Недостаток:** Стили не кешируются отдельно от страницы.

**Пример:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>Страница с внутренними стилями</title>
    <style>
        body {
            background-color: #f0f0f0;
        }
        h1 {
            color: navy;
            text-align: center;
        }
    </style>
</head>
<body>
    <h1>Это главный заголовок</h1>
</body>
</html>
```

#### 3. Внешние таблицы стилей (External Styles) — Рекомендуемый способ

Стили создаются в отдельном файле с расширением `.css` (например, `style.css`) и подключаются к HTML-документу с помощью тега `<link>` в секции `<head>`.

*   **Использование:** Это стандартный и самый правильный подход для большинства проектов.
*   **Преимущества:**
    *   Полное разделение контента и представления.
    *   CSS-файл кешируется браузером, ускоряя загрузку сайта.
    *   Один и тот же файл стилей можно подключить к неограниченному числу HTML-страниц.

**Пример:**

**Файл `index.html`:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>Страница с внешними стилями</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <h1>Это главный заголовок</h1>
    <p>Это параграф, стилизованный из внешнего файла.</p>
</body>
</html>
```

**Файл `styles.css`:**
```css
body {
    font-family: Arial, sans-serif;
    background-color: #f4f4f4;
}

h1 {
    color: darkgreen;
}

p {
    color: #333;
}
```

### Глава 2: Синтаксис CSS

Правило CSS состоит из двух основных частей: **селектора** и **блока объявлений**.

```css
/* Это комментарий в CSS */

селектор {
  свойство: значение;
  свойство2: значение2;
}
```

*   **Селектор (Selector):** Указывает, к какому HTML-элементу (или элементам) будет применено правило. В примере выше селектор — это `p`.
*   **Блок объявлений (Declaration Block):** Находится в фигурных скобках `{}`. Содержит одно или несколько объявлений.
*   **Объявление (Declaration):** Состоит из пары "свойство: значение" и заканчивается точкой с запятой `;`.
    *   **Свойство (Property):** Аспект элемента, который вы хотите изменить (например, `color`, `font-size`, `background-color`).
    *   **Значение (Value):** Значение, которое вы присваиваете свойству (например, `red`, `14px`, `#ffffff`).

Точка с запятой после последнего объявления в блоке не обязательна, но **настоятельно рекомендуется** ее ставить, чтобы избежать ошибок при добавлении новых стилей.

### Глава 3: Селекторы — Искусство выбора элементов

Селекторы — это сердце CSS. Они позволяют с высокой точностью выбирать элементы для стилизации.

#### Основные селекторы

1.  **Селектор по тегу (Type Selector):** Выбирает все элементы с указанным именем тега.
    ```css
    p { /* Все параграфы <p> */
        line-height: 1.5;
    }
    h2 { /* Все заголовки <h2> */
        color: #444;
    }
    ```

2.  **Селектор по классу (Class Selector):** Выбирает все элементы с указанным атрибутом `class`. Начинается с точки `.`. Классы можно применять к нескольким элементам.
    *HTML:*
    ```html
    <p class="highlight">Этот текст выделен.</p>
    <span class="highlight">И этот тоже.</span>
    ```
    *CSS:*
    ```css
    .highlight {
        background-color: yellow;
        font-weight: bold;
    }
    ```

3.  **Селектор по идентификатору (ID Selector):** Выбирает один-единственный элемент с указанным атрибутом `id`. Начинается с решетки `#`. **ID на странице должен быть уникальным!**
    *HTML:*
    ```html
    <div id="main-header">Главный заголовок сайта</div>
    ```
    *CSS:*
    ```css
    #main-header {
        padding: 20px;
        background-color: navy;
        color: white;
    }
    ```

#### Комбинирование селекторов

1.  **Группировка (Grouping Selector):** Применяет одни и те же стили к нескольким селекторам, перечисляя их через запятую.
    ```css
    h1, h2, h3 {
        font-family: 'Georgia', serif;
        color: #333;
    }
    ```

2.  **Селектор потомка (Descendant Selector):** Выбирает элементы, которые находятся внутри (являются потомками) другого элемента. Используется пробел.
    ```css
    /* Выбрать все ссылки <a>, которые находятся где-либо внутри <div> с классом .content */
    .content a {
        color: green;
    }
    ```

3.  **Дочерний селектор (Child Selector):** Выбирает только прямых потомков (детей) элемента. Используется знак `>`.
    ```css
    /* Выбрать только те <p>, которые являются прямым потомком <div> */
    div > p {
        margin-bottom: 0;
    }
    ```

4.  **Смежный селектор (Adjacent Sibling Selector):** Выбирает элемент, который идет *непосредственно после* другого элемента на том же уровне вложенности. Используется знак `+`.
    ```css
    /* Выбрать параграф <p>, который идет сразу после заголовка <h1> */
    h1 + p {
        font-style: italic;
    }
    ```

#### Псевдоклассы и псевдоэлементы

*   **Псевдоклассы:** Указывают на определенное состояние селектора. Начинаются с одного двоеточия `:`.
    *   `:hover` — когда курсор мыши наведен на элемент.
    *   `:focus` — когда элемент в фокусе (например, поле ввода).
    *   `:first-child` — первый дочерний элемент.
    *   `:last-child` — последний дочерний элемент.
    ```css
    a:hover { /* Ссылка при наведении */
        text-decoration: none;
    }
    input:focus { /* Поле ввода в фокусе */
        border-color: blue;
    }
    ```

*   **Псевдоэлементы:** Позволяют стилизовать определенную *часть* элемента. Начинаются с двух двоеточий `::` (старый синтаксис с одним двоеточием тоже поддерживается для обратной совместимости).
    *   `::before` — добавляет контент перед содержимым элемента.
    *   `::after` — добавляет контент после содержимого элемента.
    *   `::first-letter` — первая буква текста.
    *   `::first-line` — первая строка текста.
    ```css
    p::first-letter {
        font-size: 2em;
        font-weight: bold;
        color: red;
    }
    .important::before {
        content: "Важно: "; /* Обязательное свойство для ::before и ::after */
        color: red;
    }
    ```

### Глава 4: Каскад, Специфичность и Наследование

Эти три концепции определяют, почему CSS называется "каскадным" и как разрешаются конфликты стилей.

#### 1. Каскад (Cascade)

Если к одному элементу подходят несколько правил CSS, браузер определяет, какое из них применить, на основе следующего порядка (от низшего приоритета к высшему):

1.  **Стили браузера по умолчанию** (User-agent stylesheet).
2.  **Стили пользователя** (кастомные стили, которые пользователь может задать в своем браузере).
3.  **Стили автора** (это ваши CSS-файлы).
    *   Стили из внешних файлов (`<link>`).
    *   Стили из внутренней таблицы (`<style>`).
    *   Встроенные стили (атрибут `style`).

Если два правила имеют одинаковый источник и вес, **побеждает то, которое было объявлено в коде позже**.

#### 2. Специфичность (Specificity)

Когда несколько правил от автора (из вашего CSS) нацелены на один и тот же элемент, браузер вычисляет "вес" или специфичность каждого селектора. **Правило с более специфичным селектором побеждает.**

Вес можно грубо представить как число из трех разрядов `(ID, CLASS, TAG)`:

*   **ID (`#id`):** `1, 0, 0` — самый высокий вес.
*   **Классы (`.class`), псевдоклассы (`:hover`), селекторы атрибутов (`[type="text"]`):** `0, 1, 0`.
*   **Теги (`p`), псевдоэлементы (`::before`):** `0, 0, 1` — самый низкий вес.

**Пример:**
```html
<div id="content" class="main">
    <p class="text">Какой стиль применится к этому параграфу?</p>
</div>
```
```css
/* (0,0,1) - вес 1 */
p { color: black; } 

/* (0,1,1) - вес 11 */
.main p { color: blue; } 

/* (1,1,0) - вес 110. Этот стиль победит! */
#content .text { color: red; } 
```
Встроенные стили (`style="..."`) имеют еще больший вес, чем любой ID.

#### 3. Наследование (Inheritance)

Некоторые CSS-свойства (но не все!) по умолчанию передаются от родительского элемента к его дочерним элементам. Это называется наследованием.

*   **Наследуются:** `color`, `font-family`, `font-size`, `font-weight`, `line-height`, `text-align`.
*   **Не наследуются:** `width`, `height`, `padding`, `margin`, `border`, `background-color`.

Это логично: если вы задали для `<body>` синий цвет текста, вы ожидаете, что все параграфы и заголовки внутри тоже станут синими. Но вы не ожидаете, что все они получат рамку, если вы задали ее только для `<body>`.

### Глава 5: Блочная модель документа (Box Model)

Каждый элемент на HTML-странице — это прямоугольный блок. Блочная модель описывает, из чего состоит этот блок.



1.  **Content (Содержимое):** Область, где отображается текст, изображения и т.д. Ее размеры задаются свойствами `width` и `height`.
2.  **Padding (Внутренний отступ):** Прозрачное пространство *внутри* рамки, между содержимым и рамкой.
3.  **Border (Рамка):** Линия, которая обрамляет padding и content. Имеет толщину, стиль и цвет.
4.  **Margin (Внешний отступ):** Прозрачное пространство *снаружи* рамки, которое "отталкивает" другие элементы.

#### Свойство `box-sizing`

По умолчанию (`box-sizing: content-box;`), когда вы задаете `width` и `height` элементу, вы задаете размеры только для его *содержимого* (Content). `Padding` и `Border` добавляются сверху, увеличивая фактический размер элемента на странице. Это часто неудобно.

**Решение:** `box-sizing: border-box;`
Это свойство говорит браузеру: "Когда я задаю `width: 200px`, я хочу, чтобы *весь блок*, включая `padding` и `border`, был шириной 200px". Это гораздо интуитивнее.

**Лучшая практика:** добавить это правило для всех элементов в начале вашего CSS-файла.
```css
*,
*::before,
*::after {
    box-sizing: border-box;
}
```

### Глава 6: Основные свойства для форматирования

#### Работа с текстом и шрифтами
*   `color`: цвет текста.
*   `font-family`: список шрифтов (если первый не найден, используется следующий).
*   `font-size`: размер шрифта.
*   `font-weight`: жирность шрифта (`normal`, `bold`, `100`-`900`).
*   `font-style`: начертание (`normal`, `italic`).
*   `text-align`: горизонтальное выравнивание (`left`, `right`, `center`, `justify`).
*   `text-decoration`: подчеркивание, зачеркивание (`none`, `underline`, `line-through`).
*   `line-height`: межстрочный интервал.

#### Цвета и фоны
*   `background-color`: цвет фона.
*   `background-image`: фоновое изображение (`url('path/to/image.jpg')`).
*   `background-repeat`: повторение фона (`no-repeat`, `repeat-x`, `repeat-y`, `repeat`).
*   `background-position`: положение фона (`center`, `top right`).
*   `background-size`: размер фона (`cover`, `contain`).

#### Размеры и отображение
*   `width`, `height`: ширина и высота элемента.
*   `min-width`, `max-width`, `min-height`, `max-height`: минимальные и максимальные размеры.
*   **`display`**: одно из важнейших свойств, определяет тип блока.
    *   `block`: Элемент занимает всю доступную ширину, начинается с новой строки (`<div>`, `<p>`, `<h1>`).
    *   `inline`: Элемент занимает только необходимое пространство, не начинается с новой строки (`<span>`, `<a>`, `<strong>`). Нельзя задать `width` и `height`.
    *   `inline-block`: Гибрид. Ведет себя как `inline` (встраивается в строку), но ему можно задавать `width`, `height`, `margin` и `padding` как `block`-элементу.
    *   `none`: Скрывает элемент со страницы (он не просто невидим, он полностью убирается из потока документа).
    *   `flex`, `grid`: Современные мощные значения для создания гибких макетов (рассматриваются в более продвинутых курсах).

#### Позиционирование (`position`)
Определяет, как элемент расположен на странице.
*   `static`: По умолчанию. Элемент находится в обычном потоке документа.
*   `relative`: Элемент остается в потоке, но его можно сдвинуть относительно его *изначальной* позиции с помощью свойств `top`, `right`, `bottom`, `left`.
*   `absolute`: Элемент **вырывается из потока** документа и позиционируется относительно ближайшего родителя с `position` не `static`. Если такого родителя нет, то относительно `<body>`.
*   `fixed`: Элемент **вырывается из потока** и позиционируется относительно окна браузера (вьюпорта). Он не прокручивается вместе со страницей.
*   `sticky`: Гибрид `relative` и `fixed`. Элемент ведет себя как `relative` до тех пор, пока при прокрутке не достигнет определенной точки, после чего "прилипает" и ведет себя как `fixed`.

### Глава 7: Единицы измерения в CSS

#### Абсолютные
Не изменяются в зависимости от других элементов или настроек экрана.
*   `px` (пиксели): Самая распространенная единица. 1px — одна точка на экране.

#### Относительные
Зависят от размера другого элемента или окна браузера. Они очень важны для создания адаптивных сайтов.
*   `%` (проценты): Зависят от размера родительского элемента. `width: 50%` — ширина элемента будет равна половине ширины его родителя.
*   `em`: Относительно размера шрифта **родительского** элемента. Если у родителя `font-size: 16px`, то `2em` будет `32px`.
*   `rem` (root em): Относительно размера шрифта **корневого** элемента (`<html>`). Это предпочтительная единица для шрифтов и отступов, так как она позволяет легко масштабировать весь интерфейс, изменяя лишь `font-size` у `<html>`.
*   `vw` (viewport width) и `vh` (viewport height): 1vw — это 1% от ширины окна браузера, 1vh — 1% от высоты. Идеально для элементов, которые должны занимать весь экран.

### Заключение и дальнейшие шаги

Мы рассмотрели фундаментальные концепции CSS, которые являются основой для стилизации любого веб-документа. Понимание того, как работают селекторы, каскад, блочная модель и позиционирование, — это ключ к созданию красивых, предсказуемых и поддерживаемых веб-сайтов.

**Практика — ваш лучший учитель.** Попробуйте создать простую HTML-страницу и применить к ней все описанные свойства. Экспериментируйте с селекторами и значениями.

**Следующие шаги для изучения:**
1.  **Адаптивная верстка и медиа-запросы (`@media`):** Как делать сайты, которые хорошо выглядят и на компьютерах, и на телефонах.
2.  **Flexbox и Grid Layout:** Современные и мощные инструменты для построения сложных сеток и выравнивания элементов.
3.  **CSS-переходы и анимации (`transition`, `animation`):** Как добавить интерактивности и жизни на вашу страницу.
4.  **CSS-препроцессоры (Sass/Less):** Инструменты, которые добавляют в CSS переменные, функции, вложенность и делают работу с большими проектами намного удобнее.
