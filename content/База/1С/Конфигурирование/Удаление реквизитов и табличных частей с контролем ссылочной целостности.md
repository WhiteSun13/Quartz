## Удаление реквизитов и табличных частей с контролем ссылочной целостности

При удалении объекта конфигурации (реквизита, табличной части) платформа проверяет, не используется ли он в других местах.

### Что происходит при попытке удалить используемый объект

При удалении платформа выдаёт предупреждение:

> *«Код модулей содержит ссылки на удаляемый объект. Продолжить?»*

> [!warning]
> Нажать **«Нет»** — и сначала убрать все ссылки на удаляемый объект вручную.

### Поиск ссылок на объект

1. Выделить объект (например, табличную часть **«Услуги»**)
2. Правая кнопка мыши → выбрать нужный пункт:

| Пункт меню | Что делает |
|---|---|
| **Поиск ссылок на объект** | Ищет, **где используется** данный объект |
| **Поиск ссылок в объекте** | Ищет, **что используется внутри** данного объекта |

3. Нажать **ОК**
4. В служебных сообщениях будут показаны все места использования

### Порядок безопасного удаления

```
1. Поиск ссылок на объект
2. Устранить все найденные ссылки (удалить или исправить код)
3. Удалить объект
```

### Пример: удаление табличной части «Услуги» из «Приходной накладной»

**Шаг 1.** Поиск ссылок → найдена ссылка в модуле объекта:

```bsl
// В процедуре ПередЗаписью:
ИтоговаяСумма = Товары.Итог("Сумма") + Услуги.Итог("Сумма");
//                                        ^^^^^^^^^^^^^^^^^^^^ — ссылка на удаляемую ТЧ
```

**Шаг 2.** Исправить код — убрать обращение к табличной части «Услуги»:

```bsl
// БЫЛО:
ИтоговаяСумма = Товары.Итог("Сумма") + Услуги.Итог("Сумма");

// СТАЛО:
ИтоговаяСумма = Товары.Итог("Сумма");
```

**Шаг 3.** Вернуться на вкладку **«Данные»** → выделить **«Услуги»** → нажать **крестик** → подтвердить: **«Да»**
