# Проверка заполнения реквизитов

## Зачем управлять проверкой

По умолчанию платформа проверяет заполненность реквизитов **до выполнения кода**. Если реквизит заполняется программно (например, «Наименование» формируется из Фамилии, Имени и Отчества), платформа выдаст ошибку «Поле не заполнено» раньше,
чем код успеет его заполнить.

## Варианты свойства «Проверка заполнения»

| Значение | Поведение |
|---|---|
| **Выдавать ошибку** | Платформа не разрешает запись, если поле пустое |
| **Не проверять** | Платформа не выдаёт ошибку, поле может быть пустым |

## Как изменить проверку

### Для стандартных реквизитов (Наименование, Код)

1. Раскрыть объект в дереве конфигурации
2. Вкладка **«Данные»** → **«Стандартные реквизиты»**
3. Двойной клик по нужному реквизиту
4. Вкладка **«Представление»**
5. Свойство **«Проверка заполнения»** → выбрать нужное значение

### Для пользовательских реквизитов

1. Вкладка **«Данные»** → список реквизитов
2. Двойной клик по реквизиту
3. Вкладка **«Представление»** → **«Проверка заполнения»**

## Пример: справочник «Сотрудники»

Реквизит «Наименование» (ФИО) заполняется программно в обработчике «Перед записью» — поэтому проверку нужно отключить:

| Реквизит | Проверка заполнения | Причина |
|---|---|---|
| **Наименование** | Не проверять | Заполняется программно из Фамилии, Имени, Отчества |
| **Фамилия** | Выдавать ошибку | Обязательное поле |
| **Имя** | Выдавать ошибку | Обязательное поле |
| **Отчество** | Не проверять | Может отсутствовать |

> [!warning]
> Если оставить проверку «Выдавать ошибку» для программно
> заполняемого реквизита — пользователь получит ошибку
> при каждой записи, даже если он правильно заполнил
> Фамилию, Имя и Отчество.
