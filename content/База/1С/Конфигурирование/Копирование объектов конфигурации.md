## Копирование объектов конфигурации

Метод позволяет быстро создавать новые объекты на основе существующих — копируются все реквизиты, табличные части и код модулей.

### Как скопировать объект

1. Выделить объект в дереве конфигурации (например, **«Заказ клиента»**)
2. Правая кнопка мыши → **«Скопировать»**
3. Появится копия с именем вида **«ЗаказКлиента1»**
4. Двойной клик по копии → изменить **Имя**, **Представление объекта**, **Представление списка**

### Пример: создание «Расходной накладной» на основе «Заказ клиента»

| Свойство | Значение |
|---|---|
| **Имя** | `РасходнаяНакладная` |
| **Представление объекта** | `Расходная накладная` |
| **Представление списка** | `Расходные накладные` |

### Пример: создание «Приходной накладной» на основе «Заказ клиента»

| Свойство | Значение |
|---|---|
| **Имя** | `ПриходнаяНакладная` |
| **Представление объекта** | `Приходная накладная` |
| **Представление списка** | `Приходные накладные` |

### Проверки после копирования

> [!warning]
> После копирования обязательно проверьте:
> 1. Откройте форму скопированного объекта
> 2. Выделите любой элемент → вкладка **«Основное»** → проверьте **«Путь к данным»**
> 3. Вкладка **«События»** → проверьте, что обработчики **привязаны** (поле не пустое)

> [!danger]
> **Частая проблема:** после копирования события могут оказаться **непривязанными** — поле события пустое. Нужно вручную выбрать нужную процедуру для каждой колонки.
