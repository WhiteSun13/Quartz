## ⚡ Главная идея Linux

> [!important] Ключевая концепция
> **Всё** в Linux — это **файл**. Буквально всё:
> - **Конфигурации** — сетевые настройки, IP-адреса, интерфейсы — это файлы
> - **Устройства** — жёсткие диски, принтеры, CD-ROM — представлены как файлы
> - **Команды** — `ls`, `cat` и все остальные команды — тоже файлы

---

## 📂 Структура файловой системы Linux (FHS)

### 🗂 `/bin` — Основные команды (Binaries)

**bin** — сокращение от **binaries** (бинарные файлы). Внутри папки `/bin` находятся **основные команды** системы.

```bash
cd /bin
ls
```

Здесь живут все **essential command binaries** — основные команды, которые вы будете постоянно использовать в Linux. Среди них — `ls`, `cat`, `cp`, `rm` и многие другие.

#### 🔬 Доказательство: команды — это файлы

Раз `ls` — это файл, мы можем его **открыть**:

```bash
cat ls
```

На экран выведется куча бинарного «мусора» — и это нормально. Это то, что компьютер понимает, когда вы вводите команду `ls`. Если вы это понимаете — вы, возможно, робот. 🤖

---

### 🆕 Новые команды: `cp`, `rm`, `sudo`

#### 📋 Команда `cp` — копирование

```bash
cp <исходный_файл> <новый_файл>
```

**cp** = **copy** — копирует файл.

Пример — копируем команду `ls` в файл с названием `networkchuck`:

```bash
cp ls networkchuck
```

> [!warning] Permission denied!
> Иногда в Linux нужно сказать **«пожалуйста»**. Для этого существует команда **`sudo`**.

#### 🔐 Команда `sudo` — выполнение от имени администратора

```bash
sudo cp ls networkchuck
```

**`sudo`** позволяет **притвориться** root-пользователем (администратором) **для одной команды**. Мы подробно разберём `sudo` позже, а пока — просто знайте, что если вам не хватает прав, добавьте `sudo` перед командой.

Результат: теперь у нас есть и `ls`, и `networkchuck` — и оба работают одинаково!

```bash
networkchuck
```

Выводит то же самое, что и `ls` — потому что мы **скопировали бинарное содержимое** `ls` в новый файл.

#### 🗑 Команда `rm` — удаление

```bash
rm <файл>
```

**rm** = **remove** — удаляет файл.

```bash
sudo rm ls
```

После удаления `ls` команда перестаёт работать! Но не беда — у нас есть копия `networkchuck`:

```bash
sudo cp networkchuck ls
```

И `ls` снова работает.

#### Сводная таблица новых команд

| Команда | Расшифровка | Описание |
|---------|------------|----------|
| `cp` | copy | Копировать файл |
| `rm` | remove | Удалить файл |
| `sudo` | superuser do | Выполнить команду от имени администратора |
| `cat` | concatenate | Вывести содержимое файла на экран |

> [!tip] Забавный факт
> Можно сделать `cat cat` — вывести содержимое файла команды `cat`. Мы только что «за-cat-или cat»! 🐱

### 🗂 `/sbin` — Суперкоманды (Super Binaries)

**`/sbin`** — как `/bin`, но **для администраторов**. Здесь хранятся специальные команды, которые используются только для **администрирования системы**.

```bash
cd /sbin
ls
```

#### 🆕 Команда `adduser` — добавление пользователя

```bash
sudo adduser superbin
```

Эта команда создаёт **нового пользователя** в системе. Потребуется задать пароль и (опционально) полное имя.

> [!note] Примечание
> Команды из `/sbin` требуют прав администратора (`sudo`), потому что они управляют системой.

---

### 🗂 `/usr` — Пользовательские программы (User)

```bash
cd /usr
ls
```

Внутри `/usr` обнаруживается **кое-что странное** — здесь тоже есть `bin` и `sbin`!

```bash
cd /usr/bin
ls
# Тут тоже есть ls и другие команды!

cd /usr/sbin
ls
# И тут тоже!
```

> [!note] Почему два набора команд?
> Существует интересная **историческая причина**, связанная с **нехваткой места на жёстких дисках** в ранние годы Unix. По сути, `bin` и `sbin` в `/usr` — **то же самое**, что `/bin` и `/sbin` в корне, но `/usr/bin` и `/usr/sbin` обычно содержат **больше команд**. Между ними много пересечений.

#### 🆕 Команда `which` — где живёт команда?

```bash
which ls
# /usr/bin/ls

which cat
# /usr/bin/cat

which networkchuck
# /bin/networkchuck
```

**`which`** показывает **полный путь** к бинарному файлу команды. Позволяет узнать, **откуда именно** запускается та или иная команда.

#### Другие поддиректории `/usr`:

| Директория | Описание |
|-----------|----------|
| `/usr/bin` | Команды (дубль `/bin`, обычно больше) |
| `/usr/sbin` | Административные команды (дубль `/sbin`) |
| `/usr/local` | Место для команд, которые **вы создаёте сами** |
| `/usr/lib` | Библиотеки, которые используют команды |
| `/usr/games` | Игры! 🎮 |

---

### ⚡ Быстрый обзор (Speed Round)

| Директория | Описание |
|-----------|----------|
| `/boot` | Файлы, необходимые системе для **загрузки** (boot files) |
| `/var` | **Логи**, файлы веб-приложений и другие переменные данные |
| `/tmp` | **Временные файлы** — удаляются после перезагрузки системы |
| `/lib` | **Библиотеки**, необходимые для загрузки системы |

### 🏠 `/home` — Домашние директории пользователей

**`/home`** — здесь **живёт** каждый пользователь системы.

```bash
cd /home
ls
```

Внутри — домашние папки для каждого пользователя. Например, наша директория и директория `superbin` (пользователь, которого мы создали ранее).

```bash
cd /home/superbin
ls
# Вот его «дом» — пока пустой
```

#### А где живёт root?

> [!important] Особенность root-пользователя
> У каждой Linux-системы есть пользователь **root**, но его домашняя директория **не в `/home`**. Root живёт **отдельно** — в **`/root`**. Он особенный, живёт отдельно, где его никто не побеспокоит. Он своего рода затворник.

```bash
sudo ls /root
```

Для просмотра содержимого `/root` нужны **права администратора** (sudo).

---

### 💾 `/dev` — Устройства (Devices)

**`/dev`** — сокращение от **devices**. Здесь все устройства системы представлены как **файлы**.

```bash
cd /dev
ls
```

#### Жёсткие диски как файлы

| Файл | Описание |
|------|----------|
| `vda`, `vda1` | Виртуальные диски (virtual disk) |
| `sda`, `sda1` | Физические диски (на других системах) |

Раз это файл — можно его **cat-нуть**:

```bash
sudo cat vda
```

> [!danger] Осторожно!
> Эта команда выведет **ВСЁ** содержимое жёсткого диска в виде бинарных данных — экран «взорвётся». Чтобы остановить — нажмите **`Ctrl+C`**.

Да, ваш жёсткий диск — это буквально **файл**. Это одновременно круто и невероятно странно.

---

### ⚙️ `/etc` — Конфигурации (Etcetera / «Etsy»)

**`/etc`** — официально расшифровывается как **et cetera** (и так далее). В Linux-сообществе эту директорию часто называют **«Etsy»** (и она была первой — раньше, чем Etsy.com! 😄).

Здесь хранятся **конфигурационные файлы** системы и некоторых приложений. Сюда вы будете заходить **очень часто**.

```bash
cd /etc
ls
```

#### Пример: сетевые настройки

```bash
cd /etc/network
ls
sudo cat interfaces
```

> [!success] Результат
> Файл `interfaces` содержит **сетевые интерфейсы** и назначенные им **IP-адреса**. Именно здесь они настраиваются. Если нужно изменить сетевые настройки — часто достаточно **отредактировать этот файл**.

> [!note] Примечание
> На некоторых современных Linux-системах сетевые настройки работают иначе, но этот способ остаётся **распространённым** на многих системах.

---

### 📀 `/mnt` и `/media` — Монтирование дисков

Обе директории служат для **монтирования дисков**, но по-разному:

| Директория | Назначение |
|-----------|-----------|
| `/media` | Диски, подключённые **автоматически** системой (например, USB-флешка) |
| `/mnt` | Диски, смонтированные **вручную** пользователем с помощью команд |

> [!note] Напоминание
> Подключённый USB-диск будет представлен как **файл** в `/media` — потому что **всё в Linux — это файл**!

## 📋 Итоговая сводка

### 🗺 Карта файловой системы Linux (FHS — File System Hierarchy)

| Директория | Назначение |
|-----------|-----------|
| `/` | **Корень** файловой системы — всё начинается здесь |
| `/bin` | **Основные команды** (essential binaries) |
| `/sbin` | **Административные команды** (system binaries) |
| `/usr` | **Пользовательские программы** (содержит свои bin, sbin, local, lib, games) |
| `/usr/local` | Место для **ваших собственных** программ и скриптов |
| `/boot` | Файлы **загрузки** системы |
| `/var` | **Логи**, файлы веб-приложений, переменные данные |
| `/tmp` | **Временные файлы** (удаляются после перезагрузки) |
| `/lib` | **Библиотеки** для загрузки системы |
| `/home` | **Домашние директории** пользователей |
| `/root` | **Домашняя директория** пользователя root (отдельно от `/home`) |
| `/dev` | **Устройства** (жёсткие диски, принтеры и т.д.) — всё как файлы |
| `/etc` | **Конфигурационные файлы** системы и приложений |
| `/media` | **Автоматически** смонтированные внешние устройства |
| `/mnt` | **Вручную** смонтированные устройства |

### 🆕 Все новые команды эпизода

| Команда | Описание |
|---------|----------|
| `whoami` | Узнать, под каким пользователем вы вошли |
| `clear` / `Ctrl+L` | Очистить экран терминала |
| `cat <файл>` | Вывести содержимое файла (**concatenate**) |
| `cp <откуда> <куда>` | Скопировать файл (**copy**) |
| `rm <файл>` | Удалить файл (**remove**) |
| `sudo <команда>` | Выполнить команду от имени администратора |
| `adduser <имя>` | Создать нового пользователя |
| `which <команда>` | Узнать путь к бинарному файлу команды |
| `Ctrl+C` | Прервать выполнение текущей команды |
