## 📌 Что такое язык программирования?

> [!important]
> **Язык программирования** — это просто набор правил, по которым вы пишете текстовые инструкции к вашей программе.

**Проблема:** компьютер понимает только **машинный код**.

- **Машинный код** — особый язык программирования, понятный компьютеру, но мало понятный человеку.
- Мы хотим программировать на удобных языках, а компьютер понимает только машинный код.
- Значит, нужно решить эту проблему — научить компьютер «понимать» наши языки.

## 🔄 Компиляция vs. Интерпретация

### 🏗 Компиляция

> [!info]
> **Компиляция** — это перевод из языка программирования в машинный код с помощью специальной вспомогательной программы — **компилятора**.

**Процесс:**
1. Пишем программу на языке (например, C++).
2. Скармливаем исходный код **компилятору**.
3. Компилятор выдаёт перевод в машинном коде.
4. Полученный машинный код запускаем на компьютере.

```
Исходный код (C++) → [Компилятор] → Машинный код → Запуск
```

### 🎙 Интерпретация

> [!info]
> **Интерпретация** — аналог **синхронного перевода**. Не требует предварительного перевода: специальная программа-**интерпретатор** параллельно читает программу, синхронно переводит на машинный код и выполняет её.

```
Исходный код → [Интерпретатор] → Синхронный перевод + Выполнение
```

### ⚖️ Сравнение подходов

| Критерий | Компиляция | Интерпретация |
|---|---|---|
| Скорость работы | Быстрее | Медленнее |
| Качество перевода | Выше (анализируется вся программа) | Ниже (перевод «на ходу») |
| Кроссплатформенность | Сложнее (нужен отдельный перевод для каждого типа ОС) | Проще (свой интерпретатор на каждом компьютере) |
| Ресурсы | Эффективнее | Больше ограничений |

> [!note]
> Разные компьютеры понимают **разный машинный код**. При компиляции нужно готовить отдельный перевод для:
> - Windows / macOS / Linux
> - 32-разрядных / 64-разрядных систем
>
> При интерпретации достаточно посадить на каждый компьютер «своего» синхронного переводчика.

## ☕ Как Java решила проблему — Байт-код

> [!important]
> Java придумала **гибридный подход**: свой собственный промежуточный язык — **байт-код**.

**Схема работы Java:**

```
Исходный код Java (.java)
        ↓
   [Компилятор Java]
        ↓
  Байт-код (.class)
        ↓
[JVM — интерпретатор байт-кода]
        ↓
  Машинный код компьютера
        ↓
      Запуск
```

**Ключевые особенности:**
1. Программист пишет код на **Java**.
2. Компилятор переводит его в **байт-код** — особый машинный язык, которого не понимает ни один реальный компьютер.
3. На целевом компьютере установлен **встроенный интерпретатор байт-кода** (JVM), который переводит байт-код в машинный язык этого конкретного компьютера.
4. Так как байт-код сам является машинным языком — интерпретатор байт-кода в машинный код работает **достаточно быстро**.

> [!success]
> **Результат:** написал один раз → запускай хоть где. Байт-код одинаковый, а JVM на каждом компьютере своя.

## 🛠 Инструменты Java-разработчика

### 🖥 JVM — Java Virtual Machine

> [!info]
> **JVM (Java Virtual Machine)** — виртуальная машина Java.
> 
> Это тот самый интерпретатор байт-кода в машинный код компьютера, на котором запускается программа. По сути, программа, которая **притворяется компьютером**, понимающим байт-код. На самом деле она его синхронно переводит в машинный код.

> [!tip]
> Благодаря JVM появились **JVM-языки** — языки, которые компилируются в байт-код и запускаются на JVM:
> - **Kotlin**
> - **Scala**
> - и другие

### 📦 JRE — Java Runtime Environment

> [!info]
> **JRE (Java Runtime Environment)** — рабочее окружение Java.
>
> Минимальный набор инструментов для **запуска** готовой Java-программы. Включает JVM и стандартные библиотеки, на которые программа рассчитывает при запуске.

**Аналогия:**
> «Как один марксист пишет письмо другому и ссылается на стандартные труды Ленина — он не прикладывает к письму все томики, а рассчитывает, что у адресата они уже есть.»

### 🔧 JDK — Java Development Kit

> [!info]
> **JDK (Java Development Kit)** — набор инструментов разработчика Java.
>
> Включает в себя всё из JRE + инструменты, необходимые **во время разработки**, но не во время запуска. Например, **компилятор** из Java-кода в байт-код.

### 📊 Иерархия инструментов

| Инструмент | Расшифровка | Назначение | Содержит |
|---|---|---|---|
| **JVM** | Java Virtual Machine | Интерпретирует байт-код в машинный код | — |
| **JRE** | Java Runtime Environment | Запуск Java-программ | JVM + стандартные библиотеки |
| **JDK** | Java Development Kit | Разработка и запуск Java-программ | JRE + компилятор + инструменты разработки |

## 📝 Разбор кода Hello World

```java
class Main {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```

### Построчный разбор

| Строка | Код | Объяснение |
|---|---|---|
| 1 | `class Main {` | **Класс** — кусок программы, находящийся в одном файле. Файл `Main.java` → класс `Main` (без `.java`) |
| 2 | `public static void main(String[] args) {` | **Главный метод** — точка входа. При запуске Java находит эту фразу и выполняет всё внутри фигурных скобок |
| 3 | `System.out.println("Hello World");` | **Команда вывода на экран**. Выводит то, что указано в круглых скобках |

> [!important]
> - Всё, что находится **в фигурных скобках** после `main` — это команды программы, которые выполняются построчно.
> - **Точка с запятой (`;`)** — обязательный знак в конце каждой команды.
> - Текст в **двойных кавычках** (`"..."`) — для Java это просто набор символов (строка/текст), она не пытается его анализировать как код.

### ▶️ Запуск программы

1. Нажать большую зелёную кнопку **Run**.
2. Replit сам выполнит компиляцию и запуск.
3. В консоли появятся системные команды Replit, а затем — **вывод вашей программы**.

> [!success]
> Результат в консоли: `Hello World`

## 🖨 Команда вывода на экран

```java
System.out.println(что_вывести);
```

### Варианты использования

**1. Вывод текста (строки):**
```java
System.out.println("Hello World");
// Выведет: Hello World
```

**2. Вывод числового выражения без кавычек:**
```java
System.out.println(2 + 3);
// Выведет: 5  (Java посчитает!)
```

**3. Вывод текста как текста (с кавычками):**
```java
System.out.println("2 + 3");
// Выведет: 2 + 3  (Java НЕ считает — это просто текст)
```

**4. Склейка (конкатенация) строк:**
```java
System.out.println("hello" + " " + "world");
// Выведет: hello world
```

> [!warning]
> Если убрать двойные кавычки вокруг текста — Java попытается интерпретировать его как код. Если это не валидный Java-код, программа **не скомпилируется** и выдаст ошибку.

### 🎨 Подсветка синтаксиса в редакторе

| Цвет | Значение |
|---|---|
| Тёмно-красный | Текст в двойных кавычках (просто строка) |
| Зелёный / Чёрный | Java-код, который анализируется |

## 📦 Локальные переменные

> [!important]
> **Локальная переменная** — это ячейка в памяти, которую мы просим Java выделить для хранения данных.

**Для создания переменной нужно указать:**
1. **Тип** — что будем хранить.
2. **Название** — имя переменной.
3. (Опционально) **Значение** — что сразу кладём.

### Синтаксис объявления переменной

```java
тип название = значение;
```

**Примеры:**
```java
int x = 5;        // объявление с присвоением
int x;            // объявление без присвоения (значение добавим позже)
x = 3;            // присвоение нового значения (тип НЕ повторяем!)
```

> [!warning]
> Тип указывается **только при создании** переменной. При последующем изменении тип не пишется.

---

## 🗃 Основные типы данных

| Тип | Назначение | Пример |
|---|---|---|
| `int` | Целые числа | `int x = 5;` |
| `double` | Дробные числа | `double d = 3.5;` |
| `String` | Текст (строки) | `String name = "Петя";` |

### 🔢 int — целые числа

```java
int x = 5;
System.out.println(x);  // Выведет: 5

x = 3;                  // Теперь x = 3
System.out.println(x);  // Выведет: 3
```

> [!warning]
> В переменную типа `int` нельзя положить:
> - Дробное число (`5.5`)
> - Текст (`"hello"`)
> Только целые числа!

### 🔢 double — дробные числа

```java
double d = 3.5;
System.out.println(d);  // Выведет: 3.5
```

> [!warning]
> Дробные числа пишутся через **точку**, не через запятую: `3.5`, а не `3,5`.

### 🔤 String — текст (строки)

```java
String name = "Петя";
System.out.println("Hello " + name);  // Выведет: Hello Петя
```

> [!note]
> **String** пишется с **большой буквы**! В Java регистр имеет огромное значение — `string` и `String` это разные вещи.

## ⚙️ Операции с переменными

### Математические операции

```java
int x = 5;
int y = x + 1;  // y = 6

System.out.println(x);  // Выведет: 5
System.out.println(y);  // Выведет: 6
```

### Важное отличие от математики!

> [!danger]
> Знак `=` в Java — это **не утверждение** «y всегда равно x + 1».  
> Это **приказ**: «посчитай то, что справа, и положи в то, что слева».

```java
int x = 5;
int y = x + 1;  // Считается СЕЙЧАС: y = 6

x = 10;         // Меняем x
// y по-прежнему равен 6! Он не пересчитывается автоматически.

System.out.println(x);  // Выведет: 10
System.out.println(y);  // Выведет: 6 (не 11!)
```

**Как Java выполняет строку `int y = x + 1;`:**
1. Создаёт ячейку памяти с именем `y`.
2. Вычисляет выражение справа от `=`: вынимает значение из `x` (там `5`), прибавляет `1`, получает `6`.
3. Кладёт `6` в переменную `y`.
4. После этого Java уже **не помнит**, как вычислялся `y`. Там просто лежит `6`.

### Чтение значения переменной

> [!note]
> Вынимая значение из переменной, мы его **не удаляем** — оно остаётся там же. Мы лишь его «посмотрели».

```java
int x = 5;
System.out.println(x);  // Смотрим значение — в x всё так же лежит 5
System.out.println(x);  // Снова можно вывести — там по-прежнему 5
```

## 🏷 Именование переменных

### Технические правила Java

- Можно использовать: **буквы латинского алфавита**, **цифры**, **`_`** (подчёркивание), **`$`** (доллар).
- **Цифра не может быть в начале** имени переменной.
- Нельзя создать две переменные с **одинаковым именем** в одной области.

### Правила форматирования кода (обязательные соглашения)

> [!important]
> Java технически не запрещает нарушать соглашения, но нарушать их **не принято** — это признак плохого кода.

**Для переменных из одного слова:**
```java
int score;
String name;
```

**Для переменных из нескольких слов — `camelCase` (верблюжий стиль):**

> [!info]
> **camelCase (верблюжий стиль)** — все слова объединяются в одно, каждое следующее слово начинается с **большой буквы**. Название напоминает горбы верблюда.

```java
// ❌ Неправильно (нельзя — есть пробелы):
// int my best score;

// ✅ Правильно — camelCase:
int myBestScore;
String firstName;
double currentBalance;
```

**Правила для цифр — обычно в конце:**
```java
int score1;
int player2;
```

> [!warning]
> Локальные переменные **всегда начинаются с маленькой буквы**, даже если это аббревиатура.
