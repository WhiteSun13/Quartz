## 🎯 Задачи занятия

> [!important]
> Работа ведётся со **справочником «Сотрудники»**. Документ «Заказ клиента» на этом этапе закрыт — изменений в него вносить не будем.

### 📋 Две задачи от условного заказчика

| № | Задача | Где реализуется |
|---|---|---|
| 1 | **Формировать ФИО** (Фамилия И.О.) из трёх реквизитов: Фамилия, Имя, Отчество — при нажатии кнопки «Записать» | Модуль объекта |
| 2 | **Выводить возраст** сотрудника в заголовке карточки | Модуль менеджера |

> [!note]
> **ФИО** — это стандартный реквизит «Наименование», которому ранее был задан синоним «ФИО». В коде мы обращаемся к нему по системному имени — `Наименование`.

## 🔑 Модуль объекта и модуль менеджера — разница

### Где найти модули

1. Открыть дерево конфигурации → **Справочники** → **Сотрудники**
2. Двойной клик → вкладка **«Прочие»**
3. Сверху — две кнопки: **«Модуль объекта»** и **«Модуль менеджера»**

*Альтернатива:* правая кнопка мыши по «Сотрудники» → выбрать нужный модуль

<скриншот вкладки "Прочие" справочника Сотрудники с кнопками "Модуль объекта" и "Модуль менеджера">

### Сравнение модулей

| Характеристика | Модуль объекта | Модуль менеджера |
|---|---|---|
| **На что влияет** | На **один конкретный элемент** справочника | На **все элементы** справочника |
| **Когда запускается** | При открытии/записи **конкретной карточки** | При открытии **всего списка** справочника |
| **Аналогия** | Открыли карточку Механизмова → алгоритм работает только для него | Открыли «Сотрудники компании» → алгоритм отработал для всех записей |

> [!tip]
> **Простая аналогия:**
> - **Модуль менеджера:** открыли список — окошко появилось у **всех** сотрудников
> - **Модуль объекта:** открыли конкретного Механизмова — окошко появилось только **у него**

## 🛠 Задача 1: Формирование ФИО (Модуль объекта)

### Обработчики событий модуля объекта

При нажатии лупы в модуле объекта доступны следующие обработчики:

| Обработчик | Назначение |
|---|---|
| **Перед записью** | Дозаполнение данных перед сохранением (по стандартам — для подстановки вычисленных значений) |
| **При записи** | Контроль данных: проверка заполненности реквизитов |

> [!important]
> По стандартам 1С:
> - **Перед записью** — используется для **дозаполнения** данных
> - **При записи** — используется для **контроля** правильности данных

### Шаги создания обработчика

1. В **модуле объекта** нажать **лупу**
2. Выбрать **«Перед записью»** → нажать **«Перейти»**
3. Написать код внутри созданной процедуры

### Код формирования ФИО

```bsl
Процедура ПередЗаписью(Отказ)
    Наименование = СтрШаблон("%1 %2.%3.",
        СокрЛП(Фамилия),
        Лев(ВРег(СокрЛП(Имя)), 1),
        Лев(ВРег(СокрЛП(Отчество)), 1));
КонецПроцедуры
```

### Разбор алгоритма

| Элемент | Описание |
|---|---|
| `Наименование` | Стандартный реквизит справочника (синоним «ФИО») |
| `СтрШаблон("%1 %2.%3.", ...)` | Формирует строку по шаблону, подставляя значения на метки `%1`, `%2`, `%3` |
| `СокрЛП(Фамилия)` | Убирает лишние пробелы слева и справа у реквизита «Фамилия» |
| `Лев(Имя, 1)` | Берёт первый символ слева из строки — первую букву имени |
| `ВРег(...)` | Приводит символ к **верхнему регистру** |

> [!tip]
> В модуле объекта к реквизитам справочника обращаются **напрямую** — просто пишем имя реквизита без дополнительных путей:
> ```bsl
> Фамилия    // реквизит Фамилия
> Имя        // реквизит Имя
> Отчество   // реквизит Отчество
> Наименование // стандартный реквизит
> ```

> [!warning]
> Если отчество не заполнено — на его место будет подставлена **пустота**, и в строке появится лишняя точка. Для корректной обработки необходимо добавить проверку на заполненность отчества.

## ⚙️ Снятие проверки заполнения реквизита «Наименование»

> [!danger]
> **Проблема:** При нажатии «Записать» платформа проверяет, заполнено ли «Наименование». Поскольку мы заполняем его программно **в момент записи** — проверка срабатывает раньше алгоритма и выдаёт ошибку.

### Решение: отключить проверку заполнения

1. Раскрыть **Справочник «Сотрудники»**
2. Вкладка **«Данные»** → **«Стандартные реквизиты»**
3. Двойной клик по реквизиту **«Наименование»**
4. Перейти на вкладку **«Представление»**
5. Свойство **«Проверка заполнения»** → выбрать **«Не проверять»**

<скриншот окна свойств реквизита "Наименование" с выбором "Не проверять" в поле "Проверка заполнения">

### Настройка проверки для реквизитов Фамилия, Имя, Отчество

Для каждого реквизита: двойной клик → вкладка **«Представление»** → **«Проверка заполнения»**:

| Реквизит | Проверка заполнения |
|---|---|
| **Фамилия** | Выдавать ошибку |
| **Имя** | Выдавать ошибку |
| **Отчество** | Не проверять *(отчество может отсутствовать)* |

## 🛠 Задача 2: Отображение возраста в заголовке (Модуль менеджера)

### Обработчики событий модуля менеджера

При нажатии лупы в модуле менеджера доступны два ключевых обработчика:

| Обработчик | Назначение |
|---|---|
| **ОбработкаПолученияПолейПредставления** | Указывает, **какие реквизиты** будут использоваться для формирования заголовка |
| **ОбработкаПолученияПредставления** | **Формирует сам заголовок** на основе полученных данных |

### Параметры обработчиков

**ОбработкаПолученияПолейПредставления:**

| Параметр | Описание |
|---|---|
| `Поля` | Массив — сюда добавляем имена реквизитов, которые нужны для заголовка |
| `СтандартнаяОбработка` | Если `Ложь` — отключает стандартное поведение платформы |

**ОбработкаПолученияПредставления:**

| Параметр | Описание |
|---|---|
| `Данные` | Содержит реквизиты, добавленные через `Поля` в первом обработчике |
| `Представление` | Сюда помещаем **итоговый заголовок** — то, что увидит пользователь |
| `СтандартнаяОбработка` | Если `Ложь` — отключает стандартное поведение платформы |

> [!important]
> Чтобы переопределить стандартный заголовок платформы — в **обоих** обработчиках нужно написать:
> ```bsl
> СтандартнаяОбработка = Ложь;
> ```

---

### Код обработчика «ОбработкаПолученияПолейПредставления»

```bsl
Процедура ОбработкаПолученияПолейПредставления(Поля, СтандартнаяОбработка)
    СтандартнаяОбработка = Ложь;
    Поля.Добавить("Наименование");
    Поля.Добавить("ДатаРождения");
КонецПроцедуры
```

> [!note]
> Реквизиты добавляются в `Поля` как **строки** (в двойных кавычках). Контекстная подсказка здесь не работает — `Поля` является параметром-массивом, тип которого платформа не распознаёт автоматически.

> [!tip]
> Чтобы избежать опечаток в именах реквизитов — можно **зажать левую кнопку мыши** на нужном реквизите в дереве конфигурации и **перетащить** его в модуль. Платформа автоматически подставит корректное имя.

---

### Алгоритм расчёта возраста

> [!info]
> **Идея алгоритма:**
> 1. Текущая дата − Дата рождения = количество **секунд**
> 2. Прибавить секунды к пустой дате (01.01.0001) → получить дату
> 3. Вычесть один лишний день (86 400 секунд) — пустая дата содержит один день
> 4. Применить функцию `Год()` → получить **число полных лет**
> 5. Вычесть `1` — корректировка на один лишний год

```bsl
// Шаг 1: Разница в секундах
ВозрастВСекундах = ТекущаяДата() - Данные.ДатаРождения;

// Шаг 2: Прибавляем секунды к пустой дате
Возраст1 = '00010101' + ВозрастВСекундах;

// Шаг 3: Убираем лишний день (пустая дата = 1 день = 86400 секунд)
Возраст2 = Возраст1 - 86400;

// Шаг 4: Берём только год
Возраст3 = Год(Возраст2);

// Шаг 5: Корректируем на 1 (иначе возраст будет на 1 год больше)
Возраст3 = Возраст3 - 1;
```

> [!warning]
> Без вычитания `1` в шаге 5 — возраст рассчитывается на **один год больше**, чем реальный. Например, в день рождения программа показывала бы `32` вместо `31`.

---

### Алгоритм склонения слова «лет/год/года»

#### 📌 Таблица форм слова

| Последняя цифра числа | Форма слова | Примеры |
|---|---|---|
| `0` | **лет** | 0 лет, 20 лет, 30 лет |
| `1` | **год** | 1 год, 21 год, 31 год |
| `2–4` | **года** | 2 года, 22 года, 33 года |
| `5–9` | **лет** | 5 лет, 25 лет, 19 лет |
| Возраст `11–14` | **лет** | 11 лет, 13 лет, 14 лет *(исключение!)* |

> [!important]
> **Исключение:** числа от **11 до 14** всегда требуют формы «**лет**», несмотря на последнюю цифру (11 лет, 12 лет, 13 лет, 14 лет).

#### Получение последней цифры числа

```bsl
// Остаток от деления на 10 = последняя цифра
ПоследняяЦифра = Возраст3 % 10;
```

> [!note]
> Символ `%` — это операция **остатка от деления**. Например: `35 % 10 = 5`, `21 % 10 = 1`, `9 % 10 = 9`.

#### Код алгоритма склонения

```bsl
ПоследняяЦифра = Возраст3 % 10;

Если ПоследняяЦифра = 0
    Или (ПоследняяЦифра >= 5 И ПоследняяЦифра <= 9)
    Или (Возраст3 >= 11 И Возраст3 <= 14) Тогда
    ФормаСлова = "лет";
ИначеЕсли ПоследняяЦифра = 1 Тогда
    ФормаСлова = "год";
ИначеЕсли ПоследняяЦифра >= 2 И ПоследняяЦифра <= 4 Тогда
    ФормаСлова = "года";
КонецЕсли;
```

---

### Полный код «ОбработкаПолученияПредставления»

```bsl
Процедура ОбработкаПолученияПредставления(Данные, Представление, СтандартнаяОбработка)
    СтандартнаяОбработка = Ложь;

    Если Данные.ДатаРождения > '00010101' Тогда
        // Расчёт возраста
        ВозрастВСекундах = ТекущаяДата() - Данные.ДатаРождения;
        Возраст1 = '00010101' + ВозрастВСекундах;
        Возраст2 = Возраст1 - 86400;
        Возраст3 = Год(Возраст2) - 1;

        // Определение формы слова
        ПоследняяЦифра = Возраст3 % 10;

        Если ПоследняяЦифра = 0
            Или (ПоследняяЦифра >= 5 И ПоследняяЦифра <= 9)
            Или (Возраст3 >= 11 И Возраст3 <= 14) Тогда
            ФормаСлова = "лет";
        ИначеЕсли ПоследняяЦифра = 1 Тогда
            ФормаСлова = "год";
        ИначеЕсли ПоследняяЦифра >= 2 И ПоследняяЦифра <= 4 Тогда
            ФормаСлова = "года";
        КонецЕсли;

        // Формирование заголовка
        Представление = СтрШаблон("%1 (%2 %3)",
            Данные.Наименование,
            Возраст3,
            ФормаСлова);
    Иначе
        // Дата рождения не заполнена — стандартное представление
        Представление = Данные.Наименование;
        // Сообщить("Необходимо ввести дату рождения"); // закомментировать в продакшне
    КонецЕсли;
КонецПроцедуры
```

> [!success]
> **Результат:** В заголовке карточки сотрудника отображается:
> `Петров П.П. (22 года)` или `Петров П.П. (45 лет)` или `Петров П.П. (21 год)`

## 💡 Разбор вопросов с занятия

### Почему не используем модуль объекта для расчёта возраста?

> **Модуль объекта** — алгоритм отработает только при **открытии конкретной карточки** сотрудника.  
> **Модуль менеджера** — алгоритм отработает при **открытии всего списка** — сразу для **всех** сотрудников.  
> В обоих случаях результат в итоге будет сохранён. Выбор модуля менеджера здесь сделан **для наглядной демонстрации разницы** между двумя модулями.

### Почему `Поля` не показывает контекстную подсказку?

> Параметр `Поля` является **самописным параметром** — платформа не знает его тип, поэтому подсказку не выдаёт. Мы предполагаем, что это массив, и обращаемся к нему через метод `.Добавить()`.

### Почему в шаблоне нельзя сразу написать переменную `ФормаСлова`?

> `ФормаСлова` — переменная типа **строка**. `СтрШаблон` подставляет значения только на **метки** (`%1`, `%2`, `%3`). Поэтому используем метку `%3` и передаём переменную третьим параметром.

### Как работает алгоритм проверки `11–14`?

> Числа 11, 12, 13, 14 — **исключение** из правила «последняя цифра». Хотя последняя цифра у них 1, 2, 3, 4 — они требуют формы **«лет»** (11 лет, 12 лет, 13 лет, 14 лет).  
> Поэтому проверяем **не последнюю цифру**, а сам возраст целиком: `Возраст3 >= 11 И Возраст3 <= 14`.
