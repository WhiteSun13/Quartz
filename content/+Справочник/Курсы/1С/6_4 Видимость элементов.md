## 💡 Управление видимостью реквизитов через код

### Сценарий: переключатель скрывает/показывает реквизит

> [!important]
> Все свойства элементов формы, которые отображаются в окне свойств конфигуратора, **доступны программно через код**.

**Логика работы:**
- Нажали первый вариант переключателя → реквизит **отображается**
- Нажали второй вариант переключателя → реквизит **скрыт**

### Код управления видимостью

```bsl
// Пример: флажок (галочка) управляет видимостью реквизита
Если Галочка = Истина Тогда
    Элементы.НазваниеРеквизита.Видимость = Истина;
Иначе
    Элементы.НазваниеРеквизита.Видимость = Ложь;
КонецЕсли;
```

### Свойство «Видимость»

| Значение | Результат |
|---|---|
| `Истина` | Элемент **отображается** на форме |
| `Ложь` | Элемент **скрыт** с формы |

> [!tip]
> Путь к свойству: `Элементы.<ИмяЭлемента>.Видимость`  
> Аналогично работают и другие свойства: `Доступность`, `ТолькоПросмотр` и т.д.

> [!note]
> Существуют свойства, которые **не отображаются в окне конфигуратора**, но доступны программно. Для их поиска — использовать Синтакс-помощник или поисковые системы.

## 🐛 Исправление ошибки: директива компиляции в модуле объекта

> [!warning]
> Если процедура в **модуле объекта** работает с данными формы и выдаёт ошибку — возможно, не указана директива компиляции.

**Решение:** перед процедурой добавить директиву `&НаКлиенте`:

```bsl
&НаКлиенте
Процедура НазваниеПроцедуры()
    // код
КонецПроцедуры
```

## 📝 Разбор самостоятельной работы №1: Сообщения при изменении полей

### Условие задачи

> Требуется выводить сообщение об изменении полей: **Номер, Дата, Покупатель, Договор**.  
> Формат сообщения: `Изменено поле: <Название поля>`

### Решение: простые обработчики для каждого поля

**Шаги для каждого поля:**
1. Открыть **Форму документа** «Заказ клиента»
2. Двойной клик по элементу (Номер / Дата / Покупатель / Договор)
3. Вкладка **«События»** → **«При изменении»** → лупа → **«Создать на клиенте»**
4. Написать код

```bsl
&НаКлиенте
Процедура НомерПриИзменении(Элемент)
    Сообщить("Изменено поле: Номер");
КонецПроцедуры

&НаКлиенте
Процедура ДатаПриИзменении(Элемент)
    Сообщить("Изменено поле: Дата");
КонецПроцедуры

&НаКлиенте
Процедура ПокупательПриИзменении(Элемент)
    Сообщить("Изменено поле: Покупатель");
КонецПроцедуры

&НаКлиенте
Процедура ДоговорПриИзменении(Элемент)
    Сообщить("Изменено поле: Договор");
КонецПроцедуры
```

> [!success]
> **Результат:** при изменении любого из четырёх полей в нижней части экрана появляется соответствующее сообщение.

## 🔑 Улучшение: Универсальная процедура

> [!tip]
> Вместо четырёх одинаковых обработчиков — создать **одну универсальную процедуру** с параметром, и вызывать её из каждого обработчика.

### Код универсальной процедуры

```bsl
&НаКлиенте
Процедура ВыводитьСообщениеОбИзменении(НазваниеПоля)
    Сообщить("Изменено поле: " + НазваниеПоля);
КонецПроцедуры

&НаКлиенте
Процедура НомерПриИзменении(Элемент)
    ВыводитьСообщениеОбИзменении("Номер");
КонецПроцедуры

&НаКлиенте
Процедура ДатаПриИзменении(Элемент)
    ВыводитьСообщениеОбИзменении("Дата");
КонецПроцедуры

&НаКлиенте
Процедура ПокупательПриИзменении(Элемент)
    ВыводитьСообщениеОбИзменении("Покупатель");
КонецПроцедуры

&НаКлиенте
Процедура ДоговорПриИзменении(Элемент)
    ВыводитьСообщениеОбИзменении("Договор");
КонецПроцедуры
```

> [!success]
> Вместо дублирования строки `Сообщить(...)` в каждом обработчике — **один вызов универсальной процедуры** с нужным параметром.

## 📝 Самостоятельная работа №2: Квадраты чисел (Цикл)

### Условие задачи

> Дано натуральное число (целое, больше нуля).  
> Требуется **вывести все квадраты чисел** от 1 до исходного числа включительно.  
>
> **Пример:** число = 10 → вывести квадраты от 1² до 10²

### Решение

```bsl
// Выполняется в модуле конфигурации
// (предварительно закомментировать весь имеющийся код: Ctrl+A → Ctrl+/)

Число = 10; // Исходное число

Для Счётчик = 1 По Число Цикл
    Квадрат = Pow(Счётчик, 2); // Возведение в степень 2
    Сообщить("Квадрат числа " + Счётчик + " = " + Квадрат);
КонецЦикла;
```

> [!note]
> Вместо функции `Pow(Счётчик, 2)` можно использовать простое умножение:
> ```bsl
> Квадрат = Счётчик * Счётчик;
> ```
> Результат будет одинаковым.

### Разбор алгоритма

| Элемент | Описание |
|---|---|
| `Число = 10` | Исходное число — задаёт верхнюю границу диапазона |
| `Для Счётчик = 1 По Число` | Цикл со счётчиком от 1 до заданного числа |
| `Pow(Счётчик, 2)` | Функция возведения в степень: основание = Счётчик, показатель = 2 |
| `Сообщить(...)` | Вывод результата пользователю |

### Результат выполнения (для числа 10)

```
Квадрат числа 1 = 1
Квадрат числа 2 = 4
Квадрат числа 3 = 9
Квадрат числа 4 = 16
Квадрат числа 5 = 25
Квадрат числа 6 = 36
Квадрат числа 7 = 49
Квадрат числа 8 = 64
Квадрат числа 9 = 81
Квадрат числа 10 = 100
```

> [!tip]
> Чтобы изменить диапазон — достаточно изменить одну строку:
> ```bsl
> Число = 16; // теперь считаем от 1 до 16
> ```

> [!warning]
> **Частая ошибка:** после `Счётчик` в строке `Сообщить` пропущен `+` перед следующей частью строки:
> ```bsl
> // НЕПРАВИЛЬНО:
> Сообщить("Квадрат числа " + Счётчик " = " + Квадрат);
>
> // ПРАВИЛЬНО:
> Сообщить("Квадрат числа " + Счётчик + " = " + Квадрат);
> ```

> [!info]

   - Юридическое наименование организации
