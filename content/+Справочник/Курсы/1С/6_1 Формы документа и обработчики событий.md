## 📋 Повторение пройденного материала

### 🔑 Синтаксические конструкции, изученные на предыдущем занятии

| Конструкция | Описание |
|---|---|
| **Блок условия (ЕСЛИ)** | Основной блок; подблоки `ИНАЧЕ` и `ИНАЧЕЕСЛИ` — опциональны |
| **Цикл со счётчиком (ДЛЯ)** | Цикл с заданным количеством итераций |
| **Цикл с условием (ПОКА)** | Цикл, выполняющийся пока истинно условие |
| **ПРОДОЛЖИТЬ** | Пропуск текущей итерации цикла |
| **ПРЕРВАТЬ** | Завершение цикла досрочно |
| **Процедуры** | Блок алгоритма без возвращаемого значения |
| **Функции** | Блок алгоритма с возвращаемым значением |
| **Обработчики событий** | Процедуры/функции, привязанные к конкретному событию |
| **Области** | Логические «папки» для структурирования кода |

> [!important]
> **Процедура** — вынесенный алгоритм, который просто вызывается.  
> **Функция** — вынесенный алгоритм, который **возвращает значение** для присвоения в переменную.

> [!note]
> **Области** — используются для соответствия стандартам разработки, чтобы другие разработчики легко ориентировались в коде.

## 🎯 Задача занятия

**Контекст:** В документе «Заказ клиента» созданы реквизиты и табличная часть, но **сумма не рассчитывалась автоматически**.

**Формула расчёта:**
```
Сумма = Количество × Цена
```

**Требования к поведению:**
- При изменении **Количества** → пересчитать Сумму
- При изменении **Цены** → пересчитать Сумму
- Поле **Сумма** должно быть **заблокировано** от ручного ввода

## 🛠 Создание формы документа

### Шаги создания формы

1. Открыть дерево конфигурации → ветка **Документы**
2. Раскрыть **Заказ клиента**
3. Перейти на вкладку **Формы**
4. Нажать **лупу** возле пункта «Документ»
5. Откроется **Конструктор формы документа**:
   - Тип — не трогать
   - Галочка **«Назначить форму основной»** — стоит автоматически
   - Нажать **Далее**
6. Убедиться, что состав реквизитов устраивает
7. Нажать **Готово**

> [!success]
> Созданная форма **замещает форму по умолчанию** и позволяет реализовывать собственные алгоритмы поведения.

## ⚡ Создание обработчика события «При изменении Количества»

### Привязка обработчика к колонке

1. В окне редактирования формы **выделить колонку «Количество»** в табличной части «Товары»
2. Нажать **правой кнопкой мыши** → выбрать **События**
   - *Альтернатива:* двойной клик по элементу → вкладка **«События»*
3. Найти событие **«При изменении»**
4. Нажать **лупу** рядом с «При изменении»
5. Выбрать вариант **«Создать на клиенте»**
6. Нажать **OK**

> [!warning]
> Если алгоритм написан, но **не исполняется** — скорее всего, поле события **пустое**. Обязательно проверяйте, что обработчик привязан к событию элемента!

![[Pasted image 20260516170251.png]]

### Понятие «Клиент» и «Сервер»

| Контекст | Описание | Пример |
|---|---|---|
| **На клиенте** | Всё, что происходит «перед глазами» пользователя | Заполнение формы, ввод данных |
| **На сервере** | Операции с базой данных | Запись, чтение данных из БД |
| **Без контекста** | На сервер передаётся **не весь контекст** формы — только нужные данные | Оптимизированные серверные вызовы |

> [!tip]
> Даже в **файловом варианте** базы принято использовать клиент-серверную архитектуру — для простоты переноса на реальный сервер в будущем.
>
> Аналогия: **Сервер** = системный блок, **Клиент** = монитор.

## 💡 Программный код расчёта суммы

### Обращение к реквизитам табличной части

> [!important]
> Путь к данным: от **большего** к **меньшему**:
> ```
> Элементы → Товары (табличная часть) → ТекущиеДанные → Конкретная колонка
> ```

### Код обработчика «При изменении Количества»

```bsl
&НаКлиенте
Процедура ТоварыКоличествоПриИзменении(Элемент)
    РасчётСуммы();
КонецПроцедуры
```

### Код обработчика «При изменении Цены»

```bsl
&НаКлиенте
Процедура ТоварыЦенаПриИзменении(Элемент)
    РасчётСуммы();
КонецПроцедуры
```

### Вынесенная процедура расчёта суммы

```bsl
&НаКлиенте
Процедура РасчётСуммы()
    СтрокаРасчёта = Элементы.Товары.ТекущиеДанные;
    СтрокаРасчёта.Сумма = СтрокаРасчёта.Цена * СтрокаРасчёта.Количество;
КонецПроцедуры
```

> [!note]
> **Почему через `Элементы`, а не через `Объект.Товары`?**
> 
> - `Объект.Товары` — работает с **уже записанными** в базу данными  
> - `Элементы.Товары.ТекущиеДанные` — работает с **текущей, ещё не записанной** строкой, которую пользователь вводит прямо сейчас

> [!warning]
> **`ТекущиеДанные`** недоступны на сервере! Процедуры, использующие `ТекущиеДанные`, **обязательно** должны иметь директиву `&НаКлиенте`.

## 🔑 Директивы компиляции

### Добавление директивы вручную

1. Написать слово `direct` (или встать перед процедурой)
2. Нажать **Ctrl+Q**
3. Выбрать нужный вариант из списка

### Варианты директив

| Директива | Описание |
|---|---|
| `&НаКлиенте` | Выполняется на стороне клиента |
| `&НаСервере` | Выполняется на сервере (передаёт весь контекст формы) |
| `&НаСервереБезКонтекста` | Выполняется на сервере, передаёт только нужные данные |
| `&НаКлиентеНаСервере` | Выполняется и там, и там |

> [!tip]
> Директива `&НаСервереБезКонтекста` **оптимальнее** по нагрузке, чем `&НаСервере`, так как передаёт меньше данных. Однако в большинстве практических случаев используется `&НаСервере`.

## 🛠 Блокировка поля «Сумма» от редактирования

### Способ: установить свойство «Только просмотр»

1. Выделить элемент **«Сумма»** в колонке Товары
2. Перейти на вкладку **Основное**
3. Поставить галочку **«Только просмотр»**

### Разница между свойствами

| Свойство | Поведение |
|---|---|
| **Только просмотр** | Элемент кликабелен, можно **копировать** значение, но **изменить нельзя** |
| **Доступность** | Элемент **некликабелен** вообще — нельзя даже выделить ячейку |
| **Видимость** | Колонка **полностью скрыта** с формы |

> [!success]
> Для блокировки суммы от редактирования используется **«Только просмотр»** — пользователь видит значение и может его скопировать, но изменить не может.

## 🐛 Отладка ошибок

### Два режима запуска

| Режим | Кнопка | Для кого | Особенности |
|---|---|---|---|
| **Пользовательский режим** | 🟡 Жёлтый кружок с жёлтым треугольником | Пользователи | При ошибке — возможность «Сформировать отчёт об ошибке» |
| **Режим отладки** | 🟡 Жёлтый кружок с синим треугольником | Разработчики | При ошибке — кнопка **«Конфигуратор»**: сразу переходит в модуль на строку с ошибкой |

> [!tip]
> **Алгоритм разработчика при ошибке в режиме отладки:**
> 1. Возникла ошибка → нажать **«Подробно»**
> 2. Нажать кнопку **«Конфигуратор»**
> 3. Курсор автоматически встанет на **строку с ошибкой** в модуле

> [!tip]
> **Алгоритм пользователя при ошибке:**
> 4. Нажать **«Сформировать отчёт об ошибке»**
> 5. Скопировать **подробный текст ошибки**
> 6. Передать разработчику

## 📂 Структурирование кода: Области

### Зачем нужны области

- Группируют код в логические блоки — как «папки»
- Позволяют **сворачивать** разделы кода
- Обязательны для соответствия **стандартам разработки 1С**

### Создание области

```bsl
#Область ОбработчикиСобытийЭлементовФормы

// ... обработчики событий ...

#КонецОбласти

#Область СлужебныеПроцедурыИФункции

// ... вспомогательные процедуры ...

#КонецОбласти
```

> [!warning]
> В названии области **не должно быть пробелов**! Используйте `ОбработчикиСобытийЭлементовФормы` вместо `Обработчики событий элементов формы`.

### Быстрое создание области

1. Написать `#Область` (или аббревиатуру `obl`)
2. Нажать **Ctrl+Q** — откроется помощник

### Если области не сворачиваются

1. Меню **Сервис** → **Параметры**
2. Вкладка **Модули** → **Группировка**
3. Поставить галочку **«Области сворачивать»**
4. Нажать **Применить**

## 📋 Итоговая структура кода модуля формы

```bsl
#Область ОбработчикиСобытийЭлементовФормы

&НаКлиенте
Процедура ТоварыКоличествоПриИзменении(Элемент)
    РасчётСуммы();
КонецПроцедуры

&НаКлиенте
Процедура ТоварыЦенаПриИзменении(Элемент)
    РасчётСуммы();
КонецПроцедуры

#КонецОбласти

#Область СлужебныеПроцедурыИФункции

&НаКлиенте
Процедура РасчётСуммы()
    СтрокаРасчёта = Элементы.Товары.ТекущиеДанные;
    СтрокаРасчёта.Сумма = СтрокаРасчёта.Цена * СтрокаРасчёта.Количество;
КонецПроцедуры

#КонецОбласти
```

> [!important]
> **Принцип вынесения общего алгоритма:**  
> Если несколько обработчиков событий делают **одно и то же** — вынесите логику в **отдельную процедуру** и вызывайте её из каждого обработчика. Это исключает дублирование кода.

> [!tip]
> В идеале процедуру `РасчётСуммы()` следует вынести в **Общий модуль** с ключевым словом `Экспорт` — тогда её можно будет использовать в любых документах конфигурации.

## ❓ Разбор вопросов с занятия

### Почему система «знает», к какой колонке относится обработчик?

> Привязка происходит **не по имени процедуры**, а через **свойство события элемента формы**. Мы выделяем конкретный элемент (колонку), переходим на вкладку «События» и там указываем имя процедуры-обработчика. Именно эта связь и определяет, что сработает при изменении.

### Почему нельзя вызвать один обработчик из другого напрямую?

> Технически можно, но **правильным решением** является вынесение общей логики в отдельную процедуру. Два обработчика вызывают одну общую процедуру.

### Когда использовать `&НаСервере`, а когда `&НаСервереБезКонтекста`?

> `&НаСервереБезКонтекста` **оптимальнее** по производительности (передаёт меньше данных), но требует явной передачи нужных параметров. `&НаСервере` удобнее, но нагружает систему больше.

### Можно ли поместить расчёт суммы в модуль приложения?

> **Нет.** Модуль приложения предназначен исключительно для событий **запуска** и **завершения** приложения. Вычисления на форме должны быть в **модуле формы**.
