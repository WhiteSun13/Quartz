## 🐛 Исправление ошибки: порядок функций СокрЛП и Лев

### Суть проблемы

> [!warning]
> Если пользователь вводит имя или отчество с **пробелами в начале**, функция `Лев()` возьмёт **первый символ — пробел**, а не букву. В итоге инициал пропадёт.

**Пример:**  
Ввод: `_Пётр` (с пробелом перед именем)  
Неправильный порядок: `Лев(ВРег(Имя), 1)` → берёт пробел → инициал «П» **пропадает**

### Правильный порядок функций

```
Сначала убрать пробелы → потом взять первый символ → потом привести к верхнему регистру
```

```bsl
// НЕПРАВИЛЬНО:
Лев(ВРег(Имя), 1)

// ПРАВИЛЬНО:
ВРег(Лев(СокрЛП(Имя), 1))
```

### Итоговый исправленный код формирования ФИО

```bsl
Процедура ПередЗаписью(Отказ)
    Наименование = СтрШаблон("%1 %2.%3.",
        СокрЛП(Фамилия),
        ВРег(Лев(СокрЛП(Имя), 1)),
        ВРег(Лев(СокрЛП(Отчество), 1)));
КонецПроцедуры
```

> [!note]
> С **фамилией** порядок функций некритичен — мы её используем целиком, а не берём отдельный символ. Поэтому `СокрЛП(Фамилия)` достаточно.

## 🛠 Задача 1: Реквизит «Итоговая сумма»

### Создание реквизита

1. Раскрыть **Документы** → двойной клик по **«Заказ клиента»**
2. Перейти на вкладку **«Данные»**
3. Нажать **«+»** (добавить реквизит)
4. Имя: **`ИтоговаяСумма`**, синоним: **«Итоговая сумма»**
5. Тип: нажать **«...»** → выбрать **«Денежный тип»** (длина 15, точность 2, неотрицательное)
6. Нажать **ОК**

### Добавление реквизита на форму

> [!tip]
> Если реквизит создан **после** создания формы — он автоматически на форму **не попадает**. Нужно перетащить вручную.

1. Открыть **Форму документа** (вкладка «Формы»)
2. В правой части найти **«Реквизиты»** → раскрыть **«Объект»**
3. Найти **«ИтоговаяСумма»**
4. **Зажать левую кнопку мыши** и перетащить в область элементов формы

### Когда выполняется расчёт

> [!info]
> Расчёт итоговой суммы должен происходить **при нажатии кнопки «Записать»**, то есть в обработчике **«Перед записью»** модуля объекта документа.

### Открытие модуля объекта документа

1. Вкладка **«Прочие»** документа «Заказ клиента»
2. Кнопка **«Модуль объекта»**
3. Нажать **лупу** → выбрать **«Перед записью»**

### Код расчёта итоговой суммы

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)
    ИтоговаяСумма = Товары.Итог("Сумма") + Услуги.Итог("Сумма");
КонецПроцедуры
```

### Разбор метода `.Итог()`

| Элемент | Описание |
|---|---|
| `Товары` | Обращение к табличной части «Товары» |
| `.Итог("Сумма")` | Суммирует значения всех строк в колонке «Сумма» |
| `Услуги.Итог("Сумма")` | То же самое для табличной части «Услуги» |
| `+` | Складываем две суммы → итог по всему документу |

> [!tip]
> Параметр метода `.Итог()` — это **имя колонки в виде строки** (в двойных кавычках) или её индекс.  
> Можно посмотреть описание в **Синтакс-помощнике**: Табличная часть → `.Итог()` → «Суммирует значения всех строк в указанной колонке».

## 🔑 Параметр «Отказ» в обработчике «Перед записью»

> [!important]
> **Параметр `Отказ`** — если установить в значение `Истина`, то **вся процедура аннулируется**: данные не сохраняются, документ не записывается, пользователю выводится сообщение «Не удалось записать».

### Пример использования

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)
    ИтоговаяСумма = Товары.Итог("Сумма") + Услуги.Итог("Сумма");

    // Пример: запрет записи если сумма > 10 000
    Если ИтоговаяСумма > 10000 Тогда
        Отказ = Истина;
    КонецЕсли;
КонецПроцедуры
```

> [!note]
> По умолчанию параметр `Отказ` **всегда содержит `Ложь`** — писать `Отказ = Ложь` явно не нужно.

> [!tip]
> **Практическое применение параметра `Отказ`:** при проверке отрицательных остатков на складе. Если товара не хватает — `Отказ = Истина`, документ не проводится, данные в регистр не попадают.

## 🔑 Параметр «РежимЗаписи» и «РежимПроведения»

### Режимы записи документа

```bsl
РежимЗаписиДокумента.Запись           // Просто сохраняет данные документа
РежимЗаписиДокумента.Проведение       // Сохраняет + переносит данные в регистры
РежимЗаписиДокумента.ОтменаПроведения // Убирает галочку проведения + удаляет данные из регистров
```

### Режимы проведения документа

```bsl
РежимПроведенияДокумента.Оперативный    // Работает в режиме реального времени, будущей датой записать нельзя
РежимПроведенияДокумента.Неоперативный  // Можно записывать прошлой и будущей датой
```

| Режим | Описание |
|---|---|
| **Оперативный** | По умолчанию. Проверяет остатки «здесь и сейчас». Нельзя установить будущую дату |
| **Неоперативный** | Для ввода «задним числом». Дата может быть любой |

> [!note]
> Настройки проведения задаются в свойствах документа на вкладке **«Движение»**: «Проведение разрешено», «Оперативное проведение разрешено».

## 🛠 Задача 2: Подвал табличной части

> [!info]
> **Подвал** — серая полоска под табличной частью, где автоматически считается итог по выбранной колонке.  
> Работает **только в табличных частях** (не в динамических списках).

<скриншот документа «Заказ клиента» с подвалом «Итого: 145 000» под табличной частью «Товары»>

### Шаг 1: Включить подвал для табличной части

1. В области элементов формы — двойной клик по **«Товары»** (табличная часть)
2. Перейти на вкладку **«Использование»**
3. Поставить флажок **«Подвал»**
4. В окне предпросмотра снизу появится серая полоска

### Шаг 2: Настроить колонку «Сумма» в подвале

1. Раскрыть табличную часть **«Товары»** в области элементов
2. Двойной клик по элементу **«ТоварыСумма»**
3. Перейти на вкладку **«Основное»**
4. Прокрутить вниз — найти два свойства:

| Свойство | Значение |
|---|---|
| **Текст подвала** | `Итого:` |
| **Путь к данным подвала** | Выбрать через «...» → Объект → Товары → **Итог Сумма** |

5. Нажать **ОК**

> [!success]
> Путь к данным подвала будет выглядеть так: `Объект.Товары.ИтогСумма`

### Повторить для табличной части «Услуги»

Те же шаги, но:
- Выделяем **«Услуги»** → вкладка «Использование» → **Подвал** ✅
- Раскрываем **«Услуги»** → двойной клик по **«УслугиСумма»**
- Путь к данным подвала: **Объект → Услуги → Итог Сумма**
- Текст подвала: `Итого:`

### Настройка ширины колонки

> [!tip]
> Если «Итоговая сумма» не помещается и появляется горизонтальный скролл:
> - В **пользовательском режиме**: кнопка **«Ещё»** → **«Изменить форму»** → задать ширину
> - В **конфигураторе**: выделить элемент → вкладка **«Расположение»** → свойство **«Ширина»** (0 = автоматически)

## 🛠 Создание документов «Расходная накладная» и «Приходная накладная»

### Метод: копирование существующего документа

> [!tip]
> Вместо создания с нуля — **копируем** «Заказ клиента». Копируются все реквизиты, табличные части и весь код модуля формы.

1. Выделить **«Заказ клиента»** в дереве конфигурации
2. Правая кнопка мыши → **«Скопировать»**
3. Появится **«ЗаказКлиента1»**

### Проверка после копирования

> [!warning]
> После копирования обязательно проверьте:
> 1. Откройте форму скопированного документа
> 2. Выделите любой элемент → вкладка **«Основное»** → проверьте **«Путь к данным»** (должен быть корректным)
> 3. Вкладка **«События»** → проверьте, что обработчики **привязаны** (поле не пустое)

> [!danger]
> **Частая проблема:** после копирования события могут оказаться **непривязанными** — поле события пустое. Нужно вручную выбрать нужную процедуру для каждой колонки.

---

### Создание «Расходной накладной»

1. Двойной клик по **«ЗаказКлиента1»**
2. Поменять **Имя**: `РасходнаяНакладная`
3. **Представление объекта**: `Расходная накладная`
4. **Представление списка**: `Расходные накладные`
5. Нажать **ОК**

---

### Создание «Приходной накладной»

1. Снова скопировать **«Заказ клиента»** → появится **«ЗаказКлиента1»**
2. Двойной клик → поменять **Имя**: `ПриходнаяНакладная`
3. **Представление объекта**: `Приходная накладная`
4. **Представление списка**: `Приходные накладные`

#### Удаление табличной части «Услуги» из Приходной накладной

> [!info]
> По условию задачи: в приходной накладной **нет табличной части «Услуги»** — на склад услуги не приходуются.

**Попытка удалить → срабатывает контроль ссылочной целостности:**

> [!warning]
> При попытке удалить табличную часть «Услуги» платформа выдаёт сообщение:  
> *«Код модулей содержит ссылки на удаляемый объект. Продолжить?»*  
> Нажать **«Нет»** — и сначала убрать ссылку из кода.

#### Поиск ссылок на объект

1. Выделить **«Услуги»** → правая кнопка мыши
2. Два варианта:

| Пункт меню | Что делает |
|---|---|
| **Поиск ссылок на объект** | Ищет, **где используется** табличная часть «Услуги» |
| **Поиск ссылок в объекте** | Ищет, **что используется внутри** табличной части «Услуги» |

3. Выбрать **«Поиск ссылок на объект»** → нажать **ОК**
4. В служебных сообщениях увидим: ссылка находится в **модуле объекта** приходной накладной

#### Исправление кода в модуле объекта

```bsl
// БЫЛО (в ПередЗаписью):
ИтоговаяСумма = Товары.Итог("Сумма") + Услуги.Итог("Сумма");

// СТАЛО (для Приходной накладной — без услуг):
ИтоговаяСумма = Товары.Итог("Сумма");
```

После удаления строки с `Услуги`:
1. Перейти на вкладку **«Данные»**
2. Выделить **«Услуги»** → нажать **крестик**
3. Сообщений об ошибках больше нет → подтвердить удаление: **«Да»**

> [!success]
> Табличная часть «Услуги» успешно удалена из «Приходной накладной».

## 📝 Самостоятельная работа

> [!important]
> **Срок выполнения:** 10 минут (до 21:00)  
> **Документ:** «Заказ клиента»

### Задание

Реализовать **вывод сообщения при изменении** следующих полей:

| Поле | Сообщение при изменении |
|---|---|
| Номер | `Изменено поле: Номер` |
| Дата | `Изменено поле: Дата` |
| Покупатель | `Изменено поле: Покупатель` |
| Договор | `Изменено поле: Договор` |

### Подсказка

```bsl
// Для каждого поля создать обработчик «При изменении»
// и внутри использовать функцию Сообщить():

&НаКлиенте
Процедура НомерПриИзменении(Элемент)
    Сообщить("Изменено поле: Номер");
КонецПроцедуры
```

> [!tip]
> Шаги для каждого поля:
> 1. Выделить элемент формы (Номер / Дата / Покупатель / Договор)
> 2. Правая кнопка мыши → **«События»** → **«При изменении»**
> 3. Нажать лупу → **«Создать на клиенте»**
> 4. Написать `Сообщить("Изменено поле: [Название поля]");`

> [!warning]
> **Частая ошибка начинающих:** если «Итоговая сумма» или «Подвал» не отображаются — проверьте **горизонтальный скролл** внизу формы. Реквизит может просто находиться за пределами видимой области.

> [!note]
> **Реквизит «Основание»** в расходной накладной (ссылка на заказ клиента) — отдельный механизм, будет реализован на следующем занятии.

> [!info]
> **Подвал работает только в табличных частях.** В динамических списках (например, в списке заказов клиентов) подвал реализовать таким способом **невозможно**.
