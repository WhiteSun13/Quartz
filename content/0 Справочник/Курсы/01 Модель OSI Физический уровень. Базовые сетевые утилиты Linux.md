## 📌 Введение

Все мы ежедневно в своей жизни так или иначе используем компьютерные сети — сеть Интернет для работы, отдыха или просто общения.

> [!important] Основная задача практикума
> Разобраться, **как всё работает**: какие процессы происходят в сети, если мы, например, отправляем данные с точки А в точку Б, и **как можно ими управлять**.

Понимание принципов работы сетевых технологий довольно важно для IT-специалистов в целом, чтобы:
- Вся подконтрольная им инфраструктура **не давала сбоев**
- Они могли проводить **быстрый трабблшутинг** возникших проблем
- Своевременно **устранять** неисправности

## 🌐 Что такое компьютерная сеть?

> [!quote] Определение
> **Любая сеть** — это совокупность **устройств** и **каналов связи** между ними.

### 📋 Классификация устройств

Все устройства делятся на **две категории**:

| Тип | Описание |
|---|---|
| **Промежуточные** | Устройства, обеспечивающие передачу данных между конечными узлами |
| **Конечные** | Устройства — источники или получатели данных |

Соединяются эти устройства путём организации **проводных** или **беспроводных** каналов связи.

### 🛠 Что нужно для создания сети?

1. **Физически соединить** устройства между собой с помощью каналов связи
2. **Организовать логику** передачи данных между ними

> [!note] Как организуется логика?
> Для организации логики используются различные **сетевые службы и протоколы**, о которых будет рассказано более детально в следующих заданиях практикума.

### 📡 Сетевые интерфейсы

Для физического подключения устройств к сети у них должен быть:
- Хотя бы **один сетевой интерфейс**
- **Процессор** или чип для обработки данных

> [!info]
> Сетевые интерфейсы необязательно могут быть **физическими** — они могут быть **логическими** или **программными**. Для диагностики и настройки таких интерфейсов есть ряд утилит, с помощью которых можно просматривать и настраивать параметры этих интерфейсов.

## 🔧 Утилита `ethtool`

### 📌 Назначение

**`ethtool`** — утилита, которая выступает заменой устаревших утилит. Позволяет:
- Получать информацию о **физических параметрах** интерфейса
- Каким-то образом **менять** эти параметры
- Работает с **физическим уровнем**

### 🛠 Установка

```bash
apt install ethtool
```

### 📋 Использование

```bash
ethtool eth0
```

### 🔍 Разбор параметров вывода

| Параметр | Значение | Описание |
|---|---|---|
| **speed** | 10 / 100 / 1000 | Скорость передачи данных на интерфейсе (Мбит/с) |
| **duplex** | `full` / `half` | Режим дуплекса |
| **port** | `twisted pair` / оптика | Тип подключённой среды передачи данных |
| **auto-negotiation** | `on` / `off` | Автосогласование параметров между интерфейсами |
| **link detected** | `yes` / `no` | Наличие подключённого и активного кабеля |

> [!note] Режимы дуплекса
> - **Full-duplex** — в один момент времени интерфейс может **одновременно передавать и получать** информацию
> - **Half-duplex** — в один момент времени интерфейс может **либо отправлять, либо получать** информацию

> [!info] Параметр auto-negotiation
> Два интерфейса, находящиеся на **разных концах** одной среды, **обмениваются служебной информацией** о том, какая у них скорость и какой дуплекс, и **согласовывают их между собой** для более эффективной работы в конкретной среде передачи данных.

### 📌 Параметры физической машины (расширенный вывод)

> [!note] Дополнительные параметры на физических машинах
> - **Поддерживаемые параметры** — какая скорость, какой дуплекс поддерживается
> - **Support Wake-on-LAN** — позволяет **удалённо запускать систему** с помощью сети. Когда «**магический пакет**» прилетает на сетевой интерфейс, он отправляет команду на запуск системы — очень удобно, если нужно включить машину удалённо, когда других способов нет

### 💡 Полезная фишка

> [!tip] Мигание лампочек
> При использовании ключа `-p` с указанием интерфейса утилита позволит **настроить мигание лампочек** на физическом интерфейсе, что позволит **визуально отличить** нужный интерфейс. Очень полезно при настройке серверов с множеством сетевых адаптеров.

```bash
ethtool -p eth0
```

## 🔧 Пакет `iproute2`

### 📌 Назначение

Пакет **`iproute2`** включает в себя большое количество утилит, скриптов и инструментов. В отличие от `ethtool`, работает уже **не с физическим уровнем**, а с **канальным и сетевым**.

### 📋 Основные утилиты пакета

| Утилита | Назначение |
|---|---|
| **`ip`** | Получение информации о параметрах интерфейса L2/L3, изменение параметров |
| **`tc`** (traffic control) | Просмотр конфигурации параметров управления трафиком |
| **`ss`** (аналог `netstat`) | Просмотр текущих соединений и открытых портов на устройстве |

### 🛠 Установка

```bash
apt install iproute2
```

---

### 🔍 Утилита `ip`

#### Команда `ip link` — просмотр всех интерфейсов

```bash
ip link
```

**Что видим в выводе:**
- Список **всех интерфейсов** в системе
- **MAC-адрес** каждого интерфейса
- Ключ **`UP`** — интерфейс **включён**
- Ключ **`DOWN`** — интерфейс **выключен**

> [!info] Интерфейс loopback (`lo`)
> Всегда присутствует в системе, он **логический**. Используется:
> - Для **тестирования протоколов** в самой системе
> - Для **управляющего IP-адреса**, через который подключаются к устройству в случае недоступности одного из физических портов

#### Команда `ip link show dev` — информация о конкретном интерфейсе

```bash
ip link show dev eth0
```

> [!tip]
> Очень полезно, если в системе **большое количество интерфейсов** и нужно вывести информацию только об одном.

#### Команда `ip a` (`ip address`) — просмотр IP-адресов

```bash
ip a
```

**Выводит значительно больше информации, чем `ip link`:**
- **IP-адреса**, настроенные на каждом интерфейсе
- **Маска подсети** (через слэш после IP-адреса)
- **Broadcast-адрес**

> [!note] Стандартные адреса
> - `127.0.0.1` — стандартный **loopback**-адрес

#### Управление состоянием интерфейса

```bash
# Выключить интерфейс
ip link set eth0 down

# Включить интерфейс
ip link set eth0 up
```

> [!tip] Программное управление
> Таким образом можно **программно включать или выключать** интерфейсы, не прибегая к физическому извлечению кабеля.

## 🔧 Пакет `net-tools` (устаревший)

### 📌 Назначение

Пакет **`net-tools`** — более старый пакет, который считается **устаревшим**. На его замену пришёл пакет `iproute2`. Тем не менее утилиты из него широко используются администраторами.

### 📋 Основные утилиты пакета

| Утилита | Назначение | Современный аналог |
|---|---|---|
| **`ifconfig`** | Главная утилита настройки сетевых интерфейсов (аналог `ipconfig` в Windows) | `ip` |
| **`netstat`** | Получение отчёта о сетевых соединениях | `ss` |
| **`route`** | Работа с таблицей IP-маршрутизации | `ip route` |

### 🛠 Установка

```bash
apt install net-tools
```

### 🔍 Команда `ifconfig`

```bash
# Все интерфейсы
ifconfig

# Конкретный интерфейс
ifconfig eth0
```

> [!info] Подробный вывод `ifconfig`
> Вывод **более подробный**, чем у `ip link`. Сразу видны:
> - **IP-адрес**
> - **MAC-адрес**
> - **Маска сети**
> - Количество **переданных и принятых пакетов**
> - Количество **ошибок и дропов**
> 
> Эта служебная информация очень полезна в процессе **трабблшутинга или настройки**.

## 🔧 Утилита `ping`

### 📌 Назначение

> [!important]
> **`ping`** — команда, являющаяся чуть ли не **первостепенным инструментом** для трабблшутинга сети и выявления проблем на начальном уровне.

Работает по протоколу **ICMP** (отправляет ICMP-запросы).

### 🛠 Установка

```bash
apt install iputils-ping
```

### 📋 Использование

```bash
ping 192.168.0.3
```

Для остановки нажать **Ctrl + C**.

### 🔍 Разбор параметров вывода

#### Параметр RTT (Round Trip Time)

> [!quote] Определение
> **RTT** — время прохождения ICMP-запроса **в одну сторону плюс ответа в обратном** направлении.

| Значение RTT | Интерпретация |
|---|---|
| **Сотые миллисекунд — ~3 мс** | Норма для **LAN-сети** (локальной сети) |
| **Растёт** | Сеть под **нагрузкой** или хост находится **на большом удалении** через множество устройств |
| **Высокое (>10 мс) для ближнего шлюза** | Сеть **нагружена** на этом участке |
| **Скачет от 10 до 1000+ мс + пропадают пакеты** | **Неисправность** в сети (например, петля) — нужно срочно разбираться |

#### Параметр TTL (Time To Live)

> [!quote] Определение
> **TTL** — параметр, присваиваемый интерфейсом при отправке ICMP-запроса или ответа. Отвечает за то, **сколько маршрутизаторов** может преодолеть ICMP-пакет по пути до нужного узла.

- Если значение TTL **падает до 0** — ICMP-запрос **уничтожается** (тем самым исключается заполнение сети бесконечными пингами)
- В выводе `ping` мы видим **TTL входящих пакетов** от узла, на который отправляем пинг

> [!info] Стандартные значения TTL по ОС
> | Операционная система | Стандартный TTL |
> |---|---|
> | **Linux** | 64 |
> | **Windows** | 128 |
> | **Cisco** | 255 |

> [!tip] Определение ОС по TTL
> По значению TTL в ответном пакете можно **предположить операционную систему** удалённого узла:
> - TTL = **64** → вероятно **Linux** со стандартными настройками
> - TTL = **128** → вероятно **Windows**
>
> **Формула расчёта:** TTL в ответе = TTL на узле − количество промежуточных маршрутизаторов (каждый уменьшает TTL на 1)

## 📋 Итоговая сводка инструментов

| Утилита | Пакет | Уровень | Назначение |
|---|---|---|---|
| **`ethtool`** | `ethtool` | Физический (L1) | Параметры физического интерфейса: скорость, дуплекс, кабель |
| **`ip`** | `iproute2` | Канальный/Сетевой (L2/L3) | Просмотр и настройка интерфейсов, IP-адресов |
| **`tc`** | `iproute2` | — | Управление трафиком |
| **`ss`** | `iproute2` | — | Текущие соединения и открытые порты |
| **`ifconfig`** | `net-tools` *(устар.)* | Канальный/Сетевой | Настройка сетевых интерфейсов, подробный вывод |
| **`netstat`** | `net-tools` *(устар.)* | — | Отчёт о сетевых соединениях |
| **`route`** | `net-tools` *(устар.)* | — | Работа с таблицей маршрутизации |
| **`ping`** | `iputils-ping` | Сетевой (L3) | Трабблшутинг: доступность узла, RTT, анализ TTL |

> [!success] Итог
> Сегодня разобраны несколько важных инструментов, которые позволяют как **настраивать сетевые параметры**, так и **выявлять неисправности**. Они являются **неотъемлемыми инструментами** каждого сетевого инженера или администратора. Как и с любым инструментом — главное **правильно ими пользоваться** для решения тех или иных задач. Получив базовые знания и научившись владеть необходимыми инструментами, вам ничего не стоит построить даже самую сложную сеть.
