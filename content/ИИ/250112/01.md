# Требования для ELIBRARYSYSTEM

Вот список требований в текстовом формате для ручного добавления в MATLAB Requirements Editor или System Composer.

---

## 1. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ - АУТЕНТИФИКАЦИЯ

### REQ-AUTH-001
**Название:** Аутентификация пользователей через LDAP
**Тип:** Functional
**Текст:** Система должна обеспечивать аутентификацию пользователей через университетский LDAP-сервер с использованием интерфейса IUserCredentials.
**Компоненты:** AuthenticationService, AuthenticationClient, UniversityLDAP

### REQ-AUTH-002
**Название:** Генерация токенов доступа
**Тип:** Functional
**Текст:** После успешной аутентификации система должна генерировать JWT-токен доступа (accessToken) и токен обновления (refreshToken) в соответствии с интерфейсом IAuthToken.
**Компоненты:** AuthenticationService

### REQ-AUTH-003
**Название:** Время жизни токена
**Тип:** Functional
**Текст:** Поле expiresIn интерфейса IAuthToken должно содержать время жизни токена в секундах. Токен доступа должен быть действителен не менее 3600 секунд.
**Компоненты:** AuthenticationService, AuthenticationClient

### REQ-AUTH-004
**Название:** Передача токена для синхронизации
**Тип:** Functional
**Текст:** Компонент AuthenticationClient должен передавать полученный токен аутентификации компоненту SyncEngine через порт TokenOut.
**Компоненты:** AuthenticationClient, SyncEngine

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ - ПОИСК

### REQ-SEARCH-001
**Название:** Обработка поисковых запросов
**Тип:** Functional
**Текст:** Система должна принимать поисковые запросы от клиента через интерфейс ISearchQuery и возвращать результаты через интерфейс ISearchResults.
**Компоненты:** SearchEngineClient, SearchService, ElasticsearchCluster

### REQ-SEARCH-002
**Название:** Полнотекстовый поиск
**Тип:** Functional
**Текст:** SearchService должен выполнять полнотекстовый поиск в ElasticsearchCluster по полю queryText интерфейса ISearchQuery.
**Компоненты:** SearchService, ElasticsearchCluster

### REQ-SEARCH-003
**Название:** Пагинация результатов поиска
**Тип:** Functional
**Текст:** Система должна поддерживать пагинацию результатов поиска с использованием полей page и pageSize интерфейса ISearchQuery.
**Компоненты:** SearchService

### REQ-SEARCH-004
**Название:** Метаданные книг в результатах
**Тип:** Functional
**Текст:** Результаты поиска (ISearchResults) должны содержать массив метаданных книг в формате IBookMetadata, включая bookId, title, authors и isbn.
**Компоненты:** SearchService, CatalogService

## ИНСТРУКЦИЯ ПО ДОБАВЛЕНИЮ ТРЕБОВАНИЙ В MATLAB

### Способ 1: Requirements Editor
1. Откройте **Requirements Editor**: `Apps` → `Requirements Editor`
2. Создайте новый набор требований: `New` → `Requirements Set`
3. Для каждого требования нажмите `Add Requirement`
4. Заполните поля:
   - **ID**: например, REQ-AUTH-001
   - **Summary**: название требования
   - **Description**: текст требования
   - **Type**: Functional/Performance/Security/etc.

### Способ 2: Через командную строку MATLAB
```matlab
% Создание набора требований
rs = slreq.new('ELibraryRequirements');

% Добавление требования
req = add(rs, 'Type', 'Requirement');
set(req, 'Id', 'REQ-AUTH-001');
set(req, 'Summary', 'Аутентификация пользователей через LDAP');
set(req, 'Description', 'Система должна обеспечивать аутентификацию пользователей через университетский LDAP-сервер с использованием интерфейса IUserCredentials.');

% Сохранение
save(rs);
```

### Способ 3: Связывание с компонентами
После добавления требований свяжите их с компонентами в System Composer:
1. Откройте модель архитектуры
2. Выберите компонент
3. Правый клик → `Requirements` → `Link to Requirement`
4. Выберите соответствующее требование
