Вот таблицы для ручного создания архитектуры в System Composer (или Matlab). Я доработал компонент **Client**, добавил недостающие порты внутри подкомпонентов и составил полную карту соединений, чтобы **все порты были соединены**.

### 1. Таблица Компонентов (Структура Client)

Создайте эти компоненты внутри `Client`.

| Имя Компонента             | Тип       | Описание                                                    |
| :------------------------- | :-------- | :---------------------------------------------------------- |
| **AuthenticationClient**   | Component | Управление входом и токенами.                               |
| **BookViewer**             | Component | Отображение книги, управление закладками и перелистыванием. |
| **BookmarkManager**        | Component | Управление закладками (создание, удаление).                 |
| **DownloadManager**        | Component | Загрузка контента и передача в хранилище.                   |
| **LocalCache**             | Component | Локальный кэш метаданных и закладок.                        |
| **OfflineStorage**         | Component | Хранилище файлов книг (BLOB).                               |
| **ReadingProgressTracker** | Component | Отслеживание прогресса чтения для аналитики.                |
| **SearchEngineClient**     | Component | Интерфейс поиска.                                           |
| **SyncEngine**             | Component | Синхронизация данных с сервером.                            |

### 3. Таблица Соединений (Connections)

Соедините порты следующим образом. Источник (Source) -> Приемник (Target).

#### Соединения с границей (Boundary Connections)

| Source (Откуда)                         | Target (Куда)                          | Описание                                  |
| :-------------------------------------- | :------------------------------------- | :---------------------------------------- |
| **Client/UserCredentialsIn**            | **AuthenticationClient/CredentialsIn** | Вход пользователя передается в AuthClient |
| **AuthenticationClient/CredentialsOut** | **Client/AuthRequestOut**              | Запрос авторизации уходит на сервер       |
| **Client/AuthResponseIn**               | **AuthenticationClient/TokenIn**       | Токен приходит с сервера                  |
| **Client/SearchQueryIn**                | **SearchEngineClient/QueryIn**         | Поисковый запрос от UI                    |
| **SearchEngineClient/QueryOut**         | **Client/SearchRequestOut**            | Запрос поиска уходит на сервер            |
| **Client/SearchResponseIn**             | **SearchEngineClient/ResultsIn**       | Результаты приходят с сервера             |
| **SearchEngineClient/ResultsOut**       | **Client/SearchResultsOut**            | Результаты уходят в UI                    |
| **Client/ContentRequestIn**             | **DownloadManager/ExternalContentIn**  | Данные книги (байты) приходят с сервера   |
| **DownloadManager/RequestOut**          | **Client/ContentRequestOut**           | Запрос на скачивание уходит на сервер     |
| **Client/DownloadRequestIn**            | **DownloadManager/DownloadCommandIn**  | Команда "Скачать" от UI                   |
| **Client/SyncDataIn**                   | **SyncEngine/RemoteSyncIn**            | Данные синхронизации с сервера            |
| **SyncEngine/RemoteSyncOut**            | **Client/SyncDataOut**                 | Данные синхронизации на сервер            |
| **ReadingProgressTracker/AnalyticsOut** | **Client/AnalyticsOut**                | Аналитика (прогресс) уходит вовне         |

#### Внутренние соединения (Internal Connections)

| Source (Откуда) | Target (Куда) | Описание |
| :--- | :--- | :--- |
| **AuthenticationClient/TokenOut** | **SyncEngine/TokenIn** | Токен нужен для синхронизации |
| **BookViewer/BookmarkOut** | **BookmarkManager/BookmarkDataIn** | Сохранение закладки |
| **BookViewer/ContentRequestOut** | **DownloadManager/ViewerRequestIn** | Viewer просит подгрузить страницу |
| **BookViewer/ProgressOut** | **ReadingProgressTracker/ProgressIn** | Viewer сообщает о перелистывании |
| **BookmarkManager/SyncOut** | **LocalCache/CacheRequestIn** | Закладки сохраняются в кэш |
| **LocalCache/CacheResponseOut** | **BookmarkManager/SyncIn** | Кэш возвращает данные закладок |
| **LocalCache/ToSyncEngineOut** | **SyncEngine/LocalChangesIn** | Изменения кэша идут на синхронизацию |
| **SyncEngine/LocalUpdateOut** | **LocalCache/UpdateFromSyncIn** | Обновления с сервера идут в кэш |
| **DownloadManager/ToStorageOut** | **OfflineStorage/StoreIn** | Скачанный файл сохраняется на диск |
| **OfflineStorage/RetrieveOut** | **BookViewer/ContentIn** | Viewer читает файл с диска |
| **ReadingProgressTracker/SyncDataOut** | **SyncEngine/ProgressDataIn** | Прогресс чтения сохраняется в облако |

### Инструкция по доработке:
1.  Убедитесь, что все интерфейсы (`IUserCredentials`, `IBookContent` и т.д.) определены в словаре данных.
2.  Добавьте недостающие порты (помечены как **Новый** или уточнены в таблице портов) в соответствующие компоненты.
3.  Проведите линии соединений согласно таблице №3. Теперь у компонента `Client` нет "висящих" портов, и поток данных (Data Flow) логически замкнут.
