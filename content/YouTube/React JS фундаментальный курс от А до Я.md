## Теоретическая часть

### Для чего нужен React?

### Что такое React?

**React** — это библиотека для создания пользовательских интерфейсов. Среди начинающих часто бытует мнение, что React — это именно библиотека для создания web-приложений, но это не совсем так. React спроектирован таким образом, что он независим от браузера; он живёт сам по себе.

- **`react` (ядро):** отвечает за создание пользовательских интерфейсов и содержит основные концепции.
- **`react-dom`:** отвечает за визуализацию (рендеринг, отрисовку) React-компонентов в браузере.
- **`react-native`:** позволяет создавать приложения на Android, iOS и даже на Windows.

Таким образом, ядро React содержит большое количество концепций, а за визуализацию в том или ином окружении отвечают другие библиотеки.

### Multi-Page vs Single-Page Applications (MPA vs SPA)

#### Классические приложения (MPA - Multi-Page Application)
Мы создаём несколько HTML-файлов. Каждый HTML-файл — это отдельная страница в нашем приложении. Между страницами мы переключаемся с помощью ссылок.
- **Минусы:**
    - При каждом переходе загружается полностью новый HTML-файл, даже если общие части (навигационная панель, футер) остаются прежними.
    - Пользователь может видеть "скачки" и обновления окна браузера при медленном интернете.

#### Одностраничные приложения (SPA - Single-Page Application)
Во всем приложении мы имеем один HTML-файл (обычно `index.html`). Это минималистичный файл, все содержимое которого приходит из JavaScript. Мы управляем содержимым страницы динамически: что-то удаляем, что-то добавляем.
- **Плюсы:**
    - **Быстрая работа:** все необходимые JavaScript-файлы подгружаются изначально. Общие части (navbar, footer) не подгружаются заново при каждом переходе.
    - Приложение получается быстрым и отзывчивым.
- **Минусы:**
    - **Первичный размер файла больше:** на первичную загрузку тратится больше времени, так как нужно подгрузить большое количество данных. Однако это можно оптимизировать (настройка Webpack, lazy loading).

### Компонентный подход

React работает по принципу **компонентного подхода**. Мы создаем компоненты и используем их как кирпичики, вкладывая один компонент в другой для создания сложных интерфейсов. Компоненты можно переиспользовать.

Всегда есть какой-то **корневой компонент** (чаще всего называется `App`), который монтируется в `index.html`.

### Как React упрощает разработку?

Без фреймворков часто приходится напрямую взаимодействовать с **DOM-деревом**: получать элементы, добавлять слушатели, изменять содержимое. Это увеличивает вероятность ошибок и время разработки. Операции с DOM-деревом очень ресурсоемкие.

React позволяет разработчику больше сосредотачиваться на логике работы с данными. Взаимодействие с DOM-деревом минимизировано. Вы изменяете данные, а интерфейс под эти данные подстраивается.

### Внутренние механизмы React

1.  **Виртуальное DOM-дерево (Дерево React-элементов):** React строит в памяти легковесную копию обычного DOM-дерева. Сами разработчики предпочитают называть это "деревом React-элементов", так как оно не зависит от браузера.
2.  **Сравнение деревьев:** Когда в данных происходят изменения, React строит новое дерево элементов с обновленными значениями и сравнивает его с предыдущим. Этот процесс называется **"согласование" (reconciliation)**. За него отвечает ядро React (`react`).
3.  **Визуализация (Рендеринг):** После того как React нашел разницу между деревьями, происходит фаза визуализации. Изменения вносятся в реальное DOM-дерево. За эту фазу отвечает `react-dom`. React устанавливает приоритетность для изменений, что обеспечивает плавную картинку.

### JSX (JavaScript XML)

**JSX** — это расширение синтаксиса JavaScript, которое позволяет писать HTML-подобную разметку внутри JS-файлов.

```javascript
// С JSX
const element = (
  <div>
    <h1>Hello!</h1>
    <button>Click me</button>
  </div>
);

// Без JSX (с использованием React.createElement)
const elementWithoutJSX = React.createElement(
  'div',
  null,
  React.createElement('h1', null, 'Hello!'),
  React.createElement('button', { disabled: true, onClick: () => console.log('click') }, 'Click me')
);
```

Код с JSX гораздо читабельнее.

## Компоненты, состояние и props

### Управление состоянием (`useState`)

Проблема: если просто изменять переменную внутри компонента, React не узнает об этом и не перерисует интерфейс.

```javascript
function App() {
  let likes = 0;

  function increment() {
    likes += 1;
    console.log(likes); // В консоли значение меняется
  }
  
  // В интерфейсе значение не меняется
  return (
    <div>
      <h1>{likes}</h1> 
      <button onClick={increment}>Increment</button>
    </div>
  );
}```

Чтобы React отслеживал изменения, нужно использовать **состояние**. Для этого предназначен хук `useState`.

- **Хук (Hook)** — это специальная функция, которая позволяет "подцепиться" к состоянию и жизненному циклу React.

```javascript
import React, { useState } from 'react';

function App() {
  // useState возвращает массив из двух элементов:
  // 1. Само значение состояния (likes)
  // 2. Функция для его изменения (setLikes)
  // В скобках указывается начальное значение.
  const [likes, setLikes] = useState(0);

  function increment() {
    // Изменяем состояние через специальную функцию
    setLikes(likes + 1);
  }

  function decrement() {
    setLikes(likes - 1);
  }

  return (
    <div>
      <h1>{likes}</h1>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
}
```

Теперь при вызове `setLikes()` React понимает, что состояние изменилось, и перерисовывает компонент.

### Управляемые компоненты

Это компоненты (например, `<input>`), значение которых контролируется React через состояние.

```javascript
const [value, setValue] = useState('Текст в инпуте');

//...
return (
  <div>
    <h1>{value}</h1>
    <input 
      type="text" 
      value={value} // Связываем значение инпута с состоянием
      onChange={event => setValue(event.target.value)} // Обновляем состояние при вводе
    />
  </div>
);
```
Это называется **двусторонним связыванием**. Значение в `input` всегда синхронизировано с состоянием.

### Создание и декомпозиция компонентов

Компоненты можно выносить в отдельные файлы для переиспользования и лучшей организации кода.

1.  Создаем папку `src/components`.
2.  Создаем файл `src/components/Counter.jsx`.
    - Название компонента и файла пишется с большой буквы.
    - Расширение `.jsx` явно указывает на использование JSX.

```javascript
// src/components/Counter.jsx
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);
  
  function increment() {
    setCount(count + 1);
  }

  function decrement() {
    setCount(count - 1);
  }

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
};

export default Counter; // Не забываем экспортировать компонент
```

```javascript
// src/App.js
import React from 'react';
import Counter from './components/Counter';

function App() {
  return (
    <div className="App">
      <Counter />
      <Counter />
      <Counter />
    </div>
  );
}

export default App;
```
Теперь у нас три независимых счетчика.

### Функциональные vs Классовые компоненты

- **Функциональные компоненты (с хуками):** современный и рекомендуемый подход. Это просто JavaScript-функции.
- **Классовые компоненты:** устаревший подход, но встречается в старых проектах.

```javascript
// Классовый компонент
import React from 'react';

class ClassCounter extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    }
    // Привязка контекста this
    this.increment = this.increment.bind(this);
    this.decrement = this.decrement.bind(this);
  }

  increment() {
    this.setState({count: this.state.count + 1});
  }

  decrement() {
    this.setState({count: this.state.count - 1});
  }
  
  // Метод render обязателен
  render() {
    return (
      <div>
        <h1>{this.state.count}</h1>
        <button onClick={this.increment}>Increment</button>
        <button onClick={this.decrement}>Decrement</button>
      </div>
    );
  }
}

export default ClassCounter;
```
Функциональные компоненты с хуками более лаконичны и просты для понимания.

### Правила хуков
- Хуки всегда начинаются со слова `use`.
- Хуки можно вызывать только на верхнем уровне вложенности функциональных компонентов или других кастомных хуков. Их нельзя вызывать внутри условий, циклов или вложенных функций.

## Стилизация, Props и рендеринг списков

### Стилизация
- **CSS-классы:** Используется атрибут `className` вместо `class`, так как `class` — зарезервированное слово в JavaScript.
- **Инлайн-стили:** Передаются как объект JavaScript. Свойства в `camelCase` (например, `textAlign`).
  ```jsx
  <h1 style={{ textAlign: 'center', color: 'red' }}>Заголовок</h1>
  ```
- **CSS-модули:** Способ изоляции стилей. Файл называется `Component.module.css`. Стили импортируются как объект.
  ```javascript
  import classes from './MyButton.module.css';
  // ...
  <button className={classes.myBtn}>Кнопка</button>
  ```
  Это генерирует уникальные имена классов, предотвращая конфликты.

### Props (Свойства)

**Props** — это входные данные, которые компонент получает от родителя. Они передаются как атрибуты в JSX. Это механизм передачи данных **сверху вниз** (от родителя к дочернему компоненту).

```javascript
// Родительский компонент App.js
import PostItem from './components/PostItem';

function App() {
  const post = { id: 1, title: 'JavaScript', body: 'Описание поста' };
  
  return (
    <div>
      {/* Передаем объект post как props */}
      <PostItem post={post} /> 
    </div>
  );
}
```

```javascript
// Дочерний компонент PostItem.jsx
import React from 'react';

const PostItem = (props) => {
  // props - это объект { post: { id: 1, ... } }
  return (
    <div className="post">
      <div className="post__content">
        <strong>{props.post.id}. {props.post.title}</strong>
        <div>
          {props.post.body}
        </div>
      </div>
    </div>
  );
};

export default PostItem;
```
Можно использовать деструктуризацию для удобства: `const PostItem = ({ post }) => { ... }`.

### Рендеринг списков

Для отрисовки массива данных используется метод `.map()`, который преобразует каждый элемент массива в React-элемент.

```javascript
const [posts, setPosts] = useState([
  { id: 1, title: 'JS', body: '...' },
  { id: 2, title: 'Python', body: '...' },
]);

// ...
return (
  <div>
    {posts.map(post => 
      // Обязательно указываем уникальный `key` для каждого элемента списка
      <PostItem post={post} key={post.id} />
    )}
  </div>
);
```
**Атрибут `key`** помогает React эффективно обновлять списки, определяя, какие элементы изменились, добавились или удалились. В качестве ключа нужно использовать уникальное и стабильное значение, обычно это `id` из данных. Использовать индекс элемента (`key={index}`) не рекомендуется.

### Передача данных снизу вверх (Callbacks)

Props передаются только сверху вниз. Чтобы дочерний компонент мог передать данные родителю, родитель передает дочернему компоненту **callback-функцию** через props.

```javascript
// Родительский компонент App.js
function App() {
  const [posts, setPosts] = useState([...]);
  
  // Функция для создания нового поста
  const createPost = (newPost) => {
    setPosts([...posts, newPost]);
  }
  
  return (
    <div>
      {/* Передаем callback-функцию как props */}
      <PostForm create={createPost} />
      <PostList posts={posts} />
    </div>
  );
}
```

```javascript
// Дочерний компонент PostForm.jsx
const PostForm = ({ create }) => {
  const [post, setPost] = useState({ title: '', body: '' });

  const addNewPost = (e) => {
    e.preventDefault();
    const newPost = { ...post, id: Date.now() };
    // Вызываем callback-функцию, полученную от родителя, и передаем в нее данные
    create(newPost);
    setPost({ title: '', body: '' });
  };
  
  // ...
};
```

## Продвинутые хуки и оптимизация

### Неуправляемые компоненты и `useRef`

**`useRef`** — это хук, который позволяет получить прямой доступ к DOM-элементу. Он возвращает объект с одним полем `current`.

```javascript
const bodyInputRef = useRef();

const readValue = () => {
  console.log(bodyInputRef.current.value);
};

// ...
<MyInput ref={bodyInputRef} type="text" />
```
Такой компонент называется **неуправляемым**, так как его значение не контролируется состоянием React.

**`forwardRef`** — функция, которая позволяет "пробрасывать" `ref` через кастомный компонент к DOM-элементу внутри него.

### `useMemo`

**`useMemo`** — это хук, который **мемоизирует** (кэширует) результат выполнения функции. Он перезапускает функцию только тогда, когда изменяется одна из его зависимостей. Это используется для оптимизации производительности, чтобы избежать ресурсоемких вычислений при каждом рендере.

```javascript
const sortedPosts = useMemo(() => {
  console.log('ФУНКЦИЯ СОРТИРОВКИ ОТРАБОТАЛА');
  if (filter.sort) {
    return [...posts].sort((a, b) => a[filter.sort].localeCompare(b[filter.sort]));
  }
  return posts;
}, [filter.sort, posts]); // Зависимости: функция выполнится, если изменится sort или posts
```

### Кастомные хуки

Это способ вынести повторяющуюся логику компонентов в переиспользуемые функции. Название кастомного хука всегда должно начинаться с `use`.

```javascript
// hooks/usePosts.js
import { useMemo } from 'react';

export const useSortedPosts = (posts, sort) => {
  // ... логика сортировки из useMemo
  return sortedPosts;
};

export const usePosts = (posts, sort, query) => {
  const sortedPosts = useSortedPosts(posts, sort);
  
  const sortedAndSearchedPosts = useMemo(() => {
    // ... логика фильтрации
    return sortedPosts.filter(post => post.title.toLowerCase().includes(query.toLowerCase()));
  }, [query, sortedPosts]);
  
  return sortedAndSearchedPosts;
};
```

## Введение в анимации

Следующим этапом предлагаю разобраться с анимациями. Я хочу сделать так, чтобы при удалении и добавлении поста в список происходила какая-нибудь плавная анимация. Допустим, при удалении пост уезжает влево, а при добавлении заезжает справа.

Для этого в React есть библиотека, которая называется **React Transition Group**. С помощью нее можно без особого труда анимировать списки и отслеживать фазы анимации (когда анимация активна, когда анимация закончилась).

### Установка и настройка

1.  **Установка пакета:**
    ```bash
    npm install react-transition-group
    ```
2.  **Запуск проекта:** После установки пакета заново запускаем проект.

### Реализация анимации списка

Для того чтобы реализовать анимацию, мы будем использовать два основных компонента из библиотеки: `TransitionGroup` и `CSSTransition`.

1.  **Оборачиваем список в `TransitionGroup`:**
    Откроем наш компонент со списком постов (`PostList.jsx`). Весь список элементов, который мы итерируем с помощью `.map()`, необходимо обернуть в компонент `TransitionGroup`.

    ```jsx
    import { TransitionGroup } from 'react-transition-group';
    // ...

    <TransitionGroup>
      {posts.map((post, index) =>
        // ...
      )}
    </TransitionGroup>
    ```

2.  **Оборачиваем каждый элемент в `CSSTransition`:**
    Каждый отдельный элемент списка (в нашем случае `PostItem`) нужно обернуть в компонент `CSSTransition`.

    ```jsx
    import { CSSTransition } from 'react-transition-group';
    // ...

    <TransitionGroup>
      {posts.map((post, index) =>
        <CSSTransition
          key={post.id}
          timeout={500} // Длительность анимации в мс
          classNames="post" // Префикс для CSS-классов
        >
          <PostItem remove={remove} number={index + 1} post={post} />
        </CSSTransition>
      )}
    </TransitionGroup>
    ```

    - **`key`:** Атрибут `key` теперь необходимо указывать на компоненте `CSSTransition`, а не на `PostItem`.
    - **`timeout`:** Задает общую длительность анимации.
    - **`classNames`:** Указывает префикс для CSS-классов, которые библиотека будет динамически добавлять к элементу на разных стадиях анимации.

### Написание CSS-стилей для анимации

`React Transition Group` работает путем добавления и удаления CSS-классов к элементам в определенные моменты времени. Названия классов формируются по шаблону: `[classNames]-enter`, `[classNames]-enter-active`, `[classNames]-exit`, `[classNames]-exit-active`.

В нашем случае `classNames="post"`, поэтому классы будут следующими:

-   `.post-enter`: Добавляется в самом начале анимации появления элемента.
-   `.post-enter-active`: Добавляется на следующем кадре после `.post-enter`, определяет саму анимацию появления.
-   `.post-exit`: Добавляется в самом начале анимации исчезновения элемента.
-   `.post-exit-active`: Добавляется на следующем кадре после `.post-exit`, определяет саму анимацию исчезновения.

Добавим соответствующие стили в наш CSS-файл:

```css
/* Начальное состояние при добавлении (элемент сдвинут и прозрачен) */
.post-enter {
  transform: translateX(-350px);
}
/* Активная фаза анимации добавления (плавный переход в нормальное состояние) */
.post-enter-active {
  transform: translateX(0px);
  transition: all 500ms ease-in;
}
/* Начальное состояние при удалении (элемент уже на месте) */
.post-exit {
  transform: translateX(0px);
}
/* Активная фаза анимации удаления (плавный сдвиг и исчезновение) */
.post-exit-active {
  transform: translateX(-350px);
  transition: all 500ms ease-in;
}
```

### Результат

Теперь, если мы попробуем добавить пост, он будет плавно выезжать слева. При удалении поста он будет плавно уезжать обратно влево. Вот такую красоту мы реализовали буквально за 3 минуты.

## Работа с сервером (API)

### `useEffect` и жизненный цикл компонента

jsonplaceholder

**`useEffect`** — хук для выполнения **сайд-эффектов** в компонентах. Сайд-эффекты — это операции, которые выходят за рамки рендеринга: запросы на сервер, подписки, работа с таймерами, прямое изменение DOM.

**Жизненный цикл компонента:**
1.  **Монтирование (Mounting):** Компонент создается и вставляется в DOM.
2.  **Обновление (Updating):** Компонент перерисовывается из-за изменений в props или state.
3.  **Размонтирование (Unmounting):** Компонент удаляется из DOM.

**`useEffect(callback, dependencies)`:**
- `callback` — функция, которая будет выполнена.
- `dependencies` — массив зависимостей.

**Варианты использования:**
- `useEffect(() => { ... }, [])`: Callback выполнится **один раз** после первого рендера (монтирования). Идеально для первоначальной загрузки данных.
- `useEffect(() => { ... }, [value])`: Callback выполнится при монтировании и **каждый раз**, когда изменится `value`.
- `useEffect(() => { ... })`: Callback выполнится при монтировании и **после каждого рендера**. Используется редко.

```javascript
import { useEffect, useState } from 'react';
import PostService from './API/PostService';

// ...
const [posts, setPosts] = useState([]);

// Выполнится один раз при монтировании компонента
useEffect(() => {
  fetchPosts();
}, []);

async function fetchPosts() {
  const posts = await PostService.getAll();
  setPosts(posts);
}
```

### Индикация загрузки и обработка ошибок

Хорошая практика — показывать пользователю индикатор загрузки (лоадер) и сообщения об ошибках.

```javascript
const [isLoading, setIsLoading] = useState(false);
const [error, setError] = useState('');

async function fetchPosts() {
  try {
    setIsLoading(true); // Показываем лоадер
    const posts = await PostService.getAll();
    setPosts(posts);
  } catch (e) {
    setError(e.message); // Сохраняем ошибку
  } finally {
    setIsLoading(false); // Скрываем лоадер в любом случае
  }
}

// ...
{error && <h1>Произошла ошибка ${error}</h1>}
{isLoading
  ? <Loader/>
  : <PostList posts={posts}/>
}
```
Эту логику можно вынести в кастомный хук `useFetching` для переиспользования.

### Пагинация

Это разбиение большого объема данных на страницы. Обычно для этого в API-запросе используются параметры, такие как `_page` (номер страницы) и `_limit` (количество элементов на странице).

### Бесконечная лента

Это динамическая подгрузка данных по мере прокрутки страницы вниз. Для ее реализации используется **Intersection Observer API**.

- Создается наблюдаемый элемент (например, `div` в конце списка) с помощью `useRef`.
- `Intersection Observer` следит за этим элементом.
- Когда элемент появляется в области видимости, вызывается callback, который загружает следующую порцию данных и увеличивает номер страницы.

Эту логику также можно вынести в кастомный хук `useObserver`.

## Глобальное состояние (Context API)

**Context API** — это встроенный в React механизм для передачи данных через дерево компонентов без необходимости передавать props на каждом уровне. Это решение проблемы **"prop drilling"**.

1.  **`React.createContext()`:** Создает объект контекста.
2.  **`<MyContext.Provider value={...}>`:** Компонент, который делает данные доступными для всех дочерних компонентов.
3.  **`useContext(MyContext)`:** Хук, который позволяет дочернему компоненту подписаться на изменения контекста и получить его значение.

Используется для данных, которые можно считать "глобальными", например, тема оформления, информация об авторизованном пользователе.

```javascript
// context/index.js
import { createContext } from 'react';
export const AuthContext = createContext(null);

// App.js
import { AuthContext } from './context';

function App() {
  const [isAuth, setIsAuth] = useState(false);
  // ...
  return (
    <AuthContext.Provider value={{
      isAuth,
      setIsAuth
    }}>
      <BrowserRouter>
        {/* ... */}
      </BrowserRouter>
    </AuthContext.Provider>
  );
}

// Navbar.jsx
import { useContext } from 'react';
import { AuthContext } from '../context';

const Navbar = () => {
  const { isAuth, setIsAuth } = useContext(AuthContext);
  // ...
};
```
